Handle errors

The error envelope

Every non-2xx response uses one shape, regardless of which endpoint failed. Switch on code — never on detail (the human message can change between releases).

{
            "type": "about:blank",
            "title": "Validation failed",
            "status": 422,
            "code": "USER_CREATE_FULL_NAME_REQUIRED",
            "detail": "Field `full_name` is required.",
            "request_id": "01HE…",
            "errors": [
            {"field": "full_name", "code": "USER_CREATE_FULL_NAME_REQUIRED", "message": "Field is required"}
            ]
            }

Fields:

  • status — HTTP status; matches the response status line.
  • code — stable string identifier (e.g. USER_CREATE_FULL_NAME_REQUIRED). Switch on this in client code.
  • detail — human-readable, English-only, may change phrasing across releases.
  • request_id — server-issued correlation id; always present, always include in support tickets.
  • errors[] — only on 422 validation responses; one entry per failed field.

What to do per status class

StatusClassRetry?What client does
400Bad request — missing header / framingNoFix the request; common causes: missing Idempotency-Key.
401Auth requiredNoCheck the X-API-Key header; do not retry the same key.
404Resource not foundNoThe composite key did not match; surface to user as "not found".
409Conflict (uniqueness, idempotency mismatch)NoTreat as final; surface to user with the message from code.
413Payload too largeNoTrim the body or split the operation.
422Validation failedNoRead errors[]; surface per-field messages.
429Rate limit exceededYes, backoffHonor Retry-After when present; exponential backoff otherwise.
5xxServer-sideYes, with Idempotency-KeyRetry the original request with the same idempotency key (max ~3 attempts, exponential backoff).

Retry pseudocode

import uuid, time, random, requests
              key = str(uuid.uuid4())
              for attempt in range(3):
              r = requests.post(
              f"{base_url}/api/v1/user",
              headers={"X-API-Key": api_key, "Idempotency-Key": key},
              json=payload,
              )
              if r.status_code < 500 and r.status_code != 429:
              break
              time.sleep((2 ** attempt) + random.random())
              return r

Always retry with the same Idempotency-Key so a partially-applied first attempt does not create a duplicate. See handle idempotency.
