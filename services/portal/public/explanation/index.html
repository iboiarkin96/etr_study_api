Understanding-oriented background

Explanation pages are understanding-oriented: they answer "can you tell me about…?". They describe context, rationale, and trade-offs — useful when you have time to learn, less so when you have a job to do (in which case see how-to guides).

  • Architecture — C4 system view and service boundaries.
  • API versioning — what additive-only guarantees the API gives across versions.
  • Security model — authentication, transport, and secret handling.
  • SLO and error budget — availability and latency commitments.
