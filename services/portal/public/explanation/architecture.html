Architecture

One-paragraph summary

ETR Study API is a single FastAPI service that exposes one HTTP surface (/api/v1), persists state to a relational store, and emits structured logs and Prometheus metrics. The repository is a monorepo with three logical services — api (the runtime), portal (this documentation site), and monitoring (Prometheus / Grafana / optional Elasticsearch + Kibana). Each ships as an independent image with a service-rooted layout under services/.

Container view (C4 level 2)

Diagrams follow the C4 model expressed in PlantUML. Three views are maintained: system context, container, and component.

  • API service — FastAPI on Uvicorn; all business logic, validation, idempotency, error formatting.
  • Persistence — SQLite by default (file in repo for local dev; ephemeral in containers). The schema is portable to PostgreSQL with no business-logic change.
  • Observability — structured JSON logs, Prometheus metrics, and /live / /ready health endpoints.
  • Portal — static HTML/CSS/JS; this site, served by any static host. No runtime coupling to the API.

Request lifecycle (one HTTP call)

  1. Client sends POST /api/v1/user with X-API-Key + Idempotency-Key.
  2. Middleware verifies the key, attaches a server-issued request_id, applies rate-limit budget.
  3. FastAPI parses the body against the Pydantic schema; failures return 422 with stable codes.
  4. The route handler calls a service-layer function that resolves the idempotency key, executes the write inside a transaction, and emits a structured log + metric on completion.
  5. A response body matching the OpenAPI schema is serialised; a stable error envelope is used on every non-2xx (see error catalogue).

What stays internal

Some surfaces are not part of the public contract and may change without notice: internal database column names (vs. response field names), structured-log field shapes, Prometheus metric names, the on-disk file layout, and any container-internal port other than API_PORT. Integrators interact only with the OpenAPI surface.
