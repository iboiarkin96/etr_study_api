Environment variables (consumer view)

What this page covers

The subset of environment variables that an external integrator may need to be aware of — typically when self-hosting the service or pointing a client at a non-default base URL. Internal-only operational variables (database, logging, Prometheus, ES) are not part of the public API contract.

Profiles

The service loads a profile based on APP_ENV: one of dev, qa, prod. The profile selects a file under env/<profile>; values already exported in the parent shell take precedence over the file (so per-test overrides survive).

VariableDefaultWhat it does
APP_ENVdevPicks the profile file from env/. Normalised to dev/qa/prod.
ENV_FILE—Optional explicit dotenv path; loaded last and overrides the profile.

Consumer-relevant variables

VariableDefaultWhat it does
API_HOST127.0.0.1Bind address; 0.0.0.0 when serving outside localhost.
API_PORT8000HTTP port. Public base URL = http://{API_HOST}:{API_PORT}.
API_KEY—Long-lived API key required in the X-API-Key header. Required in prod; convenience-only in dev.
CORS_ALLOW_ORIGINS—Comma-separated list of origins permitted to call the API from a browser. Empty in prod unless you ship a first-party web client.
API_RATE_LIMIT_REQUESTS(profile)Per-window request budget per key.
API_RATE_LIMIT_WINDOW_SECONDS(profile)Window length for the rate limit, in seconds.
API_MAX_BODY_BYTES(profile)Hard upper bound for the request body. Exceeding it returns 413 SECURITY_REQUEST_BODY_TOO_LARGE.

Where they are loaded

The configuration loader reads .env at repo root, then env/<APP_ENV>, then any path in ENV_FILE — last-write-wins, except for a small list of variables (load-test rate-limit overrides) where the parent shell wins.
