# services/portal — static documentation site

This is the **portal service** per [ADR 0028](../../docs/internal/governance/adr/0028-monorepo-with-service-boundaries.html). Its job is to serve the `services/portal/` tree as static HTML/CSS/JS behind a thin web server. The image carries no Python runtime — that is the deliberate separation from the `api` service.

## Layout (current — Phase 1 placeholder)

ADR 0028 Phase 1 ("folder split, no behavior change") says: *move `app/` to `services/api/app/`, leave `services/portal/` in place but add a placeholder `services/portal/` that points at `../../docs/`*. This directory **is** that placeholder. The portal source tree remains at repo-root `services/portal/` so the IA established by [RFC 0004](../../docs/internal/governance/rfc/0004-public-vs-internal-documentation-portal-ia.html) (Diátaxis-public + Backstage-IDP-internal) is preserved without the depth change a physical move would force on every relative link.

```
study_app/
├── services/portal/                         # source of truth — public/, internal/, uml/, assets/
│   └── ...
└── services/
    └── portal/
        ├── Dockerfile            # nginx:alpine static-asset image; copies ../../docs/
        └── README.md             # this file
```

## Build and run

The Dockerfile assumes the build context is **the repo root** so that `COPY docs /usr/share/nginx/html` can pick up the source tree.

```bash
# from repo root
docker build -f services/portal/Dockerfile -t study-app-portal .
docker run --rm -p 8080:80 study-app-portal
# open http://127.0.0.1:8080/index.html
```

A `docker compose` orchestration that brings up `api`, `portal`, and `monitoring` together is ADR 0028 Phase 3 — separate backlog item.

## What lives where

- **Source content** — `services/portal/` at repo root. Authored by hand or generated by `make docs-fix`.
- **Generated artifacts** — `services/frontend/portal/assets/search-index.json`, `services/portal/internal/catalog/api/code-reference/` (pdoc), `services/portal/internal/uml/rendered/*.svg`. All committed; `make docs-check` enforces no drift.
- **Service descriptor** — this directory. No business logic; configuration only.
- **Dockerfile** — `nginx:alpine` with security headers (`X-Frame-Options`, `X-Content-Type-Options`, `Referrer-Policy`). Listens on port `80`.

## Future migration (out of scope for the placeholder)

When `services/portal/` physically moves into `services/portal/docs/` (a follow-up to ADR 0028 Phase 1), every relative link inside the tree needs `+1` level of `../` to reach repo-root assets, and several scripts under `scripts/` need their `services/portal/` prefix updated. Cost is non-trivial; the placeholder above lets the architectural separation land first and the path migration land separately.
