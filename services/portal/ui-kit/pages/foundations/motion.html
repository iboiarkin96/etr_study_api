Foundations

Motion

Three easings, three durations, one rule: every animation answers a question. If you cannot phrase the question, the animation is decoration — and decoration is debt.

Three easings

Hover (or keyboard-focus) a track to run its curve — the dot animates with the live token, so what you see is what components ship.

--ease-out
Default. Decelerates into rest — feels natural for entering or settling elements.

--ease-in-out
Symmetric. Use for state transitions that have equal entry/exit weight.

--ease-spring
Overshoots gently. Reserved for affirmative moments — modal open, toast appear, success state.

Linear is banned for visible motion — it always looks mechanical. The only exception is opacity-only crossfades under reduced-motion.

Three durations

Durations are tokens, not magic numbers — they live in tokens/_base.css next to the easings. If you typed a raw ms value, ask which tier it is.

TokenValueUse for
--duration-fast120msHover, focus, color shifts — should feel instant.
--duration-base200msDefault for component state changes (collapse, reveal, slide, drawer, modal).
--duration-slow350msPage-level moments — hero entrance, rocket launch. Sparingly.

Anything over --duration-slow looks slow on a modern display. If a motion needs longer, it is staging two motions — split them.

Three rules

  1. Motion explains state. A toggle that flips silently is broken. The flip is the explanation.
  2. Reverse motion mirrors entry. If an element slid in at --duration-base --ease-out from the left, it slides out slightly faster with --ease-in-out to the left. Symmetry is the contract.
  3. Reduced motion always wins. Every transition and animation must have a @media (prefers-reduced-motion: reduce) fallback — usually transition: none or instant opacity-only.

Reduced motion contract

/* Default — animated. */
.docs-toast {
transform: translateY(8px);
opacity: 0;
transition: opacity var(--duration-base) var(--ease-out), transform var(--duration-base) var(--ease-spring);
}
.docs-toast[data-visible="true"] {
transform: translateY(0);
opacity: 1;
}

/* Reduced — instant, opacity only. */
@media (prefers-reduced-motion: reduce) {
.docs-toast {
transition: opacity var(--duration-fast) linear;
transform: none !important;
}
      }

JS mounts read window.matchMedia("(prefers-reduced-motion: reduce)").matches and skip rAF loops entirely — never just shorten their duration.

When not to animate

  • Decorative loops on cards or icons. The user is reading, not watching.
  • Cursor parallax. It is a 2010s effect; it ages instantly.
  • Scroll-jacking. Touch users hate it; keyboard users cannot use it.
  • Loading skeletons for <200 ms data. The flash is worse than the wait.
