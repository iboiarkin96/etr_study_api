API · v1

Sessions

Reference for the /api/v1/sessions resource — fields, endpoints, error codes.

stable v1

Resource

A Session represents a single study sitting — a contiguous period in which a user reviews one or more cards. Sessions are created when the user opens a deck and closed when they leave the app or pass an idle threshold.

Fields

FieldTypeRequiredDescription
iduuidyesUnique identifier (server-assigned).
user_iduuidyesOwner; foreign key to users.id.
deck_iduuidyesDeck under review during this session.
started_attimestampyesISO-8601, UTC. Server clock.
ended_attimestampnoNull while the session is open.
duration_msintnoDerived; only set when ended_at is.
cards_reviewedintyesCount of reviews rows attached.
tagsstring[]noFree-form labels (≤16 chars each).

Endpoints

GET /api/v1/sessions/{id}

Fetch a single session.

bash$ curl -sS https://api.study-app.example/v1/sessions/$ID \
    -H "Authorization: Bearer $TOKEN" | jq .
json{
  "id": "0e5b…",
  "user_id": "0a32…",
  "deck_id": "8f01…",
  "started_at": "2026-05-11T18:24:00Z",
  "ended_at": "2026-05-11T18:42:11Z",
  "duration_ms": 1091000,
  "cards_reviewed": 24,
  "tags": ["evening", "core-vocab"]
}

POST /api/v1/sessions

Open a new session. Server fills id, started_at, and a default cards_reviewed: 0.

bash$ curl -sS https://api.study-app.example/v1/sessions \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{"deck_id":"8f01…"}'

PATCH /api/v1/sessions/{id}

Close a session and stamp ended_at. The server computes duration_ms.

Errors

HTTPCodeMeaning
400session_already_openUser has an active session — close it first.
404session_not_foundNo row matches the supplied id (or it isn't yours).
409session_already_closedCannot patch a closed session.
422invalid_deckDeck does not exist or is archived.
429rate_limitedToo many session opens — see Retry-After header.
