Components

Ten .tma-* primitives — the entire vocabulary of the Mini App. Anchors below are stable — screen pages link straight to a component's demo.

Cell · .tma-cell

44 px min-height list row — icon slot, title/subtitle, aside slot, chevron. Grouped inside .tma-section__plate. This is the workhorse of the entire app; ~70% of surface area is cells.

Section · .tma-section

Groups a batch of cells (2–6) under an uppercase header. Optional footer below carries hint text. Sections stack with --tma-space-6 between them.

Buttons · .tma-btn

Four variants — primary, secondary, destructive, ghost. Add .tma-btn--block for full-width. Scale-0.98 on press; haptic impactOccurred('light').

MainButton — the sticky bottom-bar CTA Telegram renders. Wire it via SDK on screens with a primary outcome; hide it everywhere else. Static preview: .tma-main-button.

Badges · .tma-badge

12 px caption chips for state and count. Five tones — accent, success, warn, danger, neutral. Sit in the cell aside slot or the topbar.

The identity moment. Only one on any given screen — carries the accent gradient. Used for streak / due-count / any «one big number».

Schedule strip · .tma-schedule

14 columns, one per day. Bar height maps to review count. Today is .tma-schedule__bar--today, empty days use --empty. Labels sit above the bar via .tma-schedule__label.

Empty state · .tma-empty

Centered glyph + title + one short line of body. Ships on every list — «Nothing due today» beats a blank screen.

Bottom-sheet · .tma-sheet

Modal for short-lived writes — add-error, snooze-picker. Grip on top, title, one field, one primary action. Never nest a sheet inside a sheet.

Topbar · .tma-topbar

Optional inside the Mini App — Telegram's own back button often covers this. Use only when a screen has a right-side action.
