Components · v3

The .tma-* primitive vocabulary of the Mini App, one card per class. Every preview here is a live Storybook render — the same CSS the app bundles, rendered by the real React component. This page can never drift from production: if a primitive isn't shipped in services/telegram/src/styles/primitives/, it isn't on this page.

Live · in production

Twelve primitives compose the shipped app. Each card carries a Storybook iframe of the primitive's clearest consumer, the CSS path, and a link to the machine-checkable spec under Reference.

Orb · .tma-orb

live

The D1 breathing hero — the one living element that recurs across the app. Warm on an active streak, sage when rested, ember pulse on milestone. Same primitive on Today, Profile, and the day-zero hero on Onboarding.

Session ring · .tma-orb-ring

live

In Focus the orb becomes the progress instrument: a thin conic arc wraps it and advances one increment per graded card (--ring-progress: 0..1, @property-registered so the sweep animates). Ember by default; data-state="rested" flips it to sage on completion.

Buttons · .tma-btn

live

Flat and solid — no gradients, no glows. State speaks through fill tone; tinted variants derive from tone ink at 12 %. Primary, ghost, outline, tinted; --block makes any variant full-width.

Focus card · .tma-focus

live

The SRS review flow container — full-screen prompt, tap-to-reveal, Again/Hard/Good/Easy thumb-strip. Grade keys are flat tints of each grade's tone ink (--tma-tone-danger/-warn/-accent/-success) — no borders, no shadows; state is tint depth + snap-scale.

Command palette · .tma-cmdk

live

Cmd+K spotlight overlay — portal-mounted modal with client-side substring filter over the cached title index. Full WAI-ARIA modal contract (focus save + restore, aria-activedescendant, live-region announcements). Keyboard: ↑/↓, Ctrl-J/N/P, Enter, Esc.

Bottom-sheet · .tma-sheet

live

Modal composer for short-lived writes — scrim + slide-up panel + grip + auto-focused field + primary action. Capture-phase Escape containment shared by MissSheet and NudgeSheet so the underlying screen's Esc binding can never fire while the sheet is open.

History

  • 2026-07-21 @ivan v3 · trimmed to production-only vocabulary. Removed 16 spec sections whose primitives never shipped (.tma-badge / .tma-display / .tma-hero-card / .tma-ring / .tma-ticker / .tma-toast / .tma-skel / .tma-search / .tma-segmented / .tma-tag / .tma-ach / .tma-timeline / .tma-empty / .tma-topbar / .tma-schedule + data-ambience) — they had zero usages under services/telegram/src. Hand-rolled mocks for .tma-cmdk and .tma-sheet replaced with live Storybook iframes (SearchOverlay and MissSheet). Cheatsheet + TOC trimmed to the 12 shipped classes. Every card now carries either a Storybook iframe or a spec link (per the unified visual system rule: kit = «why», reference = «how», Storybook = «show»).
  • 2026-07-14 @ivan Ember v3 hard-canon pass — demos + prose synced to the flat kit (retained for archival context; the demo blocks themselves were removed in the 2026-07-21 trim).
  • 2026-07-14 @ivan Flat redesign: buttons went solid (no gradients/glows; primary = tg-theme-button-color), added outline + tinted variants; grade keys and badges/cell-icons now derive fills from tone ink at 12% alpha (theme-proof). New screen primitives promoted from mockups: tma-screen-head, tma-peek, tma-miss-log/-row, tma-plate-row. Signature layer: session ring (tma-orb-ring, orb as progress instrument), CSS-only live ticker (tma-ticker--live), grade-exit choreography (data-exit), and the editorial Done moment (tma-done). Some of these were never adopted in production and were dropped from this page in the 2026-07-21 trim; the CSS still lives in tma-kit.css as showcase infrastructure.
