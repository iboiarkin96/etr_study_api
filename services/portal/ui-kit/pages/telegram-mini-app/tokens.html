Tokens · v3 · Ember

Two-tier system — primitives (raw values) → semantic (role-based aliases). Components consume semantic tokens only. Palette is dark-first, warmed: signature is ember (coral/terracotta), cool secondary is sage reserved for done / rested states. Values in assets_v2/ui-kit/components/tma-kit.css.

Shipped 2026-07-14 · Iris (cold purple) retired to a legacy alias — new components must read --tma-ember-*, never --tma-iris-*.

Source of truth

live

services/telegram/src/styles/tokens.css is canonical. The UI Kit's tma-kit.css starts with @import of that file, so every swatch on this page resolves through the exact --tma-* values production uses. Kit-only additions (iris legacy alias, grain URL, accent-text for portal chrome outside .tma-scope, extra motion timings --tma-dur-lazy/--tma-ease-linear/--tma-ease-decel, the display type tier --tma-fs-display*/--tma-fs-hero, elevation edges --tma-elev-0/--tma-elev-4, --tma-sp-16, and semantic extras like --tma-surface-input, --tma-text-link, --tma-action-secondary/-destructive, --tma-*-soft) live only in tma-kit.css. Any redeclaration of a palette token that already lives in src is drift — make docs-check fails on it (see docs-token-check gate).

Architecture · two tiers

Tier 1 · Primitives

Raw values, no meaning. Change only when the source palette changes.

--tma-neutral-100 · --tma-ember-500 · --tma-sage-500 · --tma-success-500 · --tma-dur-med · --tma-rad-3 · --tma-fs-body · --tma-fs-display · --tma-sp-4

Tier 2 · Semantic

Role-based aliases. This is what components read. Renaming a role stays a one-line change in the alias.

--tma-surface-plate · --tma-text-primary · --tma-text-accent · --tma-text-onaccent · --tma-tone-accent … --tma-tone-cool · --tma-action-primary · --tma-action-primary-hover · --tma-border-regular

A component that reads a primitive directly is a bug — either the semantic layer is missing an alias, or the primitive is being coupled to a role. Add the alias.

Colors

Flat-register canon — controls are flat/solid; tints are the tone ink at 12% via color-mix. The orb is the only gradient material in the whole app — every other surface is flat. No text-shadow / glow outside the orb.

Signature · Ember (warm primary)

The single most emotionally-loaded token in the system. Warm accent is the signal that this is a personal-growth tool, not a work tool. Never mix ember + iris in the same view — iris is a legacy alias only. For accent text on showcase chrome outside .tma-scope, read the :root token --tma-accent-text (ember-700 light / ember-400 dark).

Text

AA-audited pairs. --tma-text-tertiary is neutral-600 dark / #7a6d64 light — neutral-500 sat at ~3:1 on the dark canvas, below AA for the 11–13 px meta text this token carries. --tma-text-onaccent is ember-950: white on ember-500 is ~2.8:1 (AA fail on the most-tapped surface); the warm near-black ink reads ≥6:1.

Tone inks · theme-proof foregrounds

The only foreground tokens tinted components read — cell icons, badges, tinted buttons, grade keys, timeline dots, ring strokes, digest icon, achievements. Dark theme carries the pastel -500 (deep -ink values are near-invisible on dark surfaces); light theme flips every tone to its -ink. Tint recipe: color-mix(in oklab, var(--tma-tone-*) 12%, transparent).

Type · 8 body steps + display tier

Number-heavy tokens use tabular-nums automatically (streak, dates, durations). The display tier is reserved for numbers that carry emotion — never body prose.

Display · numbers only

Streak digit, ring centre, milestone counter, focus progress. Applied via the .tma-display utility (weight 800, tabular-nums, tight -0.035em tracking). Adds .tma-display--ember — solid accent colour (flat register: no gradient text, no glow; the digit carries emotion through scale and hue alone).

Motion · duration + easing

Never hard-code milliseconds inside a component. prefers-reduced-motion collapses to instant automatically. Personality (single ship: Expressive — 240 ms · ease-spring · 1–3% overshoot) is specified once on motion.html.

Density

Production ships a single density — data-density="regular" on every screen's .tma-scope root. Cells hold a 44 px min-height (iOS a11y floor) with 12 px padding. A compact variant was drafted in v2 but never applied; removed from src to keep the canon single.

Font stack

No shipped webfont — the system stack is routed through --tma-font-sans and --tma-font-mono. Inter is listed first as a soft opt-in (Chrome / Chromium builds ship it via the Google Fonts CSS the portal loads); Apple platforms fall through to -apple-system / SF Pro. Numbers use tabular-nums so digits stop jumping on animated tickers, streaks, and dates.

--tma-font-sans: 'Inter', -apple-system, 'SF Pro Text', 'Segoe UI', Roboto, sans-serif;
--tma-font-mono: 'JetBrains Mono', 'SF Mono', ui-monospace, monospace;
