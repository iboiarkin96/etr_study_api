Screen DNA · heat-map look-ahead

Amie · signature
  • Heat-map is the breathing element. D1 hero here is the 90-day ember gradient — a dense block reads «I was on fire», a sparse block reads «getting back on track». Summary strip up top answers «right now»; heat-map answers «who am I».
  • Documented DNA deviation · zero primary CTA. D2 «one obvious action» is dropped — Schedule is a status surface, users route out via chrome (back button), not a hero button. Contained: no other screen does this.
  • Reuses Today's primitives. SummaryStrip + HeatmapCalendar are the same React components Today renders — same tokens, same layout, same behaviour. Only new-code is the wrapper (route + header + empty-state card).
  • No client aggregation. Every number comes straight from GET /schedule/{summary,history} per the epic's «no client aggregation» rule. The client is a pure display layer.
  • Assemble choreography. Summary strip is the hero (fade + soft scale, in place), heat-map is slot 1 (fly-up 32 px). Same one-second rhythm as Today's cold-open.

03 · Schedule · Philosophy

Why Schedule matters

Schedule is the antidote to «open the app · surprise: 15 due today». Show the shape of the next weeks so the user pre-empts spikes. Snoozing lives on Today / Conspectus, not here — this screen is a look-ahead, not a control panel.

You can't cram a spike after it hits. The schedule is the pre-emptive lever.
