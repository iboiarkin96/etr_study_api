Anatomy of the identity mirror

Two jobs on this screen: show consistency paying off, and let the user flip the daily reminder. Zero account UX — no logout, no theme picker, no switching.

Purpose

The screen exists so the user can see, in one glance, that the daily habit is paying off — streak, month reviews, retention chart. Second job: toggle the daily bot reminder. That's it. Everything else is deliberately absent.

The identity screen for a maintainer-only app should not look like a settings screen. It's a mirror.
