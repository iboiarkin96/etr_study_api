Screen DNA · miss shape

Amie · signature
  • The shape, then the action. MissOrb answers «three this week» before the user scrolls — D1 breathing element. Rows below name the pattern; one warm CTA turns «I noticed» into «I logged one».
  • Warmer as it ages. Tone dot goes ember → warn → sage as an entry gets older. A wall of ember reads «this week was hard»; a wall of sage reads «pattern's cooling». Three tones, one accent budget (D4).
  • One warm CTA (D2). «Log a miss» ships as one ember button; the header ＋ duplicates it as an affordance, not a second action. Same «one obvious primary» rule as Today.
  • Editorial peek, not analytics chrome (D3). Timeline rows — signal dot · one-line message · quiet mono time. No filter chips, no tag facets, no delete. Append-only per methodology; hiding the shape behind a facet is exactly what the ETR frame avoids.
  • Idempotency-safe writes. POST is auto-injected with a fresh Idempotency-Key; retries on flaky 4G don't double a miss (ADR 0006). Mutation is optimistic — temp row prepended immediately, replaced with server row on 2xx or rolled back on error.
  • Blocks assemble, they don't paint. MissOrb hero, list header slot 1, MissLog slot 2, CTA slot 3 — same one-second choreography as Today / Schedule.

05 · Errors · Philosophy

Why Errors matters

Reviews tell the app what the learner knows; misses tell the learner what they don't. Three misses in a week clustered around one topic means «reset the schedule on that cluster» — the shape is the signal. Deletion is deliberately absent: the log is append-only. The point is not to feel bad about the miss — it's to notice the shape of the misses.
