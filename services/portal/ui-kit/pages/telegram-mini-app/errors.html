Purpose

Errors are the ETR methodology's memory of where I fell down. Rendering them here (grouped by week, most-recent-first) makes the pattern legible — three misses in one week around distributed systems means «reset the schedule on that cluster». Deletion is deliberately absent: the log is append-only.

Data · API endpoints

List
  • GET /api/v1/errors

Cursor-paginated · returns { id, conspectus_id?, description, created_at }. Grouped client-side by ISO week.

Add
  • POST /api/v1/errors

Body: { description, conspectus_id? }. Idempotency-Key header sent from the sheet to survive retry.

Components used

  • .tma-topbar
  • .tma-section
  • .tma-cell (danger icon)
  • .tma-sheet
  • .tma-btn
  • .tma-empty

States

Loaded · has entries

List grouped by week · newest first. Each cell is a summary of the miss.

Empty

Glyph 🎯 · «No misses yet» · body «Log the first one when a recall breaks».

Sheet open

Backdrop dims 40 % · sheet slides in 240 ms · textarea auto-focus.

Sheet · saving

MainButton label swaps to spinner · sheet stays open until 2xx.

Save error

Toast «Couldn't save» · retry inline · sheet stays open with content preserved.

Save success

Sheet slides down · list optimistically prepends the new cell · haptic notificationOccurred('success').

Interactions

  • Tap topbar ＋ → open sheet · auto-focus textarea.
  • Tap Save in the sheet → POST · optimistic list insert · haptic on success.
  • Drag sheet grip down → dismiss without saving (content lost — matches Telegram convention).
  • Tap a cell → if conspectus_id present, navigate to that Conspectus (v2).
