The one screen where a raw thought becomes a stored conspectus. Not a form — a shaped move: Encode → Trigger → Recall. Screen DNA · three-move ribbon

Amie · signature
  • Three phases, one gesture. Encode · Trigger · Recall aren't sections of a form — they're the three roles ETR needs a note to fill. Rendering them as one vertical stack lets the user feel the shape of the note before it exists.
  • Ribbon is the hero. No orb, no digit — three dots that warm as each phase is filled. When the third dot warms, the ribbon breathes once (single-pulse spring) and the Telegram MainButton unlocks. The «done» moment lives in the ribbon, not in a modal.
  • Contract-first fields. Every visible input maps 1:1 to a POST /api/v1/conspectuses field. There is no «invented» field the server would silently drop, and there is no server-required field the UI hides. The UI is a materialised contract.
  • MainButton owns Save. The Telegram-native bottom button is sticky and equidistant from both thumbs — the only place a Save CTA should live on mobile. We don't paint a competing in-page primary. When the draft isn't submittable, the button hides (not disables) so the visual state is unambiguous.
  • Right-edge actions. «+» to add, «×» to remove — all right-aligned within each row. ~⅔ of one-handed mobile sessions use the right thumb (Hoober / UXmatters); the natural reach zone is the bottom-right quadrant. Left-hand mirroring lands as a Profile toggle later.
  • Assemble on entry. Ribbon fades in first (hero), phase cards stagger up beneath (D2 → D3). Same one-second choreography as Today / Errors — the assemble grammar the app has already taught the user.

Full backend contract: Conspectus API · reference. The rejected «single big textarea + `/save` parser» alternative is captured in ADR 0038 · per-screen table (draft entry, links back here).

Contract, mapped

The whole point of the three-phase composer is that every visible input is a backend field. This is the mapping.

Phase UI element Backend field Validation
01 · Encode Inline title chip (top-right of phase card) title optional · ≤ 256 chars · client trims empty → null
Dense paragraph textarea + counter dense_paragraph required · 1..4000 chars · counter goes warn at 90% / danger over cap
02 · Trigger Term chip input (Enter / comma to commit) cue_sheet.terms[] optional · ≤ 20 items · ≤ 64 chars each · deduped client-side
Question list rows (＋ Add / × Remove) cue_sheet.questions[] optional · ≤ 20 items · empty rows dropped on submit
03 · Recall Bullet list rows (auto-grow, minimum one) bullets[] required · 1..20 items · 1..500 chars each

Idempotency-Key is auto-injected by the client middleware (ADR 0006) — a lost reply on retry never doubles a conspectus. On 2xx the mutation invalidates both the Today due-list and the Search index so the new row appears without a manual refresh.

Thumb-zone rationale

Why every «+», «×», and Save action sits on the right — and why we still ship Left-hand mode later.

Steven Hoober's oft-cited UXmatters study (2013, reconfirmed 2021) puts roughly half of mobile sessions in one-handed use, and about ⅔ of one-handed sessions are with the right hand — so ~⅓ of every session is a right thumb reaching for controls. The bottom-right quadrant is the only zone that reads as natural; the top-left is where the thumb hurts.

Three consequences for Encode:

  • Save lives in the Telegram MainButton. The SDK button is sticky-bottom and centred, equally reachable by either thumb — the one place on a Telegram Mini App a primary CTA should live. No competing in-page Save.
  • «＋» to add, «×» to remove — always right-aligned. The wide text field is the passive target; the action affordance goes to the thumb-natural edge so accidental swipes don't fire it (WCAG 2.5.3 target sizing).
  • Back is the Telegram-native BackButton. Top-left in the chrome — one of the few things a user learns to reach for regardless of the pain-zone rule, because it's system-wide.
  • Keyboard cap. iOS / Android soft keyboards eat ~40–50% of viewport when open. Every phase card lays out so the active field is above the keyboard fold — the ribbon at the top stays a visual anchor as the user scrolls between phases.

Known limit — left-hand mirroring: the same one-hand statistic gives us ~⅓ of one-handed users on the left thumb, plus the fully-symmetric «cradle» mode. A Profile toggle that mirrors the right-edge action cluster to the left is scoped for a later slice — captured in the roadmap, not yet a screen deviation.

10 · Encode · Philosophy

Why Encode matters

Every other screen in the app is a consumption surface — Today displays, Focus drills, Schedule projects, Errors journals. Encode is the only place where the user authors. A cheap open-textarea composer would ship a database of unstructured text; a wizard would ship a bureaucracy. The three-phase ribbon is the shape of the ETR method — it teaches the method by making the user perform it, once per note, without ever showing them the schema.

A cheap composer ships a database. A three-move ribbon ships a habit.
