Alternative personalities · reference only

These stay documented so the studio decision to ship Expressive is legible and reversible. Both variants keep the orb — only the pace of everything else changes.

Subtle · Vercel-tier restraint

Fast (120 ms), out-easing everywhere. Nothing overshoots. Every motion is functional — it exists to confirm a state change, not to entertain. Costs the least battery; feels the most clinical.

hover · 120 ms out
★ Expressive · shipped

Med (240 ms) with spring easing on every interactive. Slight overshoot on lift. Orb breathes / warms / pulses. Toasts spring in. Feels alive without being cartoonish. This is what production ships.

hover · 400 ms spring
Playful · Duolingo-tier delight

Lazy (600 ms), Lottie celebrations on every session-end, mascot bounces, opt-in sound. Every success becomes a mini-party. Highest retention on casual audiences; too much for serious learners.

pulse · 1.4 s infinite

Implementation stack

  • CSS transitions for hover / focus / tap states — cheapest, GPU-accelerated. Everything the orb does is CSS-only (~2 KB, no runtime cost).
  • Framer Motion for gesture-driven (drag, swipe, dismiss) and shared-element transitions between Today and Focus.
  • View Transitions API for route changes on TanStack Router. 90% browser support in 2026 — polyfill fallback via cross-fade.
  • Lottie only for milestone confetti (streak 7 / 30 / 100 / 365). Cached on disk; ~15 KB gz each. The orb's celebrate state runs alongside.
  • Auto Animate for list add/remove — zero-config sensible defaults.

Rules

  1. Never hard-code milliseconds. Always route through a duration token.
  2. Motion on outcomes, not opens. Opening a screen doesn't celebrate; completing a review does.
  3. One personality per app. Ship Expressive. Subtle / Playful exist only in this reference document.
  4. The orb is the only signature moment. Don't add a second one. Every other motion is functional feedback.
  5. Respect prefers-reduced-motion. Collapse everything — including the orb's breathe — to instant. Never bypass this for a «cool» animation.
  6. Motion is a fallback for feedback, not a replacement. Every animation should have a static equivalent (text, colour, badge) that conveys the same info.
