Motion · v3

The motion system of the Mini App — what breathes, what pulses, what stays still, and why. Every duration and easing on this page maps to a token in src/styles/tokens.css; every rendered example is a live primitive from src/styles/primitives/*.css (imports the same tma-kit.css bundle production uses). All motion respects prefers-reduced-motion — enable it in system settings to see everything collapse to instant.

Personality · single ship: Expressive.
240 ms · ease-spring · 1–3% overshoot. The orb is the motion carrier: it breathes when warm, pulses on milestone, cools to sage when the queue is done. Everything else is functional feedback — nothing else «performs».

Signature motion · the orb

The orb is the one signature moment. Three shipped states — warm (active streak — gentle breathe), rested (queue done — sage variant), celebrate (7 / 30 / 100 / 365-day milestone — pulse + expanding halo). Live-tuned in src/styles/primitives/orb.css; the React consumer is StreakOrb (Today, Profile) and OnboardingHero (day-zero variant).

Entrance choreography · Assemble

Every screen wires its hero + slotted blocks through the Assemble wrapper — the orb (hero) fades into place first, then each below-orb block flies up 32 px with a 70 ms stagger. Transform-only — a block's layout space exists from the moment it mounts, so nothing below jumps. CSS-driven (deliberately chosen over Framer to avoid a WAAPI opacity blink at animation completion); fill-mode: both holds cleanly.

Live-preview the full arrangement on screens/today · default ↗. Contract: Assemble spec.

Swipe-to-review · Framer Motion

The one place the app ships a JS-driven gesture. DueCardsList uses Framer's useMotionValue + useReducedMotion: swipe right past +100 px commits easy, swipe left past −100 px commits hard, past −200 px commits forgot. On commit the row transforms off-screen with a spring exit; on rollback it snaps back via the same spring. Reduced-motion collapses to an instant tag application.

Threshold constants live in src/screens/Today/components/swipe-thresholds.ts — the numeric contract is decoupled from Framer so tests can pin it without pulling the motion runtime. Spec: DueCardsList.

Implementation stack

  • CSS transitions + keyframes for everything a token can describe — hover, press, sheet slide, orb breathe, ring sweep, Assemble entrance. Cheapest, GPU-accelerated, respects prefers-reduced-motion for free.
  • Framer Motion for the two things CSS can't do cleanly — Today's swipe-to-review (motion values + gesture thresholds) and its due-list add/remove (AnimatePresence). Every hook site imports useReducedMotion and defaults motion off when the OS says so.

Deliberately absent: no Lottie, no Auto Animate, no View Transitions API. The two runtimes above cover every shipped motion; a third would be a review-time flag.

Rules

  1. Never hard-code milliseconds. Always route through a duration token in tokens.css.
  2. Motion on outcomes, not opens. Opening a screen doesn't celebrate; completing a review does.
  3. One personality per app. Ship Expressive.
  4. The orb is the only signature moment. Don't add a second one. Every other motion is functional feedback.
  5. Respect prefers-reduced-motion. Collapse everything — including the orb's breathe — to instant. Never bypass this for a «cool» animation.
  6. Motion is a fallback for feedback, not a replacement. Every animation should have a static equivalent (text, colour, badge) that conveys the same info.
