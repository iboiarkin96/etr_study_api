Conspectus

The outcome screen. The user opens a note to review it — everything on this screen exists to make «mark reviewed» a one-tap operation. The prose is the content; the MainButton is the exit.

SOLID · single responsibility principle

Last reviewed 4 days ago · streak 3

A module should have exactly one reason to change. «Reason to change» is a stakeholder / actor, not a technical layer.

Signal that SRP is violated: two unrelated PRs keep touching the same file. Split by actor, not by feature.

Source: Clean Architecture · Ch. 7

Purpose

One conspectus, rendered for reading. Not for editing — creation and edits live in the portal (see the epic). The success moment on this screen is the fade-out on «mark reviewed»: the streak ticks, the row disappears from Today, and the user is one review closer to done.

Data · API endpoints

Load
  • GET /api/v1/conspectuses/{id}
Mark reviewed
  • POST /api/v1/conspectuses/{id}/actions/review

Body: { outcome: "recalled" | "missed" | "snoozed", snooze_days?: number }. Response includes updated next_due_at.

Log an error inline
  • POST /api/v1/errors

Body: { conspectus_id, description }. Opens the same sheet as Errors, with conspectus_id prefilled.

Components used

  • .tma-topbar
  • .tma-badge
  • .tma-btn

States

Loading

Title + paragraph skeletons. MainButton disabled with a spinner label.

Loaded · due

Full note visible · MainButton «Mark reviewed» primary.

Loaded · not due

Same layout · MainButton hidden · a footer strip says «Next review in 3 days».

Review saved

MainButton flashes success tint 200 ms · row collapses · route pops back to Today.

Save error

Toast «Couldn't save review» · MainButton re-enabled · retry inline.

Not found · 404

Empty state «Note not found» · secondary «Back to Today».

Interactions

  • Tap MainButton → POST review · haptic notificationOccurred('success') · pop back on 2xx.
  • Long-press MainButton → context sheet with snooze options + «Log an error».
  • Tap topbar ··· → sheet with meta actions.
  • SDK BackButton → pop to previous screen (Today or Schedule).
