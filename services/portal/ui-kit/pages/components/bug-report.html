Bug report

Single-purpose component that lets readers open a prefilled GitHub issue for the page they are reading. Ported from the legacy .docs-report-bug-fab + .docs-feedback-modal pair, but reframed as a topbar action sitting next to the theme toggle instead of a fixed-position FAB anchored to the viewport corner.

  • Trigger — 36 × 36 icon button auto-injected into every .topbar__actions before the theme toggle. Hover / focus wiggles the bug's antennae.
  • Modal — single shared instance. Feedback type select, free-text area (10+ chars), submit opens GitHub in a new tab with the page URL, type, and description already filled in.
  • Clipboard — the description is copied to the clipboard as a backup before the new tab opens, so a navigation glitch never loses what the reader typed.

Live button

The button in the topbar is the live component. The demos below are wired to the same modal — click any trigger. Two modes are shown: the auto-injected icon button (left), identical to the one in the topbar, and an opt-in inline text trigger (right) you can drop inside prose.

Tip: the bug icon in the topbar also has a tooltip — hover or Tab-focus it to see the new tooltip component in action.

Hover or focus the icon button to see the antennae snap outwards; otherwise they idle-wiggle on their own (disabled under prefers-reduced-motion).

Generated markup

The runtime injects the button into every .topbar__actions automatically — you don't need to write the markup. If you want a second trigger somewhere (e.g. an inline link in long docs), drop any element with data-component="bug-report" or data-bug-report-open; the runtime upgrades it on mount.

<!-- Auto-injected into the topbar -->
<button class="docs-bug-report-btn" type="button"
        aria-label="Report a bug on this page" title="Report a bug">
  <svg class="docs-bug-report-btn__icon" …>…</svg>
</button>

<!-- Opt-in inline trigger -->
<button data-component="bug-report" type="button">Report an issue with this page</button>

Modal contents

  • Feedback type — select with six categories (incorrect content, missing explanation, broken link, visual/layout bug, accessibility, other). Routes triage faster on GitHub.
  • Description — required textarea, minimum 10 characters. Renders under the What should be improved section of the issue body.
  • Submit — primary accent-coloured button. Builds the GitHub URL via ?template=…&labels=…&title=…&body=… and opens it in a new tab with noopener,noreferrer.

A11y

  • Button uses aria-label="Report a bug on this page"; SVG is aria-hidden.
  • Modal is a role="dialog" aria-modal="true" region labelled by the heading.
  • Esc closes the modal; backdrop click also closes.
  • Focus moves to the textarea on open and returns to the trigger on close.
  • Submission status uses aria-live="polite" so screen readers hear the feedback.
  • Antennae animation is disabled under prefers-reduced-motion.

Tokens used

  • --accent, --card, --line, --line-soft
  • --text, --text-strong, --muted
  • --radius-sm, --radius-lg
  • --shadow-lg, --ease-out, --ease-spring
  • --fs-200, --fs-300, --fs-500, --fw-semibold

Configuration

The GitHub repository, issue template, and labels live as constants at the top of bug-report.js — change them in one place to retarget every portal.

const REPO     = "iboiarkin96/study_bot";
const TEMPLATE = "docs_feedback.md";
const LABELS   = ["docs-feedback"];
