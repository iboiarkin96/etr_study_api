Scene background

Page-level ambient backdrop. Two stacked fixed layers attached to <body>: a slowly drifting radial mesh (portal accent) and a soft grid lattice masked to a centre-top vignette. The mesh inherits the portal accent via --grad-mesh-1/2/3 — internal renders sky-blue, public renders violet, both adapt to data-theme.

This page is running scene-bg — look behind every section.

Pair with aurora-rail at the foot of the page for the full atmosphere stack: scene-bg sets the room, aurora-rail sets the horizon.

Same three radial lobes + grid lattice as the shipped component, but with opacity cranked ~3× so the bloom is unmistakable. This is not the production look — it's here to make the structure obvious. The production values (0.14–0.18 alpha) are shown in Production intensity below and in Markup.

Companion · aurora-rail

Aurora rail is the matching pre-footer atmosphere zone — a CSS-only port of the legacy body::before fog + body::after dot-wave system. Auto-mounts on every page above .site-footer; honours html[data-aurora="off"] and prefers-reduced-motion. No JSON to wire — just import the entry bundle.

  • scene-bg — full-page ambient, fixed to viewport, drifts gently.
  • aurora-rail — local to the foot of the page, blooms with scroll progress via --docs-footer-scroll-strength.

Markup

Scene-bg is a body-level decoration — there is no element to drop in. Add the class to <body>:

<body class="docs-shell has-scene-bg">
  <!-- page content -->
</body>

For the soft variant, also add docs-scene-bg--soft (CSS hides ::after, leaving the mesh only):

<body class="docs-shell has-scene-bg docs-scene-bg--soft">…</body>

The two layers come from body.has-scene-bg::before (mesh, z-index: -2) and body.has-scene-bg::after (grid, z-index: -1). Page content sits at the default stacking context above them.

States

Decorative — no interactive states. The mesh drifts 28s ease-in-out alternate; the drift halts under prefers-reduced-motion: reduce. The grid layer is static.

A11y

  • Pseudo-elements are inert to AT — nothing to label.
  • Keep z-index: -1 / -2 intact; never let the grid land above text.
  • Ensure foreground text retains ≥ 4.5:1 contrast over the brightest mesh point (the bottom-centre bloom in --grad-mesh-3).
  • Drift is slow (28s, <20% travel) and respects reduced-motion. Don't shorten the duration without re-checking AAA motion thresholds.

Tokens used

  • --grad-mesh-1, --grad-mesh-2, --grad-mesh-3 — portal-keyed in _portal-internal.css (blue) and _portal-public.css (violet).
  • --line — grid stroke (mixed 60% with transparent).
  • --bg — fallback paint underneath the layers.
