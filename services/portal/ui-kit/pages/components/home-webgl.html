Home WebGL

Soft sky-blue flowfield background for the landing hero. WebGL2, single full-screen quad, simplex-noise fbm with curl-style domain warp. Cursor + scroll-driven; intensity fades in so the canvas glides in instead of popping. Lazy-loaded only on ≥1025 px viewports — mobile/tablet keep the CSS gradient fallback.

Live exemplar

  • ui-kit/pages/templates/landing-portal.html — the only consumer. The canvas mounts on .lp-hero__canvas inside the hero.

Usage

The module is imported lazily so non-hero pages don't ship the shader. The opt-out happens via viewport gate and dynamic import — if either fails, the CSS gradient underneath stays visible.

<canvas class="lp-hero__canvas" aria-hidden="true"></canvas>

<script type="module">
  if (window.innerWidth >= 1025) {
    import("./home-webgl-v2.js")
      .then(({ initHomeWebGL }) => {
        const canvas = document.querySelector(".lp-hero__canvas");
        initHomeWebGL(canvas);
      })
      .catch(() => { /* CSS gradient fallback */ });
  }
      </script>

API

  • initHomeWebGL(canvas) — mounts a WebGL2 context on the canvas, sizes it to its CSS box, starts the animation loop, and returns a cancel() function.
  • cancel() — stops the RAF loop, releases GL resources, removes listeners. Call before navigating away if the canvas might be re-mounted.
  • If WebGL2 is unavailable, initHomeWebGL exits silently — the CSS gradient already paints the same colour palette underneath.

What you see

  • Calm ribbons — fbm + curl warp; not chaotic, not perfectly smooth. Reads "alive" without competing with the H1.
  • Cursor halo — a barely-there glow under the pointer. The field acknowledges movement but never chases.
  • Scroll settle — light rotation + zoom keyed to u_scroll (0…1 across hero scroll-out); the field settles as the user reads down.
  • Intensity fade-in — u_intensity ramps from 0 to 1 over the first ~600 ms. No pop.
  • Vignette mask — corners and the H1 zone stay quiet. Premultiplied alpha caps painted opacity at ~55 % anywhere.

Performance

  • Lazy import — the module (~9 kB gzipped) only loads when the viewport gate passes.
  • Single quad — one full-screen triangle pair, all work in the fragment shader.
  • RAF-driven — no setInterval; ties to display refresh.
  • Pauses off-screen — the controller stops requesting frames once the hero scrolls past the viewport.
  • DPR cap — pixel ratio capped to keep the fragment count predictable on retina laptops.

Accessibility & fallback

  • Canvas is decorative. aria-hidden="true"; nothing under the H1 is encoded in the field.
  • Mobile / tablet — no WebGL load; the CSS gradient is the entire background.
  • WebGL unavailable — the dynamic-import .catch() branch is intentional; we silently fall back to the gradient.
  • Reduced motion — the cursor halo and scroll settle still respond to input, but the autonomous time animation slows so users with prefers-reduced-motion: reduce see a near-static canvas.

Why v2

v1 (assets/home-webgl.js) had a tighter fbm but felt jittery near the cursor and pinned ~60 fps even when the hero was off-screen. v2 is a port that keeps the v1 look (so the brand feels continuous) but adds: domain warp, scroll-driven settle, intensity fade-in, off-screen pause, DPR cap, and the premultiplied-alpha vignette mask. The class hook stayed .lp-hero__canvas so the landing template didn't need to change.
