Theme toggle

Tri-state switch (light · dark · system) persisted to localStorage["docs-theme-preference"] and bootstrapped via an inline script in <head> to avoid a flash of incorrect theme.

Example

Click to cycle through light → dark → system.

Variants

Markup

<!-- Single-button cycle variant -->
<button class="docs-theme-toggle" type="button"
        aria-label="Toggle theme"
        aria-pressed="false">◐</button>

<!-- Required: no-flash bootstrap in <head> -->
<script>
(function(){
  try {
    var v = localStorage.getItem("docs-theme-preference");
    if (v === "dark" || v === "light") {
      document.documentElement.setAttribute("data-theme", v);
    }
  } catch (e) {}
})();
</script>

States

  • Light — icon ☀, aria-pressed="true", data-theme="light".
  • Dark — icon ☾, aria-pressed="true", data-theme="dark".
  • System — icon ◐, follows prefers-color-scheme.

A11y

  • Each state must have a descriptive aria-label.
  • Use aria-pressed for the current state when rendered as a toggle group.
  • Announce theme changes via aria-live="polite" region if the change is significant.
  • Persist preference across pages; never override an explicit user choice with OS-level changes.

Tokens used

  • --bg-elev, --line, --text
  • --radius-md, --space-2, --space-3
