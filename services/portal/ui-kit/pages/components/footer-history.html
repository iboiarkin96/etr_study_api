Footer history

Change-log footer rendered at the bottom of every doc page. Three columns per row: date, author, note.

Example

History

  • 2026-05-11 @ivan Split internal layout CSS; added analysis & maturity sections.
  • 2026-05-08 @ivan ADR 0030 published — portal shell token contract.
  • 2026-05-06 @ivan Unified prose column width across portals at ≥1025 px.

Variants

Single row

History

  • 2026-05-12@ivanInitial publish.

With linked commits

History

  • 2026-05-10@ivanb5a9d71 Split internal layout CSS.

Markup

<footer class="docs-history" aria-label="Page history">
  <h2 class="docs-history__title">History</h2>
  <ul class="docs-history__list">
    <li class="docs-history__row">
      <span class="docs-history__date">2026-05-11</span>
      <span class="docs-history__author">@ivan</span>
      <span class="docs-history__note">Initial publish.</span>
    </li>
  </ul>
</footer>

States

Footer history is static — no interactive states.

A11y

  • Use a real <footer> element with aria-label="Page history".
  • Title is an <h2> so it appears in the page outline.
  • Dates should be ISO-8601 (YYYY-MM-DD); wrap in <time datetime="…"> if you need machine-readable.

Tokens used

  • --muted (date), --muted-soft (author)
  • --line-soft (row separator)
  • --fs-200, --fs-300
  • --space-3
