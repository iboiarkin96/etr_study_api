Sidebar

Left-rail navigation for desktop (≥1025 px). Single component, data-driven for both portals via nav-tree-<portal>.json.

Example · expanded

Variants

Collapsed group (caret right)

With nested children (2 levels)

Markup

<aside class="docs-sidebar" aria-label="Section navigation">
  <nav class="docs-sidebar__nav">
    <div class="docs-sidebar__group">
      <button class="docs-sidebar__caret" type="button" aria-expanded="true">
        <span class="docs-sidebar__group-title">Governance</span>
        <span aria-hidden="true">▾</span>
      </button>
      <a class="docs-sidebar__item docs-sidebar__item--active" href="/internal/governance/backlog/">Backlog</a>
      <a class="docs-sidebar__item" href="/internal/governance/decisions/">Decisions</a>
    </div>
  </nav>
</aside>

States

  • Item · default — --text on --bg-elev.
  • Item · hover — background --bg-mute, accent left bar (2 px).
  • Item · active — modifier .docs-sidebar__item--active, accent background tint and accent left bar (3 px).
  • Group · collapsed — caret ▸, aria-expanded="false"; children hidden with display:none.
  • Group · expanded — caret ▾, aria-expanded="true".

A11y

  • Use <aside aria-label="…"> as the wrapper so screen readers announce a complementary landmark.
  • Inner <nav> exposes the list of links.
  • Collapse caret must be a <button> with aria-expanded and accessible label.
  • Mark current page via aria-current="page" in addition to the visual --active modifier.
  • Focus ring on items must remain visible — never set outline:none without a replacement.
  • Mobile drawer (see drawer) replaces this component below 1025 px and must visually mirror it (per memory).

Tokens used

  • --layout-sidebar (width = 280 px)
  • --bg-elev, --bg-mute, --line
  • --accent (active bar)
  • --fs-300, --fw-medium
