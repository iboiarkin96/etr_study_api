Right-rail "On this page" navigation for prose docs. Auto-generated from <h2> / <h3> headings; the current section gets a flash highlight as the reader scrolls.

Example

On this page

  1. Introduction
  2. Why it matters
    1. Cost
    2. Risk
  3. How it works
  4. Trade-offs
  5. Further reading

Variants

  • H2-only — flat list, no nesting.
  • H2 + H3 — nested two levels (default).
  • Sticky — pinned to viewport top with top: calc(var(--layout-topbar) + var(--space-4)).

Markup

<aside class="docs-toc" aria-label="On this page">
  <p class="docs-toc__title">On this page</p>
  <ol class="docs-toc__list">
    <li class="docs-toc__item docs-toc__item--active">
      <a class="docs-toc__link" href="#intro">Introduction</a>
    </li>
    <li class="docs-toc__item">
      <a class="docs-toc__link" href="#how">How it works</a>
    </li>
  </ol>
</aside>

States

  • Default — muted text.
  • Hover — text colour --text-strong.
  • Active — accent left bar (2 px), background tint, section-flash animation (200 ms).

A11y

  • Wrap in <aside aria-label="On this page">.
  • Use <ol> — order matches the document flow.
  • Each heading in the body must have an id matching the TOC href.
  • Provide a skip-link to the TOC for keyboard users.
  • Mobile: TOC is hidden; the TOC FAB opens it as a bottom-sheet.

Tokens used

  • --text, --muted, --accent
  • --fs-200, --fw-medium
  • --space-2 (item gap)
  • --ease-out (flash transition)
