Roles

Pick your role

Every role has its own hub with a tech radar and a 25-practice handbook. The order below follows the SDLC from product intent through delivery and operation.

SDLC role hubs

  • Manager — product intent, discovery, prioritisation, stakeholder alignment.
  • SA — requirements, contracts, documentation principles, system shape.
  • Architect — system boundaries, NFRs, decision records, technical roadmap.
  • Developer — implementation, tests, CI, code health, delivery workflow.
  • QA — test strategy, exploratory testing, quality gates, release sign-off.
  • SRE — reliability, on-call, runbooks, postmortems, capacity.

What you'll find in each hub

  • Overview — what the role does on this project and where to start.
  • Tech radar — 25 practices triaged as adopt / trial / assess / hold for this role.
  • Practice handbook — one self-contained page per practice with the rationale, the minimum viable workflow, and links to the supporting templates and runbooks.

Cross-role anchors

  • Methodology — project method, scope, success criteria.
  • System design — authoritative architecture and NFRs.
  • ADRs — recorded architectural decisions.
  • RFCs — implementation specifications.
  • Runbooks — cross-cutting incident procedures.
