Developer practices · handbook

Overview

This is the reference handbook for the 25 developer practices that separate engineers at top-tier startups (Stripe, Shopify, GitLab, Cloudflare, Datadog, HashiCorp, Linear, Figma, Vercel, Netflix) from the average team. Each page is written to be the only document you need to read to understand the practice — definition, why it matters, how named companies apply it, the parts it has, common pitfalls, the quality bar, and references.

The radar with all 25 axes lives on practices.html. This index lets you jump to any page directly.

Group A · Code craft & quality (5)

01Trunk-Based Development

Every engineer commits to main daily; nothing branches longer than 24 hours, anything else hides behind a flag.

3.0 → 8.5 / 10

02Test-Driven Development

Failing test first, smallest production change to green, then refactor — Kent Beck's red-green-refactor loop.

2.5 → 7.5 / 10

03Small PRs

< 400 changed lines, reviewed and merged in under a day; stacked diffs for bigger logical work.

4.0 → 8.5 / 10

04Pair & Mob Programming

Two engineers (pair) or the whole team (mob) on one keyboard for the hardest work — shared context, zero bus-factor.

2.0 → 7.0 / 10

05Continuous Refactoring

Every PR leaves the code slightly better — Boy Scout rule + Fowler's catalogue of behaviour-preserving moves.

3.5 → 8.0 / 10

Group B · Delivery & automation (5)

06CI & green-trunk discipline

Every commit deployable; main is red < 15 min/year; one of the four DORA "elite" predictors.

5.0 → 8.5 / 10

07Feature Flags

Deploy ≠ release; LaunchDarkly / GrowthBook gate every risky change; mandatory flag-lifecycle policy.

3.0 → 8.0 / 10

08Pre-commit hooks

Formatters, linters, type-check, secret-scan, conventional-commits — all run on the laptop before push.

5.0 → 8.5 / 10

09Conventional Commits

Machine-readable commit history; automated changelog + SemVer bumps from the log itself.

3.5 → 8.0 / 10

10Progressive Delivery

Canary & blue-green with automated health gates; Argo Rollouts / Flagger; rollbacks in seconds.

1.5 → 7.5 / 10

Group C · Architecture & design (5)

11Twelve-Factor App

Heroku's 12 rules for cloud-native services: config in env, stateless processes, logs as streams, dev/prod parity.

5.5 → 8.5 / 10

12API Contract-First

OpenAPI / GraphQL SDL / Protobuf written before handlers; SDKs & mocks generated from the schema.

4.0 → 8.5 / 10

13DDD tactical patterns

Aggregates, value objects, services, repositories — Evans & Vernon's tactical vocabulary.

2.5 → 7.5 / 10

14Type safety

Strict mode everywhere; any/cast/ignore reviewed individually; 15% fewer prod bugs (MSR 2017).

5.0 → 8.5 / 10

15Design Docs & RFCs

1–4 pages before code; Context / Decision / Alternatives / Trade-offs; reviewed asynchronously.

3.5 → 8.5 / 10

Group D · Observability & operability (5)

16Structured Logging

JSON logs with request_id, trace_id, user_id, severity — searchable as a database, not as text.

4.5 → 8.5 / 10

17Distributed Tracing

OpenTelemetry spans on every inbound and outbound call; the longest span is your problem.

2.0 → 7.5 / 10

18SLOs in code

Error budgets that pause releases when reliability dips; multi-burn-rate alerts from Google SRE Workbook.

1.5 → 7.5 / 10

19Error monitoring

Sentry / Datadog Error Tracking; every uncaught exception routed to a CODEOWNERS team with a triage SLA.

4.0 → 8.5 / 10

20Performance budgets

Each surface has a CI-enforced budget (LCP, p99, bundle size); PRs that breach are blocked.

2.0 → 7.5 / 10

Group E · DX, learning & ownership (5)

21Reproducible dev env

Devcontainers, Nix, or Compose: clone → one command → exactly the same toolchain on every machine.

4.5 → 8.5 / 10

22Docs-as-Code

Markdown in the repo, reviewed in the PR, built by CI, deployed with the app — Stripe / Twilio / GitLab model.

4.0 → 8.5 / 10

23Tech-Debt Register

Every debt written down, scored, owned; 15–20% of each quarter's capacity reserved to pay it down.

2.5 → 7.5 / 10

24CODEOWNERS & inner-source

Every path mapped to a team; inner-source contribution guide makes the monorepo open without ungoverned.

3.0 → 7.5 / 10

25Continuous Learning

Brown-bags, book clubs, conferences, personal learning budget — retention & quality investment combined.

3.5 → 8.0 / 10
