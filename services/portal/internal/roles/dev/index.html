Developer engineering portal

What this portal is for

This portal is the engineering home of the project. It answers one question:

"What does a top-tier startup engineer actually do, day-to-day, that an average engineer does not?"

The answer is 25 practices, grouped into 5 themes — code craft, delivery, architecture, observability, and developer experience. Each practice is described on its own page with definition, why it matters, how Stripe / Shopify / GitLab / Cloudflare / Datadog and friends apply it, the anatomy, common pitfalls, and a quality bar.

Start here

  • Practices radar (25) — the visual panorama: AS-IS vs TO-BE scores on all 25 axes plus a card-per-practice summary.
  • Practice handbook (25) — direct entry to any one of the 25 detail pages.

The 5 groups

  • Group A · Code craft & quality — trunk-based development, TDD, small PRs, pair & mob programming, continuous refactoring.
  • Group B · Delivery & automation — CI green-trunk discipline, feature flags, pre-commit hooks, conventional commits, progressive delivery.
  • Group C · Architecture & design — 12-Factor App, API contract-first, DDD tactical patterns, type safety, design docs & RFCs.
  • Group D · Observability & operability — structured logs, distributed tracing, SLOs in code, error monitoring, performance budgets.
  • Group E · DX, learning & ownership — devcontainers, docs-as-code, tech-debt register, CODEOWNERS, continuous learning.

How this set was chosen

The 25 practices were selected from the public engineering writing of teams that publish how they work (Stripe, Shopify, GitLab, Cloudflare, Datadog, HashiCorp, Linear, Figma, Vercel, Netflix, Google SRE) cross-referenced with the DORA Accelerate research on what predicts elite delivery performance. The bar for inclusion was: a practice has to be (a) attributable to a named source or originator, (b) verifiable in production at one or more top-tier startups, and (c) something an individual engineer can practice — not just an org-level policy.

The AS-IS / TO-BE scores on each practice are an honest, conservative assessment of where this project stands today and where the maturity audit (May 2026) target sits.
