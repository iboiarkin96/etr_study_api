Reference QA Checklist 0004

Regression

Run this when the change set is wide (refactor, dependency bump, infra swap) or smoke alone isn't enough confidence. Broader than smoke, shallower than feature-level acceptance.

When to run

  • Major refactor merged (touches > 1 module).
  • Dependency upgrade across the stack (FastAPI / Pydantic / SQLAlchemy / Alembic).
  • Container or runtime image change.
  • Database engine, indexing, or migration change.
  • Docs portal frontend change touching shared assets (sidebar, theme, runtime bundle).

API coverage — golden paths for every resource

#CheckExpected
1.1User: POST → GET → PUT → PATCH → GETEach call matches its spec; final GET reflects all changes.
1.2Conspectus: POST → GET due → POST review → PATCHEach call matches its spec; review action transitions state per spec.
1.3Schedule summary GETAggregates data correctly for the seeded set.
1.4Error log: POST → GETListing returns the just-posted error with correct shape.
1.5OpenAPI baseline testGreen; no contract drift.

Docs coverage

#CheckExpected

2.2Theme + responsive matrix on one representative page.Light, dark, 360px, 768px, 1280px all render correctly.
2.3Diagrams: open one PlantUML diagram in the lightbox.Renders; closes on Escape.

Defect-prone areas — re-test on every regression

Add to this list whenever a defect is closed. Sources: post-incident review, defect history, audit findings.

#AreaReasonRe-test
3.1Sidebar drawer focus trapPast a11y regression on SafariOpen drawer, Tab through, confirm trap; Esc closes.
3.2TOC FAB close behaviorReopened twice after refactorsOpen overlay TOC, click outside, Esc — all close cleanly.
3.3Idempotency window edgeWindow math has been wrong beforeReplay just-after-window; treated as new request.

Cross-cutting checks

#CheckExpected
4.1Error envelope on a 404, 422, 401, and one 5xxAll conform per envelope.
4.2Field conventions on three responsesUUIDs, ISO-8601 UTC, snake_case keys.
4.3Pagination on a list endpointFirst page → next → empty next_cursor.
4.4Auth boundary on three endpointsNo-token / wrong-token / valid-token reach 401 / 401 / 2xx.
4.5Migration up/down round-trip on a representative datasetIdempotent; row counts match.

Pass / fail rules

  • Pass: all rows pass; new defects (if any) are S3 or below.
  • Fail: any S1/S2 found; or any defect-prone area regresses; or contract drift on the OpenAPI baseline.
  • Add to defect-prone areas any newly closed defect that surfaced through this run.
