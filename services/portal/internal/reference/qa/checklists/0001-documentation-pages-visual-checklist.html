Reference QA Checklist 0001

Documentation pages visual verification

Manual visual + interaction pass for documentation pages on desktop and mobile. Run this when validating portal UI changes before release; mark every row Pass / Fail / Needs follow-up and attach the screenshots listed below.

Scope

Manual QA at desktop and mobile widths. Covers visuals, navigation, and the interactive controls a reader uses while moving through a page.

Preconditions

  • Open the docs home page and at least one page from each major section.
  • Test in one Chromium-based browser and one WebKit browser when possible.
  • Run checks in both light and dark theme modes.

Checklist by page sections

SectionWhat to verifyExpected result
Top navigationBrand link, breadcrumbs, theme toggle, active-page state.Links open correct pages, active state correct, theme persists on reload.
BreadcrumbsHierarchy for the current page; parent items clickable.Path matches page location; parent links valid.
Main contentTitles, paragraphs, tables, lists alignment.No clipping, overlap, broken spacing, or unreadable contrast.
On-this-page TOCAnchors from H2/H3, active section highlight while scrolling.Anchor list present; active state follows scroll.
History footerdocs-history rows with date / author / note.Readable in both themes; author chip resolves.

Interactive controls

ControlWhereWhat to check
Theme toggleTopbar actions.Cycles system → light → dark; aria-label updates.
Sidebar drawer (mobile)Topbar hamburger at ≤760px.Opens and closes; Escape closes; focus returns to trigger.
TOC FAB (mobile)Bottom-right when right-rail is off-screen.Opens overlay TOC; backdrop click closes.
Copy-code buttonsTop-right of every docs-code block.Copies code to clipboard; "Copied" feedback.

Screenshots required

  • Topbar with theme toggle visible (desktop, light theme).
  • Right-rail TOC with active section highlight while mid-scroll.
  • Mobile sidebar drawer open.
  • Mobile TOC FAB open.
  • One docs-code block with copy-button hover state.

Pass / fail rules

  • Pass: all rows verified; no blocking UI defects.
  • Fail: any broken navigation, inaccessible control, or severe visual regression.
  • Needs follow-up: minor visual issue that does not block content access.
