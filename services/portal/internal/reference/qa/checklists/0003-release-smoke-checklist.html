Reference QA Checklist 0003

Release smoke

The broad shallow check before tagging a release. Deeper walkthrough lives in the release smoke playbook; run this on the candidate build and paste the result into the release ticket.

Preconditions

  • CI green on the release commit; image built and tagged.
  • Migrations (if any) applied to the target environment.
  • Release ticket exists; build SHA and target environment recorded.
  • Auth token, base URL, and a clean test account ready.

Health, readiness, OpenAPI

#CheckExpected
1.1GET /health200; body matches the documented health envelope.
1.2GET /ready200 once dependencies are reachable.
1.3GET /openapi.json200; valid JSON; info.version matches release.
1.4Container logs at startupNo unhandled exception; one log line per route registered.

API golden paths

#CheckExpected
2.1Authenticated read endpoint returns 200 with valid shape.Schema validated.
2.2Create + read-back on User.POST 201, GET 200; row present.
2.3Create + read-back on Conspectus.POST 201, GET 200; row present.
2.4Create + read-back on Error log.POST 201, GET 200; row present.
2.5Idempotency replay on one POST with the same key.Same response twice; no duplicate row.
2.6One forced 422 and one 401.Both conform to the error envelope.

Docs portal

#CheckExpected
3.1Open /internal/index.html.Loads with no console errors; sidebar populates.
3.2Open one page from each major section (ADR, runbooks, API ref, QA).Page renders; breadcrumbs correct.
3.3Theme toggle cycles light / dark.Persists on reload.
3.4Drawer (mobile width) opens, closes on Escape.Focus trapped while open.
3.5Open Swagger UI page.Renders; one operation expanded shows correct request/response.

Observability

#CheckExpected
4.1Pick one X-Request-ID from a 2.x call.Found in the structured log; method/path/status match.
4.2Pick one X-Request-ID from a 4xx call.Logged at warn level (or per policy); no PII leaked.
4.3Metrics scrape (if enabled).Endpoint reachable; counters incremented after smoke calls.

Sign-off

  • Go: all rows pass; defects (if any) below S2.
  • Conditional go: S2 defects with explicit acceptance from the release owner.
  • No-go: any S1 open or any row in Health, API golden paths, or Observability failed.

Paste the sign-off note from the release smoke playbook into the release ticket with evidence.
