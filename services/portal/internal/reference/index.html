What a Reference page looks like

01

Identification

Page head, eyebrow, lede, pill row (status / version / owner). Same chrome on every reference page so readers can scan it in two seconds.
02

Tables and field rows

Structured data lives in docs-table--sticky-head or docs-field-row. Required/optional badges, type cells, constraint notes — never a wall of prose.
03

Endpoint cards & code

API operations use docs-endpoint-card with docs-method-badge; sample requests and responses use docs-code with copy buttons.
04

History footer

Every page closes with docs-history: date + author-chip + one-line note. Audit trail without leaving the page.
