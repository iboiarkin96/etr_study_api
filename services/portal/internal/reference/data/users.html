Table: users

End-user identity row. Every user is keyed by an external system_user_id assigned by the source system, and an internal stable client_uuid exposed in API surfaces. Each user belongs to exactly one source system (systems) and has exactly one IANA timezone (timezones); soft-deleted rows carry an explanatory reason from invalidation_reasons.

Constraints

  • Primary key: client_uuid.
  • Unique constraint: uq_users_system_user_id_system_uuid over (system_user_id, system_uuid) — a source system never assigns the same system_user_id to two rows.
  • Indexes (non-FK): none beyond FK indexes (system_uuid, invalidation_reason_uuid) which carry index=True per ADR 0029.

Relationships

  • Belongs to systems via system_uuid (1-N from systems to users).
  • Optionally belongs to invalidation_reasons via invalidation_reason_uuid (1-N from reasons to users; nullable when is_row_invalid = 0).
  • References timezones via timezone (1-N from timezones to users; default UTC).

Lifecycle

  • Created by: POST /api/v1/user.
  • Updated by: PATCH /api/v1/user/{system_uuid}/{system_user_id} · PUT /api/v1/user/{system_uuid}/{system_user_id}.
  • Read by: GET /api/v1/user/{system_uuid}/{system_user_id}.
  • Retention: soft-delete via is_row_invalid = 1 with mandatory invalidation_reason_uuid. Rows are never hard-deleted.
