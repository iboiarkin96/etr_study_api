2026-Q2 Accessibility audit — WCAG 2.2 AA

Quarterly portal-wide accessibility assessment against WCAG 2.2 Level AA. Covers all 487 internal pages plus the public docs surface. Method: automated scan via validate_docs_a11y.py (axe-core 4.x in headless Chromium) + manual sample of 12 representative pages. This audit is the canonical ratification of the a11y state for Q2 2026; the next audit lands at 2026-Q3 end.

Verdict: AA · partial — portal meets AA on 94% of pages; three critical findings (see § 03) block the «100% AA» claim. Two of three are scheduled for fix in 2026-Q3 Week 1; the third needs a Visual Kit token update tracked in ADR backlog.

02

Method

Three layers — automated, sampled-manual, assistive-tech. Each layer catches what the others miss.

  • Layer 01 · Automated scan. scripts/validate_docs_a11y.py runs axe-core 4.x in headless Chromium across every services/portal/internal/**/*.html. Catches contrast, missing alt text, ARIA mismatches, focus-order traps, missing form labels.
  • Layer 02 · Manual sample. 12 pages picked to cover the kit (hero · principle cards · matrix tables · dep-lanes · status-timeline · concept-layers · live-tickers · roles-radar). Manual keyboard walk + zoom-to-200% + reduced-motion + dark-theme.
  • Layer 03 · Assistive tech. Spot-check with VoiceOver (macOS Sonoma) on 4 hub pages — internal home, SA reference, principles, services/api. Catches reading-order issues axe doesn't see.

Sampled pages (Layer 02): internal/index.html, foundations/reference/sa/index.html, foundations/reference/sa/principles.html, foundations/reference/sa/process.html, foundations/reference/sa/catalog.html, foundations/reference/sa/spine-matrix.html, foundations/reference/sa/entity-card.html, foundations/reference/sa/voice.html, foundations/reference/sa/freshness.html, foundations/reference/sa/ownership.html, services/api/index.html, team/roles/sa/radar.html.

03

Findings — by WCAG criterion

Ordered by severity. Each finding names the WCAG 2.2 success criterion, the affected pages, and the remediation path.

SC 1.4.3
FAIL
Contrast (Minimum) — muted-soft text on bg-mute fails 4.5:1.

The --muted-soft token on --bg-mute background measures 3.8:1 in light theme; 4.5:1 is the AA threshold. Affects 23 pages, mostly in metadata rows («Owner · Lifecycle · Last verified») on entity cards and the kicker text on principle cards.

Fix: bump --muted-soft from #6b7280 to #5c6470 in tokens/_theme-light.css. Dark theme already passes (5.4:1). Owner: Architect. ETA: 2026-Q3 Week 1.

SC 2.4.7
FAIL
Focus visible — interactive checkbox in principles.html checklist invisible on focus.

The interactive five-question checklist hides the native <input type="checkbox"> with absolute positioning to render a custom .check-item__box. The :focus-visible outline is on the input (not the box), so keyboard focus is invisible.

Fix: already authored — .check-item__input:focus-visible ~ .check-item__box selector adds a 2px outline. Will land in the same PR as this audit. Owner: SWE lead. ETA: 2026-Q3 Week 1.

SC 1.4.10
FAIL
Reflow — spine matrix table overflows on iPhone SE (375px) without horizontal scroll.

The 21-row × 6-column .spine-matrix on spine-matrix.html § 04 wraps inside .docs-table-wrap which has overflow-x: auto, but the parent .docs-prose clips the scroll region on mobile, hiding the right-most «Resource» column.

Fix: add contain: inline-size to .docs-table-wrap and verify on iPhone SE viewport. Owner: Architect (UI Kit). ETA: 2026-Q3 Week 2.

SC 1.3.1
WARN
Info & relationships — RACI matrix cells use colour alone to convey R/A/C/I.

On ownership.html § 02 The matrix, the R/A/C/I pills use background colour but the letter is already in the cell, so this is borderline — VoiceOver reads «R» / «A» / «C» / «I» correctly. Warning, not failure.

Fix (optional): add title attributes on cells for hover-tooltip («Responsible», «Accountable», etc.). Owner: SA lead. ETA: not scheduled.

SC 2.5.8
WARN
Target size (Minimum) — sidebar caret icons measure 22×22px on mobile.

WCAG 2.2 added SC 2.5.8 with a 24×24px target-size minimum (AA). The sidebar's expand/collapse caret icons fall 2px short on mobile.

Fix: increase .docs-sidebar__caret padding from 6px to 8px (renders at 26×26px). Owner: SWE lead. ETA: 2026-Q3 Week 2.

SC 1.4.11
PASS
Non-text contrast — UI components ≥ 3:1 against adjacent colours.

All tone pills (accent · info · success · warn · rose · teal) carry borders at 3.2:1 minimum against both light and dark backgrounds. Verified via axe-core.

SC 2.1.1
PASS
Keyboard — every interactive element reachable via Tab.

Manual walk on 12 sample pages: theme toggle, search box, sidebar nav, in-page TOC, checklist checkboxes, dep-lanes chips — all reachable, all activated via Enter / Space. No keyboard traps.

SC 2.4.4
PASS
Link purpose (In context) — link text describes the destination.

«See: 🧭 Diátaxis deep-dive →» — link text + surrounding card describe the target. No «click here» / «read more» anywhere in the portal. Validated by manual sample.

SC 2.3.3
PASS
Animation from interactions — respects prefers-reduced-motion.

Five-question checklist progress-bar transition, sidebar caret animation, hover-lift on cards — all gated behind @media (prefers-reduced-motion: reduce). validate_docs_a11y.py enforces this on every CSS file in the kit.

SC 1.3.2
PASS
Meaningful sequence — reading order matches DOM order.

VoiceOver spot-check on 4 hub pages: headings read in order, sidebar reads before main content, in-page TOC reads last. No CSS-driven reorderings break the reading flow.

SC 4.1.2
PASS
Name, role, value — interactive elements expose role + state.

Theme toggle has role="button" + aria-pressed; checklist checkboxes are real <input type="checkbox"> so role + checked state are native; sidebar nav is a real <nav> with aria-label.

05

Scope & non-goals

  • In scope: WCAG 2.2 Level AA on every hand-authored HTML page under services/portal/internal/ + public docs surface (services/portal/public/).
  • Out of scope (this audit): WCAG 2.2 Level AAA — we target AA. Pursuing AAA is a separate decision (open ADR if needed).
  • Out of scope: Section 508 specifically — we treat it as a subset of WCAG 2.2 AA. Cross-check: zero pages had a Section-508-only violation.
  • Out of scope: Auto-generated reference (OpenAPI explorer, pdoc) — those are built artefacts owned by their upstream tools. Tracked separately in audits/docs.
  • Out of scope: 3rd-party embeds (Backstage screenshots, external diagram services) — we don't embed any today; if added, they need their own a11y sign-off.
