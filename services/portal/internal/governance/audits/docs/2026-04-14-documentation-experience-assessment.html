1. Scope and methodology

1.1 Scope

Documentation experience (DX) here means using our docs end to end: nav, search, examples, API reference, runbooks, and trust signals (changelog, errors). Table 1 is generic; Table 2 is about this repo's services/portal/ tree and automation.

1.2 Method

The review proceeds in four steps:

  1. Reference list — Table 1 describes good DX practices in neutral terms.
  2. Map to repo — Table 2 scores each practice (1–10) for this repo, with pointers (paths, ADRs, tools).
  3. Summarize — overall score and gaps in section 4 (narrative, weighted model, Top-five gap status).
  4. Follow up — page history.

Scores are subjective guesses.

1.3 What was examined

  • Main page: services/portal/index.html
  • Design system: services/portal/style/index.html
  • ADRs: services/portal/internal/governance/adr/
  • RFCs: services/portal/internal/governance/rfc/
  • How-to guides: services/portal/internal/handbook/howto/
  • Internal service pages: services/portal/internal/
  • Runbooks: services/portal/internal/sre/runbooks/
  • OpenAPI / Swagger UI (static): services/portal/public/reference/api/index.html
  • Python API reference (pdoc): services/portal/pdoc/index.html
  • Quality gates: make docs-check, changelog policy, OpenAPI governance references in ADRs.
  • Tools for automated documentation pipeline: make docs-fix, make docs-check, make docs-a11y-check

1.4 What this assessment is not

  • Not a substitute for tests with real readers or support data.

1.5 Industry context, PET scope, and standards we do not chase (yet)

Big tech often runs central doc portals, search and analytics, translation workflows, and AI assistants with dedicated teams. We use a lighter Table 1 / Table 2 rubric in git per ADR 0024.

The docs are PET-scale: static HTML and generated API docs, not a paid developer platform. Many Table 1 rows (multi-SDK tabs, enterprise portal, huge search) are benchmarks, not requirements. Low scores often mean "not worth it yet" or "English-only by choice," not failure. Priorities sit in section 4.3 (Top five gaps).
