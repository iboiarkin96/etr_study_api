Requests for Comments

RFCs spell out how we build and run things — implementation detail, design rationale, and operational steps. ADRs record what we decided and why; an RFC usually follows an ADR and provides the specification the ADR intentionally omits. New RFCs start from the RFC template and follow the lifecycle in ADR 0018.

Multi-service IA program

The growable internal portal: Backstage-subset entity catalog (System / Component / Library / API / Resource), per-type spine matrix, the four missing systems-analyst genres (Requirements / Domain model / Process documentation / NFRs) inside every Component, a Foundations layer for cross-cutting content, and a Practices layer for cross-service ways of working (testing pyramid, docs-as-code, observability discipline, …). Sized to scale from 4 to 50+ entities without structural rework. Refines ADR 0026 and operates inside the internal layer landed in RFC 0004.
