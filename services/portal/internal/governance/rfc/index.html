Multi-service IA program

The growable internal portal: Backstage-subset entity catalog (System / Component / Library / API / Resource), per-type spine matrix, the four missing systems-analyst genres (Requirements / Domain model / Process documentation / NFRs) inside every Component, a Foundations layer for cross-cutting content, and a Practices layer for cross-service ways of working (testing pyramid, docs-as-code, observability discipline, …). Sized to scale from 4 to 50+ entities without structural rework. RFC 0006 extends this with audience-portal (L1) and domain-hub (L2) layers above the entity catalog so the portal also scales from 250 to 1000+ pages. Refines ADR 0026 and operates inside the internal layer landed in RFC 0004.

RFC 0006

Portal IA scalability — three-layer model

Proposed
Three orthogonal layers — L1 audience portals (sa · dev · sre · qa · pm · manager · security · architect), L2 domain hubs (api · data · observability · security · governance · delivery · platform), L3 entity catalog (per RFC 0005). One page, one URL, three entry points. Three-phase migration with zero-broken-link redirect contract.
