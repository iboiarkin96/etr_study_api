02

Page lifecycle — driven by git

Every page lives in git. To ship one you go through four steps: draft on a branch, open a PR, pass review (LLM checks + human review), then merge into main. After merge the page is live and search picks it up.

  1. Branch
  2. Pull request
  3. Review & audit
  4. Merged
  • 1 · Branch git switch -c docs/<slug> Cut a feature branch from main. The draft lives there — your local workspace, no one else sees it yet.
  • 2 · Pull request gh pr create Push the branch and open a PR. CI runs make docs-check (links, h1, a11y, acronym casing) before any human looks at it.
  • 3 · Review & audit LLM prompts + human Two gates in one stage: an LLM runs canned audit prompts (Diátaxis fit, tone, broken claims) and a human reviewer signs off the substance.
  • 4 · Merged git merge → main Squash-merge into main. The page is live on the portal, the search index rebuilds, and the history block records the change.
