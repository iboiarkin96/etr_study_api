Overview

The On this page FAB is a fixed-position floating button that gives readers quick access to in-page section navigation. It is built at runtime from the article's h2/h3 headings — authors don't write it by hand. The FAB is always collapsed on first render (a 52 × 52 px circle), floating at the left edge of the content area. Clicking it expands to a 232 px card showing the full heading list. As the reader scrolls, the entry matching the current section is highlighted (scroll-spy) with a dot-indicator ::before pseudo-element and accent colouring.

The FAB position is controlled by the CSS custom property --docs-toc-fab-left set on .internal-layout__shell and inherited by the position: fixed aside. The value shifts automatically when the sidebar collapses — no JavaScript required for the slide animation. On viewports ≤ 1024 px the FAB is hidden entirely via CSS; the spy still mounts but does no visible work.

An expanded card also exposes a dim button that sets the card to 22 % opacity so it stops obscuring the article text. A one-time promo toast ("On this page") fires on the reader's first visit and offers an "Open" action that expands the FAB directly.

Source-of-truth for the contract is this page; source-of-truth for the code is services/frontend/portal/assets/docs-nav.js (functions initAutoInPageToc(), initInPageTocScrollSpy(), ensureTocAnchorIds(), collectTocEntries()) and services/frontend/portal/assets/docs.css (selectors prefixed .docs-inpage-toc*). Grid layout helpers (.docs-page-layout__inner) are documented in Navigation, search, discovery §On this page.

Goals and non-goals

GoalsNon-goals
Always show the reader where they are in the article without forcing them to scroll the rail. Replicate the left sidebar tree — the TOC is page-scoped, not site-scoped.
Build itself from the actual content (h2/h3) so docs authors don't drift. Index every heading level — h4+ are deliberately excluded to keep the rail scannable.
Keep window scroll under the reader's control: the rail's auto-scroll only moves its own overflow, never the page. Smoothly animate nav.scrollTop while the user is mid-flick — that would fight the reader's input.
Survive page-level DOM rewrites (e.g., page-history fold, lifecycle status injection) without pinning the active state to a stale entry. Re-build the TOC on every layout change. Targets are captured once; the spy is defensive but the list is stable.
Hide entirely on viewports ≤ 1024 px so the article gets the full reading column. Replace the FAB with a drawer/sheet on tablet/phone — that role belongs to the in-page hero chips.

Anatomy

  1. FAB circle (collapsed state) — aside.docs-inpage-toc.docs-inpage-toc--collapsed rendered as a 52 × 52 px circle via border-radius: 999px. Position: position: fixed !important; left: var(--docs-toc-fab-left, 12px); top: 68px;. The !important overrides the [data-tooltip] attribute selector (specificity 0,1,0) which appears later in docs.css and would otherwise silently reset position to relative. When collapsed, the aside carries data-tooltip="On this page" so the label appears on hover.
  2. Position CSS variable system — --docs-toc-fab-left is set on .internal-layout__shell (the nearest non-fixed ancestor):
    • Sidebar open: calc(var(--shell-left) + 262px)
    • Sidebar collapsed (.is-sidebar-collapsed): calc(var(--shell-left) + 10px)
    • --shell-left: max(0px, calc((100vw - 1280px) / 2)) — handles viewports wider than 1280 px
    Because position: fixed children still inherit CSS custom properties from their DOM ancestors, this lets a non-fixed parent drive the FAB's horizontal position. The transition left 320ms ease produces a smooth slide when the sidebar toggles.
  3. Head row (expanded) — .docs-inpage-toc__head: flex row containing the title span (.docs-inpage-toc__title, "On this page"), the dim button (.docs-inpage-toc__dim-btn), and the collapse toggle (.docs-inpage-toc__toggle with a chevron-down SVG). In collapsed state the head shrinks to exactly 52 × 52 px and centers the list-lines icon. The toggle button carries aria-expanded, aria-controls, and an accessible label.
  4. Dim button — .docs-inpage-toc__dim-btn: visible only when expanded. Clicking toggles .docs-inpage-toc--dim on the aside, which sets opacity: 0.22; hovering over a dimmed card recovers it to opacity: 0.9. Carries data-tooltip="Dim — hides behind text". Dim state resets whenever the card is collapsed.
  5. Inner scroll container — .docs-inpage-toc nav with max-height: min(60vh, 28rem); overflow-y: auto; overscroll-behavior: contain;. This is the container the auto-scroll moves; the page never moves because of the spy. Animated via max-height: 0 → min(60vh, 28rem) and opacity: 0 → 1.
  6. Entry list & dot-indicator — flat <ul> of li > a. h3 entries get .docs-inpage-toc__item--nested for the indent. The active link class .docs-inpage-toc__link--active sets accent text colour, accent-tinted background, font-weight: 500, and reveals a 4 × 4 px filled circle via the ::before pseudo-element (transform: scale(0) → scale(1)). The same anchor also gets aria-current="location". There is no left border stripe — the dot replaces it.
  7. Heading anchor ids — every h2/h3 gets a stable id assigned by ensureTocAnchorIds() if the author didn't write one. Slugs are based on the heading text and de-duplicated against ids already present on the page (see Failure modes).
  8. Promo toast — one-shot "On this page" toast on every first-visit to a docs page. Title: "On this page". Body: "Quick section navigation — click the floating button on the left to open." Dismiss label: "Later". Primary action label: "Open" — calls applyCollapsedState(false) directly. Duration: 4 000 ms. Lives in showStickyTocPromoToast() in docs-nav.js; rendering contract in Popups and overlays.

Behaviour contract

The scroll-spy in initInPageTocScrollSpy() runs on every scroll, resize, and once on window.load. Each run it computes a single active target against this contract:

RuleWhy
Activation line at 96 px from viewport top. The active entry is the last one whose h2/h3 top is ≤ 96 px. Picks the section the reader is "in" rather than the one they scrolled past five lines ago. 96 px clears the sticky top-bar zone.
Default before first crossing = first entry. When no heading has crossed the line yet (reader is at the top of the page), entry #0 is active. Otherwise the rail would render with no active highlight at the canonical reading position (top of page).
Near-bottom safety net is gated on scrollY > 0. The "you've reached the end → activate last entry" branch only fires after the reader actually scrolls. At scrollY = 0 a short page (or a long page with assets still loading) trivially satisfies scrollBottom ≥ docBottom − 8; without the gate the last entry is incorrectly pinned on initial load. See Failure modes #2.
Detached targets are filtered out per tick. updateActive() recomputes liveTargets = targets.filter(t => t.el.isConnected) before measuring. Detached elements return top: 0 from getBoundingClientRect(), which would always satisfy top ≤ 96 and pin that entry forever. See Failure modes #1.
Auto-scroll inside the rail keeps the active link visible. When nav.scrollHeight > nav.clientHeight, keepActiveLinkVisibleInNav() nudges nav.scrollTop just enough to land the active <a> within a 24 px margin from the top or bottom of the visible band. On long articles the TOC overflows its max-height: min(75vh, 36rem). Without this the active highlight scrolls out of sight and the rail looks broken.
Window scroll is never moved. The auto-scroll only writes to nav.scrollTop; it never calls scrollIntoView() on the link. scrollIntoView() walks every scrollable ancestor including the document. Moving the page in response to scroll-spy would feel like the browser fighting the user.
Updates are rAF-throttled. Multiple scroll events between two animation frames coalesce into one measurement. getBoundingClientRect() in a tight scroll loop is the canonical reflow trap.
Page-history section is excluded from the TOC. normalizeDocsPageHistory() runs before initAutoInPageToc(), removing the <h2>Page history</h2> (it becomes the <summary> of a fold) before the TOC sees it. If the TOC captured that h2 first, the later normalization would detach it. See Failure modes #1 for the historical bug.

Implementation

Init order in document.addEventListener("DOMContentLoaded", …) (services/frontend/portal/assets/docs-nav.js):

  1. normalizeDocsPageHistory() — folds the page-history section into a <details> and removes its <h2>. Must run before the TOC is built.
  2. initAutoInPageToc() — wraps content in .docs-page-layout__inner, calls ensureTocAnchorIds(), builds the rail from collectTocEntries(), attaches the collapse toggle and the (optional) promo toast.
  3. restoreInitialHashPosition() — re-runs scrollIntoView() on the URL hash target after layout transforms.
  4. initInPageTocScrollSpy() — captures targets via document.getElementById, registers scroll + resize + one-shot load listeners, runs updateActive() once.

Accessibility

  • Container is <aside> with aria-labelledby="inpage-toc-heading"; the heading paragraph has a unique id.
  • The inner <nav> is also labelled by the same heading id.
  • Active link gets aria-current="location"; the dot-indicator is purely visual.
  • Collapse toggle exposes aria-expanded, aria-controls, and a label that toggles between "Hide On this page" and "Show On this page".
  • When the aside is collapsed, it carries data-tooltip="On this page" for hover context; this attribute is removed when expanded so the tooltip doesn't appear on an already-visible card.
  • Dim button carries data-tooltip="Dim — hides behind text" for discover­ability; it is only rendered (and reachable via Tab) when the card is expanded.
  • Auto-scroll only writes scrollTop on the nav — focus is never moved as a side effect.
  • prefers-reduced-motion: reduce is honoured by upstream CSS; the spy itself does instant updates.

Failure modes

  1. Detached active target pinned forever. If a later DOM step removes a heading the spy already captured, getBoundingClientRect() on the now-detached node returns top: 0, which always satisfies the ≤ 96 px rule and pins that entry as active.
    Guard: updateActive() filters targets by el.isConnected every tick. Root-cause fix: normalizeDocsPageHistory() runs before initAutoInPageToc() so its <h2> never enters the TOC.
  2. Last entry pinned at scrollY = 0 on short or slow-loading pages. The "you scrolled to the end" branch incorrectly fires because innerHeight ≥ scrollHeight when assets (e.g., PlantUML images) haven't grown the document yet.
    Guard: nearBottom requires scrollY > 0; a one-shot window.load listener re-runs the spy after subresources settle.
  3. Heading-id collision with section id. If the author writes <section id="page-history"><h2>Page history</h2>, ensureTocAnchorIds() assigns the h2 a de-duplicated id (e.g., page-history-2) and the TOC links there. Anything that later removes that h2 orphans the link.
    Guard: avoid putting id on a wrapping <section> that has the same slug as its inner heading. The page-history fold sidesteps this by removing the h2 before the TOC sees it.
  4. Author writes a manual "Contents" block. removeLegacyManualContents() scrubs <nav> elements that contain a "Contents" heading; if a custom variant slips through, the page renders both the manual and the auto rail. Mitigation: rely on the auto-TOC; do not write inline contents.

Acceptance checklist

  • FAB renders as a 52 × 52 px circle on viewports > 1024 px; hidden at ≤ 1024 px.
  • FAB is collapsed on initial render — unconditionally, regardless of page type or TOC length.
  • Promo toast fires on first visit; "Open" in the toast expands the FAB; "Later" dismisses. Toast does not reappear after first dismissal.
  • Hovering over the collapsed FAB shows the data-tooltip="On this page" tooltip; tooltip disappears when the card is expanded.
  • Clicking the collapsed circle expands to 232 px card; clicking the chevron-down collapses back to a circle. Transition is smooth (width + border-radius).
  • FAB left position slides smoothly when the sidebar collapses or expands (transition: left 320ms). On wide viewports (> 1280 px) the position still aligns with the sidebar toggle.
  • Dim button appears in the expanded head; clicking it makes the card semi-transparent. Hovering over a dimmed card recovers opacity. Collapsing resets dim.
  • At scrollY = 0 the first entry is active; "Page history" is not in the list.
  • Scrolling past each h2 moves the active dot-indicator to that entry within one frame.
  • On a TOC longer than the card's max-height, the active link is always inside the visible band and the page itself does not move when the card auto-scrolls.
  • No regressions on pages with no h2/h3: layout falls back to a single-column article with no FAB rendered.
