Postmortem template

Template. Copy this file into a dated incident page (e.g. 2026-05-08-api-5xx-spike.html) and fill in each section. Adapted from sre.google example postmortem.

Incident summary
One paragraph: what happened, when, what was affected.
Impact
Users affected, requests dropped, SLO budget burnt, money / brand impact.
Timeline
UTC timestamps for: detection, declaration, mitigation, all-clear, resolution.
Root cause
The contributing factors. Multiple causes are normal.
Resolution
What was done to stop the bleeding and what was done to make it durable.
Detection
What told us. If we found out from a customer, that is a finding too.
Action items
Linked tickets with owners and ETAs. Each item answers "would this fix it next time?"
Lessons learned
What we now know that we did not before. Blameless prose only.
