Internal service docs — layout and how to add pages

The layout of every page under services/portal/internal/, the assets behind the sidebar and the top nav, and the steps to add a new API resource.

Shared assets

Every page under services/portal/internal/ that renders the sidebar loads the assets below and mounts the UI Kit v3 data-component="sidebar" region.

AssetRole
services/frontend/portal/assets_v2/runtime/internal/entry.cssUI Kit v3 entry bundle — tokens, layout, all .docs-* components.
services/frontend/portal/assets_v2/runtime/internal/entry.jsUI Kit v3 runtime — mounts data-component="sidebar", theme toggle, lightbox, TOC FAB.
services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.jsonThe single source of truth for the left nav tree. Edit when adding or renaming pages.

Page anatomy

Internal pages set class="docs-shell" on the <body>, data-portal="internal" on <html>, and follow a fixed top-to-bottom order:

  1. <header class="topbar"> — brand link, breadcrumbs, theme toggle.
  2. <aside data-component="sidebar" data-nav-src="…"> — JSON-driven sidebar.
  3. <main> containing <article class="docs-prose">.
  4. Inside the article: <header class="page-head"> with a docs-chip-row, <h1>, and <p class="lede">.
  5. One or more <h2 id="…"> sections (optionally with nested <h3>). Tables wrapped in .docs-table-wrap; code blocks as <pre class="docs-code"><code class="language-…">.
  6. <footer class="docs-history"> — page history list, last block inside <main>.
  7. <aside class="docs-toc"> + <button data-component="toc-fab"> — right-rail TOC and mobile FAB.

Canonical example: services/portal/internal/index.html.

How to add a new API resource

Worked example: adding Order.

  1. Create the resource folder and files:
    • services/portal/internal/api/order/index.html — hub page with the business context, the contract summary, and a method index.
    • services/portal/internal/api/order/operations/ — one HTML file per HTTP endpoint, named from the HTTP method and the path template (for example post-api-v1-order.html, get-api-v1-order--order_id.html). Do not use the Python operationId as the filename.
  2. Register in the sidebar. Open services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json and extend the tree with the new paths, relative to services/portal/internal/.
  3. Link from the API hub. Add a card and a link on services/portal/internal/reference/api/index.html.
  4. Write the content. Put business rules, field contracts, and technical notes on the resource index.html. Add one operation page per endpoint. Use internal/reference/api/user/index.html as a reference.
  5. Run make docs-fix to normalize the HTML and rebuild the search index.

How to edit existing pages

  • Sidebar labels or order: edit nav-tree-internal.json.
  • Top navigation / breadcrumbs: hand-edit per page; the UI Kit topbar is plain HTML.
  • Page body: edit the HTML directly, then run make docs-fix to normalize the file and rebuild the search index.
  • Renaming a page: update the file path in nav-tree-internal.json and any cross-links. Add a redirect stub at the old URL if outside bookmarks may exist.
