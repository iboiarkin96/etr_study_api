Information architecture

How the nine screens connect. Sitemap, route table, navigation model, deep-link grammar. Each answer here is a design constraint, not a technical implementation — the router just enforces what this page decides.

Deep-link grammar

Telegram bots deliver deep-links via the startapp parameter of the HTTPS launch URL:

https://t.me/<bot_username>/<app_short_name>?startapp=<payload>

Telegram passes <payload> to the WebApp via WebApp.initDataUnsafe.start_param. The client parses it and dispatches to a route. The grammar below is enforced client-side; a malformed payload lands Today with the payload logged for debugging (not shown to the user).

Payload shape Route dispatched to Emitted by
empty / absent/ (Today)Bot launch button, default.
conspectus_<uuid>/conspectus/:uuidBot notification for a specific card.
focus/ (Today), then auto-launch /focus if GET /api/v1/conspectuses/due returns ≥ 1 card; otherwise stay on Today with a mini-toast «Nothing due right now».Bot «start review» quick-action.
errors_new/errors?compose=1Bot «log error» quick-action.
schedule_<YYYY_MM_DD>/schedule?day=<YYYY-MM-DD> (router re-inserts dashes)Bot pushed a heat-map digest.
anything else/; log payload; do not surface an error—

Design constraint: payloads are opaque to the user. They never appear in copy, never appear in URLs the user sees, never in error messages. Bot authors treat them like access tokens — short, one-purpose, hard to mistype (all-lowercase with underscores, no hyphens or dots).

Back-navigation model

Two rules resolve every back-button question:

  1. Deep-linked entry always returns to Today. If the user opened /conspectus/<uuid> via a bot push, Back goes to Today — not «previous conspectus» (there wasn't one). This preserves the mental model «Today is home».
  2. Sequential entry returns one step. If the user tapped through Today → Conspectus list → Conspectus detail, Back retraces one hop at a time. Router history is the source of truth.

Exceptions:

  • Focus. Back on Focus prompts a confirm-exit dialog («Exit session? Progress won't be lost.»). Rationale: accidentally closing an in-progress review is expensive.
  • Search overlay. Back closes the overlay, doesn't route away.
  • Onboarding. Back is hidden — the only exit is «Skip» or «Get started», both of which persist dismissal.

URL-shape rules

  • No user data in the path. UUIDs only; no titles, no email fragments, no personal identifiers.
  • No client-generated state in the URL. Focus session state is Zustand-only; you can't share a Focus-mid-session URL.
  • Query params are always optional. Stripping ?cursor=… from a URL must load a valid screen (first page).
  • Trailing slashes are absent. /conspectus, not /conspectus/. Router normalises.
  • Kebab-case only for path segments. No /errorHistory — but this app has no multi-word segments today.
