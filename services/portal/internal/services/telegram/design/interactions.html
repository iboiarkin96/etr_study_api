Interactions

The vocabulary every screen shares: motion, haptics, gestures, and Telegram chrome. Centralised here so screens don't reinvent — pick a token, don't tune a value.

Motion vocabulary

Seven spring configurations. Every animation in the app picks one and only one. Values live in src/shared/motion.ts; touching a screen animation without picking a token is a code-review block.

Token Character Use for Springs
enterQuick, confidentCards / screens appearing on route change.stiffness: 320, damping: 30
exitFast, resoluteCards leaving after grade / swipe.stiffness: 280, damping: 26
revealCurious, slightly bouncyFocus card back → front on tap.stiffness: 240, damping: 20
swipeDirect 1:1 to fingerDue-card drag; no spring on drag, spring on release.Drag: linear · Release: stiffness: 400, damping: 32
popSnappy micro-interactionButtons on tap-down / tap-up.stiffness: 500, damping: 24
liftGentle continuousProgressRing fill on streak update.stiffness: 180, damping: 22
toastAttentive, then stillToast entering / dwelling / leaving.stiffness: 300, damping: 28

Global rule: prefers-reduced-motion: reduce collapses every token to instant (duration: 0). No functional path depends on animation completing.

Haptics vocabulary

Seven patterns mapping to the Telegram SDK's hapticFeedback API. Every screen picks from this list — never call impact('heavy') ad-hoc.

Pattern SDK call Fires when
selectselectionChanged()User taps a due card, opens Focus, picks a search result.
tap-lightimpactOccurred('light')Button press, tab switch, minor state change.
tap-medimpactOccurred('medium')Card swipe committed, grade selected in Focus.
tap-heavyimpactOccurred('heavy')Streak milestone, destructive action confirm.
notify-successnotificationOccurred('success')Error logged, review committed after a struggle-hard.
notify-warnnotificationOccurred('warning')Skip a due card («not today» tag).
notify-errornotificationOccurred('error')Mutation failed (rare — most errors show as toast, not haptic).

Gesture vocabulary

Four intents on the due-card. One card, four directions, one intent per direction. Nothing else on a due card is gesture-driven — tap opens Focus, long-press opens context menu, that's it.

Direction Intent Review tag posted Haptic
Swipe right«I know this» — mark reviewed, no work needed.easytap-med
Swipe left«Not today» — snooze until next slot.forgotnotify-warn
Swipe down-right (short)«Star» — mark high-priority for the day.(no review — profile flag)tap-light
Swipe down-left (short)«Delete» — archive from the list. Destructive; BiometricManager confirms.(no review — archive)tap-heavy

Contract note: swipe → review tag maps to the same POST /api/v1/conspectuses/{id}/actions/review. Snooze is not a separate endpoint. The forgot tag tells the server «bump next-interval, don't count as reviewed» — the server computes what «not today» means in ETR terms, the client just declares intent.

Reduced-motion contract

Every animation in the vocabulary above collapses to zero-duration under @media (prefers-reduced-motion: reduce). Haptics don't have a reduced equivalent (they're system-level), but every haptic call is opt-out via user setting (Profile → Haptics: On/Off) which defaults to On in Telegram and Off if the device explicitly requests reduce.

Gestures work identically under reduced motion — the drag itself isn't «animation», it's finger-tracking. Only the spring-back release is collapsed.
