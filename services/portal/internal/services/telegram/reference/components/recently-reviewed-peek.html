Component spec: RecentlyReviewedPeek

Quiet editorial mini-list on the .tma-peek--lined primitive — up to five «recently reviewed» conspectuses. Each row is a real <Link> to /conspectus/$conspectus_uuid so the peek is a proper drill-down surface, not decoration.

Overview

Consumes the first five items from the due-cards cache — until a dedicated GET /conspectuses?sort=-last_reviewed_at&limit=5 lands, «recently reviewed» is derived from the same list that feeds DueCardsList (see the source's inline TODO). Self-hides on an empty list — a zero-state peek would be a phantom surface. Rows sit inside a .tma-section plate; the peek's own lined dividers replace section-internal borders.

Accessibility

  • Labelled section. aria-labelledby="recent-h" ties the header id to the group; screen readers announce the section title before the rows.
  • Rows are real links. Not clickable divs — Tab / Enter / middle-click / open-in-new-tab all work.
  • Tooltip. Section header wears the shared .tma-tip pattern with the localised «Cards you've touched recently…» hint.
