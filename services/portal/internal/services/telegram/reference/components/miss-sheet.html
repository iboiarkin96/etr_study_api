Component spec: MissSheet

The bottom-sheet composer for a new miss entry — scrim + slide-up panel + textarea + Cancel/Save. Born screen-local on Errors (T-20), promoted to a shared surface when the Focus debrief (SessionRecap) started reusing it. The component stays deliberately dumb: it renders and emits; the owning screen wires the mutation.

Overview

One composer, three flows: the Errors screen's blank «Log a miss» (no context chip), the /errors?prefill_from=session deep-link (chip from query params), and the Focus debrief (chip from the tapped recap row). In every flow the sheet owns only presentation — auto-focus on open, trimmed-empty Save gating, a saving lock with spinner label, an optional error line. The POST itself, optimistic caching and rollback live in the parent's useCreateError call.

Props

PropTypeRequiredDefaultPurpose
open boolean Yes — Mount/unmount gate. Closing clears the draft text — the sheet never leaks a draft across opens.
saving boolean Yes — Locks the sheet open (scrim tap / Esc / Cancel ignored), disables the textarea, flips Save to the spinner label. Parent maps it from useCreateError().isPending.
errorText string | null No null Inline role="alert" line under the textarea. The draft survives the error — the user retries, never retypes.
contextLabel string | null No null Ember-tinted chip above the textarea naming the linked conspectus. Fed by the deep-link params on Errors and by the tapped row on Focus.
onClose () => void Yes — Scrim tap, Esc, or Cancel. Parent guards it against saving.
onSave (message: string) => void Yes — Fired with the trimmed draft. Disabled while the trim is empty.

Accessibility

  • Modal semantics. role="dialog" + aria-modal="true"; the scrim is a real labelled <button>, not a click-catching div.
  • Escape containment. The Esc listener binds at the window capture phase and stops propagation — host screens with their own Escape bindings (Focus exits to Today on Esc) can never be triggered underneath the open sheet.
  • Auto-focus. The textarea receives focus on open, so the keyboard flow is type → Save without a tab stop.
  • Save gating. Native disabled on Save while the trimmed draft is empty or a save is in flight.

Used on

  • Screen · Errors — blank add («+» / CTA) and the ?prefill_from=session deep-link flow.
  • Screen · Focus — complete phase, opened per-row by SessionRecap.
