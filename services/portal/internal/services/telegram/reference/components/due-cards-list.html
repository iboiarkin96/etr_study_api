Component spec: DueCardsList

The «up next» block on Today — a dense list of conspectuses due for review right now. Each row is a horizontally-draggable card: tap opens the detail; a swipe past the commit threshold fires one of three review outcomes and drops the row optimistically. Pure UI — the mutation itself lives on the parent (Today/index.tsx) so the component tree stays free of TanStack Query + Auth context and can be story-rendered in isolation.

Overview

Two intents share the row: read this note (tap) and log a review outcome (swipe). The tap target sits on the same <Link> the previous non-swipe version shipped, so a11y users, keyboard users, and screen-readers keep the tap semantics — Focus mode (T-18) will be the primary place they log an outcome. Swipes are the thumb-driven shortcut for sighted mobile users who don't want to open every card before grading it.

Three design constraints shape the implementation. First, the tap-to-detail contract has to survive — screen readers announce «Open «X»», keyboard focus rings render, and a short tap can't accidentally register as a swipe. That's why the drag surface is an inner .tma-swipe__fg around the <Link> and thresholds are large enough (100 px arm) that stray horizontal jitter never fires. Second, snooze is a tag, not an endpoint — the ETR contract offers only easy · hard · forgot, and forgot is the semantic snooze (schedule resets, card resurfaces sooner). Third, the mutation itself lives outside the component so stories and tests don't need auth context.

Anatomy

  1. Root list. <div class="tma-section__plate tma-section__plate--overflow tma-tip"> with role="list" and the section tip. Owns the tooltip copy and the section framing; no per-row logic.
  2. Row wrapper. <motion.div class="tma-swipe"> per item, carrying three data-attrs — data-armed (past ±40 px), data-commit (past ±100 px / -200 px), data-committing (mutation firing). Wraps two layers.
  3. Action bar · right. <div class="tma-swipe__bg tma-swipe__bg--right" data-tone="accent"> — the «Easy» acknowledgement that fades in as the row drags right. Left-justified, accent tint.
  4. Action bar · left. <div class="tma-swipe__bg tma-swipe__bg--left"> — «Hard» (warn tone) or «Forgot» (danger tone) depending on drag depth. Right-justified so the copy rides the trailing edge.
  5. Draggable foreground. <motion.div class="tma-swipe__fg"> — the top layer. Owns drag="x" and the spring-back animation. Suppresses drag entirely under prefers-reduced-motion: reduce or when the row is already committing.
  6. Cell. The inner <Link className="tma-cell"> — icon + main (title + subtitle) + aside + chevron, exactly the same shape as the pre-swipe version. Tap opens the detail; draggable={false} stops native browser drag-and-drop interfering.

Props

Prop Type Required Default Purpose
items DueConspectus[] — alias of components['schemas']['ConspectusResponse'] Yes — The due list. Only conspectus_uuid, title, slot, next_review_at, schedule_revision are consumed; the rest of the payload is carried but not read (kept so future rows can enrich without a prop change).
onReview (uuid: string, tag: ReviewTag, expected_schedule_revision: number | null) => void No no-op Fired when a swipe crosses the commit threshold. Parent decides how to mutate — production wires useReviewConspectus().mutate; stories pass a console log. The optional expected_schedule_revision is forwarded for optimistic concurrency (server sends 409 if it disagrees). Not called on taps or on sub-arm swipes.

Gesture map

Thresholds live in swipe-thresholds.ts. The pure resolver resolveTag(offsetX) is exported so tests and this doc share the numeric contract.

Gesture Offset (px) Emitted tag Action-bar tone Effect
Tap — — — Route to /conspectus/$conspectus_uuid. No review fired.
Short swipe (either side) |Δx| < 100 — Fades in past ±40 px (armed), fades out on release Spring back to x: 0. No callback.

Swipe left Δx ≤ −100 and > −200 hard Warn Same optimistic-remove path as easy, warn-toned bar.
Swipe left (deep) Δx ≤ −200 forgot Danger The «snooze» outcome — schedule resets. Danger tone on the bar; icon glyph flips from · to ↺.

Mutation contract

The parent wires useReviewConspectus().mutate to onReview. That hook is where the interesting bits live:

  • Optimistic removal. onMutate reads the current ['today.conspectuses.due', client_uuid] cache, filters out the reviewed row, writes it back, and returns the snapshot. The row leaves the list the instant the swipe crosses commit — no round-trip wait.
  • Rollback on error. onError restores the snapshot verbatim. The parent (Today/index.tsx) also mounts an ErrorInline under the list bound to review.reset() so the user sees «Couldn't save the review — reverted» with a Retry.
  • Cache invalidation. onSuccess invalidates today.schedule.summary, me.stats, and schedule.history — the streak orb, summary strip, and heat-map refetch. The due list itself is already fresh (we mutated it in onMutate).
  • Idempotency-Key. Auto-injected by the shared API middleware (T-12) per ADR 0006. A lost 4G reply on retry never doubles the review.
  • expected_schedule_revision. Forwarded on every request. The server 409s if the row's revision has moved since we last read it (another device also reviewed it, say); the mutation surfaces the error and the row snaps back.

Design tokens

  • Tone inks. Action bars are tinted from --tma-tone-accent (easy), --tma-tone-warn (hard), --tma-tone-danger (forgot). At armed the bar is color-mix(… 22%, transparent); at commit it deepens to 45%.
  • Motion. Spring config { type: 'spring', stiffness: 400, damping: 40 } for the drag-return; commit animates the foreground off-screen with the same spring. All keyframes suppressed under prefers-reduced-motion: reduce.
  • Elevation & surface. Foreground uses --tma-surface-plate so the row sits opaque over the action bar. Row separator (1 px --tma-border-regular) is on the swipe wrapper, not the cell, so it stays put while the fg travels.
  • Typography & spacing. Action-bar copy — --tma-fs-small, --tma-fw-semi, uppercase, wide letter-spacing. Cell body inherits all tier-1 spacing tokens from cell.css.

Accessibility

  • Semantics preserved. The list is role="list"; every row is role="listitem" around the same <Link> that shipped before swipes — screen readers still announce «Open «X»» and Enter opens the detail. Swipe is a sighted-user shortcut, not the only path to review.
  • Action bars hidden. Both .tma-swipe__bg nodes carry aria-hidden="true". Screen readers never see the pending-tag chrome.
  • Tap-priority. The 100-px commit threshold + 40-px arm dead-zone ensure a short thumb-tap doesn't accidentally register as a swipe. draggable={false} on the anchor stops native drag-and-drop from stealing the pointer.
  • Reduced motion. useReducedMotion() disables the drag prop entirely — the row falls back to tap-only. The row-exit transition also collapses to a plain opacity fade.
  • Keyboard. Not swipe-driven. Focus mode (T-18) is where keyboard-first users log reviews via the Again/Hard/Good/Easy grade buttons — the design contract makes Today's swipes an affordance for touch, not the primary review surface.

Behaviour contract

  • Mount. Renders once useConspectusesDue.data has at least one row; empty-state and loading are handled by the parent (EmptyToday / DueCardsSkeleton).
  • Commit animation. On a valid commit the foreground animates to x: ±480 with opacity: 0. Once the mutation resolves (success or error), the row unmounts from the list (AnimatePresence plays the exit height-collapse).
  • Concurrent swipes. Each row owns its own motion value and state, so committing one row doesn't block another's drag. The mutation hook itself allows multiple in-flight calls.
  • Error path. On failure the cache rollback re-inserts the row at its previous position and the parent renders ErrorInline under the list. The next successful swipe on any row clears the error.
  • Ownership of exit animation. The list wraps rows in <AnimatePresence initial={false}>; the row emits exit variants. Reduced-motion path skips the height collapse to keep the layout snappy.

Tests

  • Four resolveTag cases pinning the numeric contract: right-past-100 → easy, left-100..−199 → hard, left-past-200 → forgot, taps/short-swipes → null.
