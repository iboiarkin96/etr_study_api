Toast primitive

Non-blocking notifications rendered at the bottom of the viewport above the Telegram MainButton safe zone. Four tones, optional single action, motion-token-driven slide-and-fade, stack cap of three. «This happened, no confirmation needed» — WebApp.showPopup / showAlert still handle the cases that need one.

API

const { toast, dismiss, toasts } = useToast();

toast({
  message: 'Miss noted',
  tone: 'success',                     // 'info' (default) | 'success' | 'warning' | 'error'
  durationMs: 4_000,                   // defaults: 4 s / 6 s w/ action / 0 = sticky
  action: { label: 'Undo', onAction: () => ... },
});

Returns the toast's id — pass to dismiss(id) to close it programmatically before its timeout. The stack caps at 3; a 4th push shuffles the oldest out.

Focus grade errors keep their existing inline ErrorInline banner instead — a full-viewport banner has more space for the retry affordance than a toast would, and the user's flow is already interrupted by a failed grade.

Behaviour

  • Stack. Up to three visible at once. A 4th push drops the oldest — the toast that has been on screen longest.
  • Auto-dismiss. 4 s by default, 6 s when an action is present (user needs a beat to reach the button). durationMs: 0 keeps the toast open until dismissed programmatically or via the close «✕» button — reserved for critical state that must not vanish.
  • Action. Single ghost-button on the right; tapping fires the callback and dismisses the toast. Not composable with a second action — if the user needs two choices, that's a modal, not a toast.
  • Motion. 260 ms slide-in from below + fade (cubic-bezier(0.2, 0, 0, 1) — the app's EASE_OUT). prefers-reduced-motion: reduce collapses to instant.

Degrade paths

  • SSR / build. Provider is a plain React context; renders nothing when there are zero toasts. No window access at import time — the timer setup lives inside useEffect / callbacks.
  • Called outside provider. useToast() throws with a clear message pointing at providers.tsx. That's a wiring error, not a legit fallback — a silent no-op would swallow feedback that the caller depends on.
  • Unmount mid-toast. Provider clears every pending dismissal timer in its cleanup effect, so React never gets a stale setState on an unmounted tree.
