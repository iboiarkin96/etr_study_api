Component spec: AchievementChips

The achievements shelf on the Profile screen — a fixed, additive-only set of badges (12 keys) from GET /api/v1/me/achievements, rendered as a chip row. Unlocked chips carry their earned tone ink; locked chips sit dimmed with a mono progress fraction («86/100» — binary badges hide it). A shelf of small facts, not a gamified wall.

Overview

The chip row is a pure projection of the server's computed achievement set — the component holds no unlock logic of its own. The server derives every badge on read (streak badges ride the LONGEST streak, so a broken current streak never re-locks them); the client only maps the closed key set to icon + copy and splits the row into unlocked-first ordering. Because progress arrives pre-clamped, the fraction can never read «102/100».

Props

PropTypeRequiredDefaultPurpose
items Achievement[] — { key, unlocked, progress, target } from GET /me/achievements Yes — The server's computed set, passed through unchanged. Unknown keys are ignored (forward-compatible with additive server keys).

Everything else is internal: key → icon/copy mapping and unlocked-first ordering live inside the component. The parent only supplies the payload from useMeAchievements.

Canonical keys

The closed set the client maps — mirrors the server contract in getMeAchievements. Order in the response is part of the contract; the component only lifts unlocked chips ahead of locked ones.

Binary badges (target = 1) hide the mono fraction when locked — «0/1» reads as noise; the dimmed label already says «not yet». The set is additive-only: keys this build doesn't know are skipped silently.

Chip states

StateVisualExtra
unlockedTone-ink chip — 8% tint plate, 22% border, ink-coloured icon + label (flat register, no gradients)—
lockedNeutral plate, 45% opacity, muted iconMono progress fraction «progress/target» as the trailing label

Tone assignment: streak badges ride ember (--tma-tone-accent), milestone counts ride success (--tma-tone-success) — the earned ink is part of the badge's identity, mirroring how grade buttons own their tones on Focus.

Design tokens

  • Tint recipe. Plate color-mix(in oklab, var(--chip-ink) 8%, var(--tma-surface-plate)), border 22% — the same flat ink-tint register as MissPeek and SessionRecap rows.
  • Type. Label at --tma-fs-small semi; progress fraction in --tma-font-mono at --tma-fs-micro.
  • Shape. Full-radius pills (--tma-rad-full) — the shelf reads as badges, not buttons.
  • Motion. None of its own — the row rises with its Assemble slot; reduced-motion collapses with the choreography.

Accessibility

  • Semantic list. A <ul> labelled from i18n («Achievements»); each chip is a non-interactive <li> — there is nothing to activate, so nothing pretends to be a button.
  • Progress in the name. Locked chips carry an aria-label with the localized progress («Reviews · 86 of 100») so the fraction isn't a bare visual.
  • Icons decorative. All SVGs are aria-hidden; the label text carries the meaning.
  • Contrast. Locked-state 45% opacity applies to the chip surface, not the text token — labels stay ≥ AA on both themes.
