Component spec: SearchOverlay

The global command-palette overlay for the Telegram Mini App — a single-instance role="dialog" that mounts above every screen once SearchProvider flips open=true (either via the topbar search glyph or the global Cmd/Ctrl+K shortcut). Fuzzy-filters the user's whole conspectus library in-memory, autofocuses its input on open, and navigates to /conspectus/$uuid on Enter. The React shell is thin; the visual comes from the shared .tma-cmdk CSS primitive so it stays consistent with the rest of the kit.

Overview

Search in the TMA is a global primitive, not a screen — a user opens it from anywhere (topbar glyph on desktop-shaped viewports, Cmd/Ctrl+K everywhere) and lands directly on a conspectus. The overlay is the visible half of that primitive; the invisible half is SearchProvider, which owns the open/close state and the global keybinding so no consumer has to wire either. Only one instance is ever mounted — the root layout owns it.

The component's design pressure is speed over cleverness. The whole library is fetched once with staleTime: Infinity so subsequent opens are keystroke-fast; the filter is pure and unit-tested; substring match ranks earlier-match first + shorter-title next so the closest hit is always at the top. The React shell handles the two things CSS can't own — focus save/restore across open/close and the keyboard grammar (Enter · Esc · arrows · Ctrl-J/N/P) — and delegates every pixel to .tma-cmdk.

Props

Prop Type Required Default Purpose
open boolean Yes — Controls visibility. When false, the component returns null — it fully unmounts, so state (query string, cursor, focus snapshot) is discarded between sessions. The parent SearchProvider owns the flag.
onClose () => void Yes — Fired for every close path: Esc, backdrop click, result click, Enter-after-navigate. The overlay never mutates its own open — the parent flips it in response.

Selector Element Purpose

.tma-cmdk__box <div> Inner box — fluid up to 560 × 460 px, animated rise on mount.

.tma-cmdk__list <ul role="listbox"> Rendered only when hits.length > 0. Carries an id so aria-activedescendant on the input can resolve.
.tma-cmdk__group-title <li role="presentation"> Non-selectable section header — «N matches».

Data model

The overlay owns none of the fetching. It reads two things: useConspectusesList() for the corpus and a pure filterConspectuses() for the ranking.

  • Hook. useConspectusesList() — TanStack Query, staleTime: Infinity, enabled gated on auth so an unauthenticated open doesn't fire a request. Fetches the whole library once (limit: 500) and returns the array plus { isPending, fetchStatus }. Subsequent opens are pure cache reads → keystroke-fast filtering.
  • Filter. filterConspectuses(rows, query, lowercaseTitles?) in search-filter.ts — pure, unit-tested, substring match. Ranks earlier match position first, then shorter title. The overlay pre-lowers the title index once per corpus and passes it as the third argument so per-keystroke filtering doesn't re-lowercase the entire library.

Behaviour

  • Enter. Navigates to /conspectus/$conspectus_uuid of the highlighted row via TanStack Router's useNavigate, then calls onClose(). Enter is a no-op when hits.length === 0.
  • Escape. Calls onClose(). React synthetic on the dialog root — no global listener needed.
  • Arrows & Emacs bindings. ↑ / ↓ move the cursor by one; Ctrl-N / Ctrl-J move down, Ctrl-P moves up (Emacs muscle memory for keyboard-driven users). All four are guarded on hits.length === 0 so there's no phantom cursor when the listbox isn't rendered.
  • Cmd+K / Ctrl+K. Not bound in the overlay — SearchProvider owns the global shortcut as the open/close toggle. Duplicating it here would double-fire on close.
  • Hover-follows-pointer. mouseenter on an item sets the cursor, but only when a pointerMovedRef flag is true. Autoscroll from arrow-key navigation calls scrollIntoView which fires a spurious mouseenter under the pointer's resting position; the flag suppresses that so the cursor doesn't yank back to wherever the mouse happens to sit.
  • Backdrop close (drag-safe). The overlay tracks both mousedown and mouseup targets — it closes only if both hit the outer .tma-cmdk element. This prevents a text-selection drag that starts inside the box and releases outside from closing the overlay mid-copy.
  • IME composition. Enter is ignored when e.nativeEvent.isComposing is true or keyCode === 229. Both checks together handle every current CJK IME (macOS Japanese IME sets isComposing; some Android IMEs still surface only the legacy 229).
  • Focus save + restore. On open, snapshots document.activeElement and autofocuses the input; on close, restores focus to the snapshot (WAI-ARIA modal contract). The topbar search glyph — the usual trigger — receives focus back so keyboard users don't get dropped at the top of the document.

States

State Trigger Visible content
unmounted open === false Returns null. No DOM, no listeners, no focus impact.
loading open + fetchStatus === 'fetching' + no cached corpus Empty-state block reading «Loading library…». Cold first open only — cache-hit path skips this state.
hint open + query.trim() === '' Empty-state block reading «Start typing…». The default landing state.
match open + query non-empty + hits.length > 0 Listbox — group header («N matches») + one .tma-cmdk__item per hit.
no-match open + query non-empty + hits.length === 0 Empty-state block reading «No matches for «xyz»». Query echoed back so the user sees what was searched.

The empty-state block and the listbox are mutually exclusive by design — never rendered together. This keeps the live-region announcements ambiguity-free («N matches» vs «No matches» vs «Loading library»).

Accessibility

  • Dialog contract. role="dialog" + aria-modal="true" + aria-label on the root. Focus is trapped inside the box for the whole open lifetime.
  • Combobox pattern. The input carries aria-activedescendant pointing at the current option's id. The listbox has role="listbox"; each row has role="option" and aria-selected reflecting the cursor. Screen-reader users hear the row content as the cursor moves without focus leaving the input.
  • Live region. .tma-visually-hidden[role="status" aria-live="polite"] announces match-count / loading / empty transitions. Polite (not assertive) so it doesn't interrupt what SR is already reading; and separate from the input's own value so the two announcements don't collide.
  • Focus save + restore. Explicit snapshot on open + explicit restore on close — the platform's own dialog focus-return is not reliable across in-app portal mount points, so the overlay owns it.
  • Keyboard shortcuts. Every binding (Esc, Enter, ↑, ↓, Ctrl-J/N/P) works from the input without ever moving keyboard focus — the input stays the tab-index owner throughout.

Do / Don't

Do Don't
Use the shared .tma-cmdk kit primitive for the whole visual — the React shell only owns behaviour. Mount a second overlay. There's exactly one, owned by the root layout via SearchProvider; a second instance would double-fire the global Cmd/Ctrl+K binding.
Keep Cmd/Ctrl+K reserved to SearchProvider as the global toggle. The overlay only handles keys that make sense inside it (Esc, Enter, arrows, Emacs cursor moves). Hard-code slot tones on .tma-cmdk__item-icon. Every tone read comes from the shared SLOT_TONE map so a slot-palette change ripples cleanly.
Keep filterConspectuses pure and unit-tested — it's the only place ranking lives. Add a title="…" to the trigger or to any kit primitive — the kit forbids native tooltips (they collide with .tma-tip's pseudo affordance).
Pass a pre-lowered title index into filterConspectuses so per-keystroke filtering doesn't re-lowercase the corpus. Fetch inside the overlay. Data lives in useConspectusesList so staleTime: Infinity can do its job across opens.
