Component spec: MissPeek

Quiet one-line pill under YesterdayDigest — surfaces the weekly miss count and links to /errors. Reads as D3 editorial peek, not D2 CTA (that lives above it on the «Start Focus» button). Silence, not zero, is the empty state.

Overview

Reuses the Errors screen's useErrors query, filters client-side for entries within the trailing 7 days, renders a single pill: «N misses this week →». Renders nothing when the count is 0 (no misses is silence, not a zero-badge) and nothing while the query is pending (a late-arriving pill would flicker the compose above it).

Props

None. The pill reads directly from useErrors — passing a count as a prop would duplicate cache access and let the pill drift out of sync with the Errors screen.

Accessibility

  • Real link. Renders as <Link to="/errors"> — keyboard focus, Enter, middle-click all just work.
  • Pluralised label. i18n handles the 1-miss / N-misses split; the aria label carries the raw count for AT users.
