Component spec: StreakOrb

The signature block of the Today screen — a living, breathing radial-gradient bubble carrying the current streak. Three visual states, three sizes, one CSS primitive. Renders a StreakStats shape verbatim (no adapter layer) and resolves its data-state from just (streak, dueToday).

Overview

The orb is the D1 breathing element of Today — see ADR 0038 · TMA screen DNA for the app-wide rule and reference · Today · pin 2 for how it sits inside the screen. It occupies the full width of the hero, moves on its own, and reads as a physical object under a light source. Whether the user reviews or not, the surface stays alive.

Two design constraints shape the implementation. First, the orb has to feel like a physical object under a light source — hence the four-layer radial body, the rotating conic sheen and the double glare. Second, it has to communicate zero-work without going grey — hence the sage twin under data-state="rested" instead of a flat empty state.

All motion (bubble morph, breathe, iris rotation, glare drift, celebrate pulse and ring) sits in CSS; the React component picks a state and emits the DOM. Every animation halts under prefers-reduced-motion: reduce.

Anatomy

  1. Root. <div class="tma-orb"> — carries data-state, role="img", and the aria-label with the streak count. Also owns two pseudo-elements: ::before = radial body (4 stacked gradients), ::after = celebration halo (dormant until data-state="celebrate").
  2. Sheen. <span class="tma-orb__sheen"> — 10-band conic rainbow, mix-blend-mode: overlay, rotates continuously.
  3. Glare. <span class="tma-orb__glare"> — two radial specular highlights (primary top-left, counter bottom-right), mix-blend-mode: screen, drifts on a 9s ease.
  4. Number. <span class="tma-orb__num"> — the streak count, tabular numerics, heavy weight. Font size scales with the size tier (see orb.css).
  5. Cap. <span class="tma-orb__cap"> — «day streak» copy (i18n key today.streak.unit), 10 px uppercase eyebrow.
  6. Tip wrapper (external). The tap-target ancestor is <div class="tma-tip tma-tip--below"> that carries data-tip={t('today.streak.tip')}. It has to sit on a wrapper because .tma-tip's own ::before / ::after would collide with the orb's pseudo-elements.

Props

State machine

The visual state is derived by resolveState(data, dueToday) in StreakOrb.tsx and applied as data-state. Order matters — a milestone streak with zero due still fires celebrate:

function resolveState(data, dueToday) {
  if (data.current_days > 0 && data.current_days % 30 === 0) return 'celebrate';
  if (dueToday === 0) return 'rested';
  return 'warm';
}

Why milestone wins over rested — and stays ember, not sage

Rested = sage. Empty day (nothing due today) is a calm, cool moment. The orb's four-layer radial body swaps to the mint/ice/sea-foam family; sheen bands stay in a single cool gamut so the surface never reads «tired» or «grey». See .tma-orb[data-state="rested"] in orb.css.

Celebrate = ember. A milestone streak (every 30 days) is the party moment — the reward the whole habit product exists to serve. The body stays in the warm ember gradient; on top of that we fire a one-shot tma-orb-pulse (900 ms scale) and reveal the ring halo (::after · tma-orb-ring). Turning it green here would undersell the win — sage is calm, ember is joy.

Precedence. Because celebrate carries stronger meaning than rested (finishing a 30-day streak deserves the party even on a light-workload day), the state machine picks it first. If we ever want a rested-milestone variant (party bubbles on a sage body), it becomes its own data-state — never a silent override of the ember celebrate.

Design tokens

  • Ember palette (warm, celebrate): --tma-ember-300 / -400 / -500 / -600 / -900. Radial body sweeps from -300 through -500 to -900; box-shadow is a soft -600; halo border is -400.
  • Sage palette (rested): --tma-sage-500, --tma-sage-700. Box-shadow keyed off -700; body sweeps through -500.
  • Motion. Six named keyframe loops layered on the body and its children: tma-orb-bubble (slow shape morph), tma-orb-breathe (scale breathing), tma-orb-iris (sheen rotation), tma-orb-glare (specular drift), tma-orb-pulse and tma-orb-ring (one-shot celebrate pair on --tma-ease-spring / --tma-ease-out). Durations and easing values live in orb.css — the stylesheet is the source of truth for exact numbers.
  • Typography. --tma-font-sans, heavy weight, tabular numerics, tight tracking on the numeral; small uppercase eyebrow on the cap.
  • Custom property. --orb-warmth drives the root filter: hue-rotate() (warm state shifts it a few degrees toward red). Overridable inline for per-instance tuning.

Accessibility

  • Role & label. role="img" + aria-label={t('today.streak.aria', { count: data.current_days })}. i18next resolves plural forms per locale — «Current streak is 1 day» / «Current streak is 12 days».
  • Decorative sub-parts. __sheen, __glare, __num all carry aria-hidden="true". Only __cap stays announceable, but its text is the same «day streak» copy already covered by the aria-label.
  • Focus / keyboard. Not focusable — the orb is a display primitive with no interactive behaviour. Tap is handled by the ancestor .tma-tip wrapper (tooltip only, no navigation).
  • Contrast. The numeral sits on a saturated ember gradient. Text is white with a warm text-shadow (0 1px 6px rgba(74, 27, 12, 0.5)). Verified ≥ 4.5:1 against the darkest zone of the body (--tma-ember-900).
  • Motion. All keyframe animations halt under prefers-reduced-motion: reduce. Body freezes to a stable blob shape so the primitive still reads as an object under a light source. Celebrate halo stays fully transparent.

Behaviour contract

  • Mount. Renders only once useMeStats.data resolves. No skeleton — the parent omits the block entirely until data lands, per the «don't dress a rare state as a normal one» rule (an orb-shaped placeholder would look like a broken orb).
  • Re-render. Re-renders on every prop change. State resolution is pure and cheap; the animations are CSS-driven and continue seamlessly across renders.
  • State transitions. Switching between warm / rested triggers a 0.6 s transition on filter and box-shadow. Entering celebrate plays the pulse + ring once (900 ms); leaving celebrate falls back to the running bubble/breathe loop.
  • Ownership of pseudo-elements. The root owns ::before (body) and ::after (halo). Any consumer wrapping the orb in a component that also uses pseudo-elements (e.g. .tma-tip) has to keep them on a sibling wrapper, not on the orb itself.
  • Isolation. The root sets isolation: isolate. Blend modes (overlay on the sheen, screen on the glare) stay contained; adjacent surfaces don't tint.

Used on

  • Screen · Today — hero block, size="lg", driven by useMeStats and dueToday.
  • EmptyToday component (services/telegram/src/screens/Today/components/EmptyToday.tsx) — size="sm", data-state="rested" forced (no data prop; renders just the body + sheen + glare without the numeral cap).
