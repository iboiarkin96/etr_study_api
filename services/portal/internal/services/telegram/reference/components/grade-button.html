Component spec: GradeButton

One of four buttons on the Focus grade strip. The primitive itself is intentionally minimal — a stateless button that emits a grade — but the (grade → label → tone → hotkey) contract it renders is pinned by the shared GRADES array. Consumers can't accidentally paint an Easy button warn-tinted or bind Hard to key 1.

Overview

The Focus grade strip lives on the single most-tapped surface in the app — see ADR 0038 for the app-wide DNA that treats it accordingly. The primitive is deliberately dumb: a button that renders label + hotkey chip, tones itself via data-tone, and emits its grade on press. Everything else — mutation wiring, keyboard binding, enable/disable gating — lives on the parent.

The contract that matters is not the component API; it's the shared GRADES array in grade-spec.ts. Both the visible order (Again · Hard · Good · Easy) and the underlying (label key, tone, hotkey) tuple ship as a single readonly literal so a render, a keyboard binding, and a Storybook story all read the same source.

Props

PropTypeRequiredDefaultPurpose
spec GradeSpec — one entry from the shared GRADES array Yes — Pins the whole (grade, labelKey, tone, hotkey) tuple. Callers pass one of the four canonical entries; ad-hoc specs are a type-error.
label string Yes — Resolved i18n string (parent calls t(spec.labelKey)). Kept out of the component so it can be story-rendered without the i18n bootstrap.
onPress (grade: FocusGrade) => void Yes — Fired with the spec's grade (not with the click event). Parent wires it to useFocusSession.grade.
disabled boolean No false Focus screen disables all four while in prompt (answer hidden) and grading (mutation in flight).
showHotkey boolean No false Renders the mono hotkey chip. Parent gates on hover: hover so mobile stays clean.

Design tokens

  • Tone inks. Each button is tinted by --grade-ink which flips with data-tone: danger → --tma-tone-danger, warn → --tma-tone-warn, accent → --tma-tone-accent, success → --tma-tone-success. Body is color-mix(in oklab, var(--grade-ink) 15%, transparent); border deepens to 35%; hover to 28% body / 55% border.
  • Motion. Press animates transform: scale(0.97) over --tma-dur-fast. Reduced-motion suppresses the transform.
  • Density. Minimum tap-target 56 px height (over the kit's 44 px baseline — grade press is the primary action, treat it accordingly).
  • Hotkey chip. Only rendered under @media (hover: hover). Uses --tma-font-mono at --tma-fs-micro.

Accessibility

  • Semantic button. Plain <button type="button"> — keyboard focus, screen reader announcement, Enter/Space activation all come from the platform.
  • aria-keyshortcuts. Every button advertises its hotkey via aria-keyshortcuts so screen-reader users know Space + 1-4 are available on desktop.
  • Disabled state. Uses native disabled, not a style hack — buttons are removed from the tab order when in prompt / grading phases.
  • Contrast. Each tone body sits at 15% opacity on --tma-surface-canvas; button label reads at the full tone ink. Verified AA against every dark-mode canvas.

Used on

  • Screen · Focus — grade strip (region 5). Four instances, one per grade.

Tests

GradeButton is covered indirectly through useFocusSession.test.ts (state-machine assertions + grade→tag mapping) and the five Storybook stories that pixel-diff each tone + the disabled variant. Consider adding a direct render test if we ever tweak the aria contract.
