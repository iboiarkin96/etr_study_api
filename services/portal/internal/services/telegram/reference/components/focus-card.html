Component spec: FocusCard

The tap-to-reveal 3D flashcard on the Focus review flow. Two faces — front (title + «Tap to reveal») and back (title + dense paragraph + bullets). Renders as a <motion.div role="button"> — not a real <button>, because block content inside a button is spec-illegal and Safari / Firefox reparent invalid DOM in ways that break the 3D-preserve layout the flip depends on.

Overview

Tap the body → parent's onReveal fires; the card flips 180° over 550 ms; two faces share one grid cell, back is pre-rotated 180° so it reads correctly at the flipped position; paragraph + bullets stagger in half-way through the flip. The parent's useFocusSession guards the toggle against phase='grading', so a stray tap on a card the user just graded can't cancel the commit.

Reduced-motion branch. Under prefers-reduced-motion the container stops rotating; the component conditionally renders front OR back instead of both. Without this branch the back face's own rotateY(180°) keeps it facing away from a container that never rotates — reduced-motion users would toggle revealed forever but never see the answer.

Accessibility

  • role="button" + tabIndex=0. The tap surface is keyboard-focusable; Enter and Space fire onReveal explicitly.
  • Reduced motion honoured. The reveal doesn't rotate — the visible face swaps instead, so AT users never get stuck on a back face they can't see.
  • Live regions for the back copy. Dense paragraph + bullets are inside a semantic <article>; AT re-reads them when the reveal fires.
