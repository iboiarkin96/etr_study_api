Component: ErrorScreen

The full-screen error state for blocking failures — auth handshake refused, server unreachable, anything that stops the app from proceeding. Occupies the whole viewport, centres a broken-orb illustration + title + body + primary retry button, and reads as a calm signal rather than a stack-trace dump.

Overview

When a hook errors out on a data block, the user still sees the rest of the screen and the block itself gets an ErrorInline plate — that's a partial failure. When the whole app can't proceed (auth failed, backend gone, no JWT to work with), the calm-but-clear thing is a total error surface — a full-viewport screen with one job: get the user to retry.

ErrorScreen is that surface. It's rendered instead of the normal screen output — the Today / detail entry points return an ErrorScreen-wrapped <main> and short-circuit before rendering their real content.

Why the broken-orb SVG

The signature visual on Today is the living orb. When the app breaks, the orb doesn't render — the broken-orb SVG on this screen is its absence made visible: a dashed grey ring where the orb should be, plus a small ember-tinted dot floating off-axis. It says «something signature is broken» without a red exclamation mark or a sad face.

All motion is optional — the SVG is static in v1. Adding a slow rotation on the dashed ring is a natural follow-up if the visual feels too still.

Anatomy

  • Root. <div role="alert" aria-live="assertive"> — flex-centred column, min-height var(--tma-viewport-h), padding var(--tma-sp-8) var(--tma-sp-6). Not focusable itself; the retry button carries the primary action.
  • Broken orb. <svg> 120×120 · aria-hidden · three shapes: soft ember radial glow, dashed grey ring (rotated -15°), ember-500 dot at (76, 52) with an outlined halo. Grey ring is --tma-border-bold, dot is --tma-ember-500.
  • Title. <h1> — --tma-fs-h2, --tma-fw-heavy, --tma-lh-tight, letter-spacing -0.02em. Copy comes in via props.
  • Body. Optional <p> — --tma-fs-body, --tma-text-tertiary, max-width 300 px so it wraps to a readable line-length.
  • Retry CTA. <button class="tma-btn tma-btn--primary tma-btn--block"> capped at max-width 240 px so the button reads as a decisive action, not a page-wide bar. Label comes in via props; on-click calls the retry handler the parent supplies.

Props

Prop Type Required Notes
title string required H1 copy. Short, human, no jargon — e.g. «Server unreachable», not «HTTP 502».
body string (optional) — One-sentence explanation + hint. Falls through to no paragraph when omitted; the visual sits closer to the CTA in that case.
ctaLabel string required Button copy. Usually «Retry» / «Try again» / «Повторить».
onRetry () => void required Handler for the retry click. Consumers call auth.retry(), query.refetch(), or their own state-reset — ErrorScreen stays dumb.

Usage — auth-gate short-circuit pattern

Both consumers share the same shape: they early-return an ErrorScreen-wrapped <main> when auth.status === 'error', instead of trying to render the screen with an empty auth.user. The i18n namespace is picked from the exception message via a small helper.

function authErrorNs():
  'auth.unreachable' | 'auth.denied' | 'auth.error' {
  const msg = auth.error?.message ?? '';
  if (/failed to fetch|networkerror|fetch failed/i.test(msg)) return 'auth.unreachable';
  if (/401|unauthori[sz]ed|token|jwt|signature/i.test(msg))    return 'auth.denied';
  return 'auth.error';
}

if (auth.status === 'error') {
  const ns = authErrorNs();
  return (
    <main className="tma-scope">
      <ErrorScreen
        title={t(`${ns}.title`)}
        body={t(`${ns}.body`)}
        ctaLabel={t(`${ns}.cta`)}
        onRetry={() => auth.retry()}
      />
    </main>
  );
}

The .tma-scope wrapper stays on <main> so Ember tier-2 tokens resolve inside the error screen too — the button, the H1 and the SVG all read tokens off that scope.

Copy · three failure modes

The i18n keys under auth.* are split by the underlying failure so the copy matches the actual cause. All three carry a title, body, and CTA.

Namespace Trigger EN title · body · CTA RU title · body · CTA
auth.unreachable fetch() threw (server down, DNS out, network dropped) «Server unreachable» · «The server isn't responding. Check your connection and retry — everything should come back on its own.» · «Retry» «Сервер не отвечает» · «Проверь подключение и повтори — обычно всё возвращается само.» · «Повторить»
auth.denied 401 / bad signature / expired token «Authorization failed» · «The Telegram handshake was refused. Try again; if it keeps happening, reopen the app from the bot.» · «Try again» «Ошибка авторизации» · «Telegram-handshake не прошёл. Попробуй снова; если не помогает — перезапусти приложение из бота.» · «Повторить»
auth.error Anything else — unexpected exception, malformed response «Something went wrong» · «We couldn't sign you in. Please try again.» · «Try again» «Что-то пошло не так» · «Не удалось войти. Попробуй ещё раз.» · «Повторить»

Design tokens

  • Spacing. --tma-sp-8 block padding (32 px vertical breathing room), --tma-sp-6 horizontal, --tma-sp-4 gap between elements, --tma-sp-2 extra above the CTA.
  • Typography. Title uses --tma-fs-h2 (28 px) · --tma-fw-heavy (800) · --tma-lh-tight. Body uses --tma-fs-body (15 px) · --tma-lh-normal · --tma-text-tertiary.
  • Surfaces. Root inherits --tma-surface-canvas from the parent .tma-scope. Button uses --tma-action-primary / -hover / -press.
  • Ember highlights. The floating dot on the broken orb is --tma-ember-500; the soft radial glow behind it is a radial-gradient from --tma-ember-400 at 32 % opacity.
  • Neutral wireframe. Dashed outer ring is --tma-border-bold, so it reads as «missing content» rather than «error red». No --tma-tone-danger in the visual — the copy carries the tone.
  • Motion. No CSS animations in v1. Adding a slow rotation on the dashed ring or a scale-in on mount are natural follow-ups; both should honour prefers-reduced-motion: reduce.

Accessibility

  • Role & live region. Root carries role="alert" + aria-live="assertive" so screen readers announce the title as soon as the component mounts.
  • Decorative visual. The <svg> is aria-hidden="true". All meaning is in the title + body + CTA.
  • Focus order. Only interactive element is the retry button; auto-focus is not applied so a screen reader user can read the title / body before being pushed onto the action. Consider adding autoFocus if the ergonomics need it.
  • Contrast. Title uses --tma-text-primary on --tma-surface-canvas — same combination as the rest of the app, AA passing across dark + light themes.
  • Reduced motion. No motion in v1 — nothing to gate.

Non-obvious cases

  • Body omitted. The paragraph collapses; the visual gets one --tma-sp-4 gap between title and CTA instead of two. Use when the title alone tells the whole story (rare).
  • Long titles. Copy over ~60 chars wraps naturally at the max-width. Very long titles look better if rewritten shorter — this is a UX hint, not a UI constraint.
  • Very small viewports. Padding scales with tokens, so on a 320×568 iPhone SE the visual still centres inside comfortable margins. Below that the SVG's fixed 120 px could overlap padding; there's no responsive scale in v1.
  • Retry that immediately fails again. The parent's retry counter isn't visible here — ErrorScreen just fires onRetry. If the same failure re-lands, the same error screen re-renders; parent can vary the copy or freeze the button on Nth attempt if that becomes a UX problem.

Tests

No dedicated test file yet — the component is pure props-in / DOM-out with no state. Tests worth adding in a follow-up: (a) all four title / body / ctaLabel / onRetry props threaded to the DOM, (b) onRetry fires on button click, (c) omitting body collapses the <p>.
