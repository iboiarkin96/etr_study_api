Component spec: SessionRecap

The end-of-session debrief ledger on the Focus complete state. One row per card the user marked Again or Hard this round; each row carries the ink of the grade that produced it and opens the shared MissSheet composer pre-linked to that conspectus — every miss of the session can be annotated without leaving Focus.

Overview

Before this component, the Focus completion screen offered a single quiet «Log a miss →» link that deep-linked to /errors?prefill_from=session carrying only the last missed card — a session with N misses could log exactly one, and doing so threw the user off the screen. SessionRecap inverts that: the misses come to the user as a ledger, and the composer comes to the ledger as a bottom-sheet. The screen's exit hierarchy is untouched — «Back to Today» stays the one hero action (D2), the ledger reads as content, not chrome.

The signature move is tone continuity: each row is tinted by the same ink as the grade button that produced it (danger = Again, warn = Hard), so the debrief visually quotes the grade strip. When a note is saved the row cools to success and locks — the session's embers go out one by one.

Props

PropTypeRequiredDefaultPurpose
misses readonly SessionMiss[] — { conspectus_uuid, title, grade: 'again' | 'hard' } Yes — The session's miss ledger, straight from useFocusSession.sessionMisses (grade order, one entry per card — the queue is a frozen snapshot, so uuids never repeat). Empty array → the component renders null.

Everything else is internal: the active-row selection, the logged-set, and the useCreateError mutation all live inside the component. The parent only supplies the ledger.

Row states

Good is recall (client-side distinction — server tag is still hard), Easy is obviously not a miss: neither enters the ledger. The logged-set is session-local state — a restart clears the ledger with the rest of the session.

Design tokens

  • Tone inks. Row tint rides --recap-ink, flipped by data-grade (again → --tma-tone-danger, hard → --tma-tone-warn) and by data-logged (→ --tma-tone-success). Plate is color-mix(in oklab, var(--recap-ink) 8%, var(--tma-surface-plate)), border 22%, hover 14%/40% — the same flat ink-tint register as the grade strip, no gradients.
  • Type. Eyebrow and grade label in --tma-font-mono at --tma-fs-micro, uppercase, wide tracking — the same voice as the N/M session counter. Title at --tma-fs-body semi, single-line ellipsis.
  • Motion. Hover deepens the tint over --tma-dur-fast; press scales to 0.99. Reduced-motion suppresses both.
  • Density. Rows are min 52 px tall — above the 44 px tap-target baseline.

Accessibility

  • Semantic list. A <section> labelled with the localized count («Session debrief — N cards to note») wrapping a <ul> of <button> rows — focus order, Enter/Space activation and SR announcement come from the platform.
  • Logged lock. Native disabled removes annotated rows from the tab order; the «✓ Noted» tail is part of the button's accessible name.
  • Escape containment. While the MissSheet is open it captures Esc at the window capture phase — the Focus screen's own Esc → exit-to-Today binding cannot fire underneath the modal.
  • Grade dot. Decorative (aria-hidden); the mono grade label carries the same information as text.

Used on

  • Screen · Focus — complete phase, between the end-state copy and the exit buttons.
