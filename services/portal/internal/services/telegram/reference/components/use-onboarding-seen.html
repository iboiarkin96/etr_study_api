Hook spec: useOnboardingSeen

Single source of truth for the Onboarding «has the user finished the flow?» flag. Reads and writes WebApp.CloudStorage['onboarding_done'] with a localStorage mirror so a warm reopen resolves synchronously and a fresh device syncs once from the cloud.

Overview

The hook returns a three-state machine that avoids the first-frame flash that a naive «cloud read on mount» would cause. The trick is a lazy initializer that reads the local mirror on render #1:

  1. On mount, the useState initializer reads window.localStorage['onboarding_done'] synchronously. If '1', initial state is 'seen' — the first render already knows, so OnboardingGate mounts already resolving to Today with no blank frame between navigations.
  2. Otherwise the initial state is 'unknown' and an effect fires the cloud read; while it's in flight OnboardingGate renders nothing so no wrong surface flashes.
  3. When the cloud read resolves, state flips to 'seen' or 'unseen'. If the initial state was already 'seen', the effect short-circuits and no cloud round-trip fires at all.

markSeen() is fire-and-forget: it writes the local mirror synchronously (flips the state on the next commit), then writes the cloud in the background. A dropped cloud write only re-shows the screen once on a fresh device — an acceptable failure mode that saves the surface from having to render a loading spinner on the write path.

Signature

type OnboardingSeenState = 'unknown' | 'unseen' | 'seen';

function useOnboardingSeen(): {
  state: OnboardingSeenState;
  markSeen: () => void;
};

State machine

StateWhenGate behaviour
'unknown'No local mirror at first render; cloud read in flight.OnboardingGate renders nothing. First frame is neutral.
'unseen'Cloud read resolved without a value and no local mirror.OnboardingGate effect-navigates to /onboarding.
'seen'Local mirror already '1' at first render (lazy init) or cloud read returned '1' or markSeen() was called this session.OnboardingGate renders Today inline.

Test coverage

  • Fresh user hydrates to 'unseen'. No local mirror, cloud returns null → state resolves after microtask.
  • Local mirror short-circuits to 'seen'. Cloud is not consulted; useful for the warm-reopen path.
  • Prior cloud value hydrates to 'seen'. A device that finished the flow elsewhere in Telegram syncs on first mount.
  • markSeen() flips state + writes both mirrors. Local write synchronous; cloud write recorded on the mock setItem log.
  • Missing CloudStorage shim → falls back to 'unseen' without crashing. Plain-browser dev (no Telegram host) still renders the flow.
