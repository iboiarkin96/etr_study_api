Component spec: OnboardingHero

The day-zero variant of the signature streak orb on the Onboarding screen. Wraps the shared .tma-orb primitive at digit «0» with the cap «day one» — the same emotional carrier the app will use every morning, introduced before the first review.

Overview

Purely presentational — no props, no data. The digit «0» and the «day one» cap are hard-coded so the surface never depends on a network read to look right (a real /me/stats query that returns «current_days: 0» is visually indistinguishable from a broken query, and the whole point of Onboarding is to look reassuring in second zero of the app). On Today the same .tma-orb primitive re-mounts against the live query key and the digit ticks up on the first review — the primitive is the same, only the source of the number changes.

Props

None. The component takes no configuration — a second variant means a second primitive, not a prop toggle.

Accessibility

  • Role. role="img" on the orb with the localized onboarding.eyebrow as its label («Day one» / «День первый»). Decorative pseudo-elements (.tma-orb__sheen, .tma-orb__glare) are aria-hidden.
  • Motion. Inherits the .tma-orb primitive's motion budget — the breathing animation respects prefers-reduced-motion. No motion of its own.
  • Contrast. The digit is rendered on ember ink; the cap on tertiary text token. Both pairs are AA on light and dark themes (verified against ember-500 + surface-plate contrast pairs).
