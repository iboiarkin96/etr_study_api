Component spec: HeatmapCalendar

GitHub-contribution-style 90-day heat-map — columns are weeks (Mon-first), rows are days-of-week (Mon top → Sun bottom). One cell per calendar day; density expressed as a 0..4 intensity bucketed server-side. Used on Today's pin 8 and as the slot-1 block on Schedule. Non-interactive: it's a visualization primitive, not a picker (tap-to-drill-down deferred).

Overview

The heat-map is a retention primitive — a habit is only a habit when the user can see the pattern of it. Days align to a weekly grid (Mon-first per most-of-the-world convention) with leading + trailing padding so the first week's Monday sits at the top and the last week's Sunday at the bottom. Month labels above the grid appear only where the month changes to avoid label-per-column clutter.

Intensity is computed server-side in me_service.py::_bucket_intensity (0 = none · 1 = 1–2 · 2 = 3–6 · 3 = 7–11 · 4 = 12+). The client never re-buckets — cell colour reads directly from data-level. Below the grid: a legend (less → more), then three text stats (Total · Active days · Best day) computed once from the same array via a simple reduce for screen-reader users (matches the visual grid but text-native).

Anatomy

  1. Section frame. <section class="tma-section" aria-labelledby="heatmap-h"> with the header carrying the tooltip. Same wrapping as every other Today block.
  2. Heat frame. .tma-heat-frame — layout container that owns --tma-heat-week-count so the CSS grid can size columns.
  3. Month row. .tma-heat__months — one .tma-heat__month label per month change.
  4. Grid. .tma-heat — grid-auto-flow: column. Cells push in chronological order, stacking top→bottom per week before advancing right.
  5. Cell. .tma-heat__cell[data-level] — one per real day. data-count + data-date render tooltip via CSS pseudo. Padding cells use visibility: hidden to preserve the grid slot without a background.
  6. Legend. .tma-heat__legend — «less» → «more» chip strip with five sample intensities.
  7. Stats. .tma-heat__stats — three tabular-numeric cells (Total · Active days · Best day) computed via .reduce().

Props

Prop Type Required Purpose
data HistoryDay[] — { date: string, count: number, intensity: 0..4 }[] Yes The window to render (server-side windowed to whatever days the caller requested). Cells assume calendar-day granularity, gap-less; the layout builder handles month boundaries and week padding.

Design tokens

  • Cell tint. color-mix(in oklab, var(--tma-ember-500) <pct>%, transparent) where pct scales with data-level (0 → 0 %, 1 → 22 %, 2 → 45 %, 3 → 70 %, 4 → 100 %). Level 0 shows a faint grid line via the container border; deep levels saturate to the ember accent — reinforcing the D4 «one accent» clause.
  • Today marker. Cells with data-today="true" get a subtle outline via the kit primitive, so the user can eye-scan their «right now» position without hunting.
  • Padding cells. visibility: hidden — reserves the grid slot without a background, so the first week doesn't shift when leading Mondays are missing.
  • Motion. None on the primitive itself. Entrance is owned by the consumer (Today wraps in Assemble slot 6; Schedule wraps in slot 1).

Accessibility

  • Role. The grid itself uses role="img" with a single aria-label summarising the 90-day total («Review activity over the last 90 days — 128 reviews in total»). A role="grid" would demand per-cell focus semantics the surface doesn't have, so the primitive chooses image + textual stats instead.
  • Cells hidden from AT. Every .tma-heat__cell is aria-hidden="true" — the visual grid isn't announced. Screen readers get the summary via the grid's aria-label AND the three stats below (Total · Active · Best day), which are announced normally.
  • Tooltip. Cell hover / long-press reveals date + count via the kit's CSS pseudo — sighted-user affordance only. AT users have the stats.
  • Colour independence. The stats block («128 reviews · 42 active days · best 12») carries the same information as the colour intensity, so users who can't perceive ember tint aren't blocked.

Behaviour contract

  • Server-computed only. No client aggregation, no re-bucketing — the primitive is a pure display layer over the HistoryDay[] shape. Density buckets change server-side; the client never lags.
  • Empty is calm, not error. Zero-review days render as level-0 cells (faint grid line, no ember). A fully-empty 90-day window looks like an empty grid, which the Schedule screen decorates with an «No history yet» copy card; Today's placement (below other blocks) doesn't need it — the block just reads calm.
  • Layout invariants. Mon-first week rhythm regardless of user locale; month labels align to the first week that contains the 1st of the month; today's cell always sits somewhere in the rightmost column.

Used on

  • Screen · Today — pin 8 (90-day heat-map grid) + pin 9 (stats footer).
  • Screen · Schedule — slot 1 (below the summary strip hero). Same 90-day window.
