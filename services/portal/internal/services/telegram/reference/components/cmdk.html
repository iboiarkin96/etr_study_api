Component spec: .tma-cmdk

The kit primitive behind every command-palette-style overlay in the TMA — a full-viewport dialog surface with a backdrop-blur tint, a centred box that rises softly on mount, and the selectors for the input row, list, items, keycaps and empty states. Flat design, reduced-motion aware, token-only. The SearchOverlay React shell is currently its only consumer.

Overview

The primitive is two surfaces stacked: a full-viewport backdrop (dark tint + backdrop-filter: blur()) and a centred box (fluid up to 560 × 460 px). Both animate on mount — the backdrop fades in over 160 ms, the box rises with a soft translate over 220 ms — and both use animation-fill-mode: both so the final state holds cleanly without the WAAPI opacity blink the epic already avoids elsewhere (see the Assemble spec for the same rationale in the Today choreography).

The primitive follows the TMA flat-design register — no gradients, no glows, no keycap embossing. Backdrop is a single tint; box is a solid surface with a hairline border; items switch tone on [aria-selected='true'] with a flat wash. Selectors are tone-agnostic — the React shell decides which item is «cursor» by flipping aria-selected.

Design tokens

The primitive reads only from the shared --tma-* token layer — no hard-coded pixels, colours, or timings inside cmdk.css. Any theming change ripples via the kit tokens.

States

  • Cursor. .tma-cmdk__item[aria-selected='true'] — flat tone wash, subtle border tint, ↵ keycap fades in on the trailing edge. No motion beyond the wash's opacity transition.
  • Match highlight. <mark> inside .tma-cmdk__item-title renders as a flat accent underline/wash — never the browser's default yellow. Inherits the row's own colour so the highlight stays legible on both cursor and non-cursor rows.
  • Empty vs list. .tma-cmdk__empty and .tma-cmdk__list are mutually exclusive by consumer contract — the primitive doesn't hide one from the other, the React shell renders exactly one.
  • Keycap idle vs active. The Esc keycap in the input row is always visible; the ↵ keycap on items is revealed only on the cursor row.

Motion

  • Backdrop fade — 160 ms, --tma-ease-out, animation-fill-mode: both. Runs once on mount; holds the final state cleanly (no WAAPI-style opacity blink on completion — same reasoning as the Assemble primitive).
  • Box rise — 220 ms, --tma-ease-out, animation-fill-mode: both. Small upward translate + fade so the box arrives from below rather than appearing in place.
  • Reduced motion. @media (prefers-reduced-motion: reduce) { animation: none } on both keyframe rules — the backdrop and box appear immediately in their final states. No jank, no delay.
  • Cursor / keycap transitions. Colour and opacity transitions on [aria-selected] and the trailing keycap use short (~120 ms) tweens on --tma-ease-out; they are considered decorative and also suppressed by the reduced-motion query.

Accessibility

  • Kit helper. .tma-visually-hidden is the kit's standard SR-only helper — position: absolute, clip-path: inset(50%), width/height: 1px. The overlay's role="status" live region depends on it; documented here so future consumers reach for the same helper.
  • Focus ring policy. The primitive suppresses the default focus ring on .tma-cmdk__input because keyboard focus stays on the dialog root and the cursor is announced via aria-activedescendant instead. If the primitive is ever reused in a context where the input is the actual focus target, the consumer must add its own ring.
  • Contrast. Ink tokens are AA-verified against --tma-surface-canvas in both themes; muted --tma-ink-tertiary is used only for supporting labels (sub, group title, hint copy), never for interactive text.
  • Reduced motion. Both mount keyframes and the decorative cursor transitions are suppressed under prefers-reduced-motion: reduce — the overlay still opens instantly and cleanly.

Do / Don't

Do Don't
Use kit tokens exclusively — the box, item, keycap, spacing and easing all resolve through --tma-*. Hard-code widths on .tma-cmdk__box. It's fluid up to a 560 px ceiling by design so the primitive works from a 320 px phone up to a wide desktop viewport.
Let aria-selected drive the cursor tone — the React shell owns cursor state, the primitive just paints it. Put primary actions inside the box beyond keycap chips (Esc / ↵). The overlay is a picker, not a form — buttons here would compete with the row cursor for the user's attention.
Reuse .tma-visually-hidden for any SR-only helper the consumer needs. It's the kit's canonical hidden-live-region helper. Introduce gradients, glows, or embossed keycaps. The primitive follows the TMA flat-design register — solids and hairlines only.
