Component spec: Assemble

The entrance-choreography wrapper for Today's blocks — the orb (hero) fades into place first, every other block flies in from below with a stagger set by its slot order. Together the seven blocks assemble in ~1 s, framing the D1 breathing element (orb) as the anchor that everything else settles around. Chosen over framer-motion for one specific reason: framer's WAAPI-driven opacity drops the element back to opacity: 0 for one frame on animation completion (visible blink); a CSS animation with animation-fill-mode: both holds the final state cleanly.

Overview

Cold-open on Today lasts about a second — enough time for a habit product to make a first impression. Rather than paint the whole screen at once, blocks arrange themselves: the orb settles into place, then Yesterday's digest floats up, then the summary strip, then «Start Focus», then the due list, then the recently-reviewed peek, then the heat-map. Every entrance is transform-only (opacity + Y-translate for stagger blocks, opacity + scale for the hero), so each block's layout space is reserved from mount — nothing pushes neighbours around as siblings finish animating in.

Two entrances, one primitive. The hero role belongs to exactly one block per screen (the orb on Today) and enters in place with a soft fade + scale so it never appears to «arrive» — it just resolves out of nothing where it always was. Every other block (the slotted role) flies up 32 px with a 70 ms stagger between slots, so the top-down assembly rhythm reinforces the D1 hierarchy without duplicating it in code.

Anatomy

  1. Root. A single <div> — no ARIA landmark, no interactive behaviour. The class flips between .tma-assemble (default: fly-up + stagger) and .tma-assemble--hero (fade + scale, no stagger).
  2. Order custom property. --assemble-order is written as an inline style from the order prop; consumed by the CSS animation-delay: calc(180ms + var(--assemble-order, 0) * 70ms). Hero variant ignores it (animation-delay is hard-set to 0 ms).
  3. Children. Whatever the caller passes — the wrapper is a pure «costume». It doesn't inspect or transform the subtree.

Props

Prop Type Required Default Purpose
hero boolean No false Marks this block as the screen's D1 breathing element (exactly one per screen). Enters first, in place, via a fade + soft scale over 420 ms; ignores order. When false, the block flies up from below.
order number No 0 Stagger position among the slotted (non-hero) blocks. Slot 1 = first below-orb block; slot 6 = last. Delay is 180 ms + order × 70 ms. Convention: pass a non-zero order whenever hero is false — the default 0 collides with the hero delay-line and reads as «orb second» in the rhythm.
children ReactNode Yes — The block to animate. The wrapper adds no padding, no margin, no landmark — pass whatever markup the block would normally render.

Loading & late-arrival contract

Each block wrapped in <Assemble> mounts only when its data resolves — see Today's {stats.data && …} guards. Because CSS animations fire on mount, blocks whose data arrives later animate later:

  • Cache-hit path (common). All hooks resolve in the same tick → every <Assemble> mounts in one render → the delay-line plays as designed (orb first, blocks 1–6 stacking up over the next 800 ms).
  • Cold-network path. If me/stats lands last, the orb assembles after the yesterday/summary/history blocks — the top-down rhythm inverts briefly. The OrbSlotPlaceholder in Today/index.tsx reserves the orb's footprint during stats.isPending so at least the layout doesn't jump; the visible ordering is still «other blocks first, orb last» in this rare case.
  • Skeleton → data swap. When a block toggles between skeleton and real data (e.g. the summary strip), the outer <Assemble order={2}> stays mounted — the animation fires once at first mount and does NOT replay on skeleton→data swap.

Design tokens

  • Easing. --tma-ease-out (both variants) — soft deceleration, no overshoot.
  • Duration. Hero 420 ms, slotted 380 ms. Hero is ~10 % slower to emphasise its role as the anchor.
  • Delay slope. 180 ms base + 70 ms per slot — the base gives the orb a head-start (it plays 420 ms alone before any slot fires), the slope gives each slot a legible «click» before the next starts.
  • Travel. 32 px upward — enough to read as a rise, small enough that no block appears from off-screen (all slots are at least partially visible before they animate).
  • Hero scale. 0.92 → 1 — a soft inflation, not a pop.
  • No colour, no shadow. All movement is opacity + transform; the primitive owns nothing that would fight the child's own styling.

Accessibility

  • Reduced motion. Under @media (prefers-reduced-motion: reduce), both animations are set to animation: none — the block appears immediately in its final state (opacity 1, no transform). No layout shift, no delay.
  • No landmark impact. The wrapper is a plain <div>; it doesn't add roles or aria attributes that would confuse assistive tech about the block underneath.
  • Focus order. Animations don't change tab order — the DOM position and default focus flow are identical to «no Assemble wrapper».
  • Screen reader. Content is announced as it enters the accessibility tree at mount. The 250–600 ms staggered delays don't delay announcement; SRs read content on DOM insertion, not on visual transition-complete.

Behaviour contract

  • Fire-once semantics. The CSS animation runs once at mount and holds the to state via animation-fill-mode: both. Re-renders of the child do not re-fire the animation.
  • Unmount. No exit animation. If a block conditionally unmounts (say the parent flips the query from data back to isPending), it just disappears; the next mount will animate again from the beginning.
  • Isolation. No position, no z-index, no isolation. The wrapper participates in the parent's flow like a plain block.
  • CSS-only. Deliberately not framer-motion — framer's WAAPI opacity blinks at completion (drops to opacity: 0 for one frame on animation-fill-mode: forwards semantics). CSS with both holds cleanly. Comment in assemble.css pins this rationale.

Used on

  • Screen · Today — 7 wrappers (orb as hero, then 6 slotted blocks) drive the cold-open choreography. See the screen-map's «Entrance choreography» section.

Focus does NOT use Assemble — the Focus card enters via framer-motion (initial/animate/exit) because it also needs an exit transition when advancing between cards, which Assemble intentionally doesn't offer.

Tests

Assemble is a thin pure-UI wrapper — no state, no callbacks, no branching beyond the hero flag. It's covered indirectly through the Today Storybook stories (screens-today--default) which render every wrapped block in place. A direct unit test would be tautological; consider adding one only if the primitive grows a third variant or takes a callback.
