Observability

Two transports, one thin façade. Sentry collects uncaught render / effect errors from AppErrorBoundary and the router's defaultErrorComponent; PostHog collects the fixed event vocabulary from Event spec. Both are env-guarded — dev laptops without VITE_SENTRY_DSN / VITE_POSTHOG_KEY ship zero network cost and log to the console instead.

Layout

Everything lives under src/shared/observability/. Four small files, one barrel:

FilePurpose
sentry.tsinitSentry() + captureError(err, {componentStack}). If VITE_SENTRY_DSN is unset, captureError falls through to console.error.
posthog.tsinitPostHog(), identify(distinctId, props), reset(), low-level track(name, props). All no-op when VITE_POSTHOG_KEY is unset.
events.tsTyped emitters — trackAppOpened, trackReviewCompleted, etc. TypeScript refuses to compile a call that spells a property wrong; a grep for the emitter name finds every callsite. This is the only entry-point call-sites should use.
index.tsBarrel — re-exports everything above plus initObservability() which brings both transports up in one call from main.tsx.

Environment variables

All optional — see Env vars for the full table. In short:

  • VITE_SENTRY_DSN — DSN for the Sentry project. Absent → Sentry disabled.
  • VITE_POSTHOG_KEY — PostHog project API key. Absent → PostHog disabled.
  • VITE_POSTHOG_HOST — ingest host, defaults to https://eu.i.posthog.com. Pick us.i.posthog.com for the US region.
  • VITE_APP_VERSION — Sentry release tag; CI sets it to the git short-SHA of the build commit.

Wiring — where transports hook into the app

SiteFileWhat it does
Bootmain.tsxinitObservability() runs before React mounts, so a crash inside the first render is captured too.
Auth handshakeauth-provider.tsxOn success, identifyUser(user.client_uuid, {locale}) aliases the anonymous PostHog session into the identified one. See Event spec § Identity — telegram_user_id / _username / first_name are NEVER sent.
Global error catchAppErrorBoundarycomponentDidCatch calls captureError(error, {componentStack}). Sentry stores componentStack as contexts.react.componentStack, visible in the issue's React panel.
Route-level catchrouter.tsxTanStack Router's defaultErrorComponent wraps each route in its own catch boundary. It renders the shared BoundaryFallback and also reports to captureError.
Session startApp.tsxtrackAppOpened fires once per mount inside useEffect — after WebApp.ready(), before bootstrap resolves. Attaches to the anonymous session; identify() later aliases it.

Events wired

The Event spec is the authoritative vocabulary. This section names which of those events fire from real callsites today; new callsites go in without ceremony as long as they use the typed emitters in events.ts.

EventCallsiteNotes
app_openedApp.tsxEvery cold mount. launch_source guessed from WebApp.initDataUnsafe.start_param + initData presence.
review_completeduseFocusSession.submitGrade onSuccessreveal_ms = Date.now() − revealed-at (null when the user never revealed, e.g. instant-grade via swipe).
focus_session_starteduseFocusSession seed effectFires once per epoch when a non-empty queue lands. Empty queues (no due cards) never fire — the user didn't do a session.
focus_session_endeduseFocusSession complete effectFires on the happy path (reason: 'completed'). 'exited' / 'backgrounded' reasons need screen-level cleanup + visibilitychange — deferred to a follow-up.
error_loggedErrors submit onSuccessfrom_screen: 'errors'. The Focus-side «Log a miss» path (session recap) will pass from_screen: 'focus' once wired.
search_openedSearchProvider open + Cmd+KTwo via values: cmd_k (keyboard shortcut), route (any api.open() caller — Today search button, deep-link).

Follow-ups (deferred): screen_viewed (needs TanStack Router beforeLoad hook), search_result_picked, streak_broken / streak_milestone (T-26c ships milestones), review_snoozed, command_ran.

Privacy — what we never send

  • No Telegram identifiers. telegram_user_id, telegram_username, first_name stay server-side. PostHog only ever sees the internal client_uuid as distinct_id.
  • No autocapture. PostHog's autocapture (arbitrary click/scroll events) is disabled — event-spec is a fixed vocabulary, ad-hoc events dilute the funnel.
  • No session recording. Disabled by default; needs a documented product need + user opt-in to enable.
  • Sentry PII off. Default SDK sends error.message + stack + componentStack; no user attributes are attached from the client. Server request-id headers do reach Sentry via API errors — that's the traceability we want, not a privacy leak.

Local development

Both transports are off unless you set the env keys in services/telegram/.env.local. The recommended local flow:

  1. Leave both keys unset — captureError logs to console.error, track() logs via console.debug in non-prod modes so you can eyeball the vocabulary without shipping anything to a cloud endpoint.
  2. To test the Sentry integration end-to-end, add VITE_SENTRY_DSN to .env.local, restart Vite, and force a boundary catch (see AppErrorBoundary § Testing).
  3. To test PostHog: add VITE_POSTHOG_KEY + optionally VITE_POSTHOG_HOST, restart Vite, and check the PostHog live-events view.
