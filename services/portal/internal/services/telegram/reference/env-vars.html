Env vars

Every variable the build reads. All are build-time — the Mini App ships a static bundle with values baked in. There is no runtime env in the client; nothing is read from process.env at runtime. Adding one that starts with anything but VITE_ is a Vite hard-fail — that's the rule this table follows.

Variables

Name Required Where read Default (dev) Purpose
VITE_API_BASE_URL Yes (prod) · optional (dev) src/shared/api/client.ts http://localhost:8000 Base URL for /api/v1 calls, including /auth/telegram. Overridden for on-device testing when the API sits behind a tunnel.
VITE_DEV_INIT_DATA Optional (dev only) src/main.tsx (unset — real Telegram) Pre-signed initData string produced by tools/dev/sign_init_data.py. Loads a synthetic Telegram user in the plain-browser dev loop. Vite refuses to include it in a prod build.
VITE_SENTRY_DSN Optional src/shared/sentry.ts (unset — Sentry disabled) DSN for the Sentry project. Absent → Sentry init skipped, no data leaves the browser.
VITE_POSTHOG_KEY Optional src/shared/analytics.ts (unset — PostHog disabled) Project API key for PostHog. Absent → analytics init skipped.
VITE_POSTHOG_HOST Optional src/shared/analytics.ts https://eu.i.posthog.com PostHog ingest host. EU by default — pick this or us.i.posthog.com to match your project region.
VITE_APP_VERSION Yes (CI) · optional (dev) src/shared/version.ts git short SHA Injected as Sentry release tag + shown in Profile footer. CI sets this from the git SHA of the build commit.
VITE_TG_APP_SHORT_NAME Yes (prod) src/shared/deep-links.ts study_app Bot's Mini App short-name as registered in BotFather. Used when the client builds share-links.

Conventions

  • Every var starts with VITE_. Vite exposes only these to the client bundle. This is a security boundary — anything you don't want in the bundle must not have this prefix.
  • No secrets in the bundle. Nothing under VITE_ may hold long-lived credentials. The bearer JWT is minted server-side after HMAC-verifying Telegram initData; the bot token stays exclusively in services/api/env/.
  • Server-side env stays in services/api/env/. This table is client-only. If it starts with VITE_, it lives here; if not, it lives in the API service.
  • Missing required var fails the build. A required var absent at vite build time causes an explicit error message; no silent fallbacks that ship broken clients.

Where each layer reads env

Environment Source of values
Dev (local)services/telegram/.env.local (git-ignored). Falls back to defaults for optional vars.
CI (verify)Repo secrets injected as env vars into the runner. Only required vars set — optional vars stay unset to prove the app boots without them.
Prod (Cloudflare Pages)Cloudflare Pages project settings → Environment Variables → Production. Values baked into the bundle at build time.
