Deep-link grammar

Formal grammar for the startapp payload the bot sends. The router at src/app/router.tsx enforces this table verbatim — the design page at Information architecture · Deep-link grammar is the design-narrative version; this is the machine-parseable one.

URL shape

Telegram bots produce deep-links using the modern HTTPS format only. Legacy tg:// schemes are not supported.

https://t.me/<bot_username>/<app_short_name>?startapp=<payload>

Telegram passes <payload> to the WebApp via WebApp.initDataUnsafe.start_param. The router reads it once on mount, dispatches, and never re-reads (guards against navigation loops).

Grammar (EBNF)

payload   = empty | intent
empty     = ""
intent    = navigate | action | error
navigate  = "conspectus_" uuid
          | "schedule_" date
          | "focus"
action    = "errors_new"
uuid      = 32 hex chars, no dashes (compact form)
date      = YYYY_MM_DD (four-two-two digits, underscores; router parses back into an ISO date)
error     = any other string  →  dispatched to "/" with payload logged

Compact UUIDs. Payloads are size-constrained by Telegram (~64 chars). We strip the four dashes from a UUID (36 → 32) and normalise back inside the router.

Payload table

Payload Regex Route dispatched to Emitted by
(empty)^$/ (Today)Bot launch button, default.
conspectus_<uuid>^conspectus_[0-9a-f]{32}$/conspectus/:uuid (dashes re-inserted)Bot per-conspectus notifications.
schedule_<YYYY_MM_DD>^schedule_\d{4}_\d{2}_\d{2}$/schedule?day=<YYYY-MM-DD> (router re-inserts dashes for the ISO route param)Bot pushes a heat-map digest for a day.

anything else(fallthrough)/; payload logged via console.warn; PostHog deep_link_unknown—

Design properties

  • Opaque to users. Payloads never appear in copy, never in error messages, never in URLs the user sees or types.
  • One-way. The client parses payloads but never generates them for other links. Sharing between Telegram accounts is a P2 feature.
  • Idempotent dispatch. Same payload → same route, always. No time-based or user-based dispatch.
  • Safe fallback. A malformed payload never blocks the launch. Worst case: user lands on Today.
  • All-lowercase + underscore. No hyphens (breaks Telegram's parser in edge cases), no dots, no percent-encoding.

Examples

# Push a specific card
https://t.me/study_app_bot/study_app?startapp=conspectus_ab12cd34ef567890abcdef0123456789

# Push today's schedule
https://t.me/study_app_bot/study_app?startapp=schedule_2026_07_15

# Quick-action: start review
https://t.me/study_app_bot/study_app?startapp=focus

# Default launch (no payload)
https://t.me/study_app_bot/study_app
