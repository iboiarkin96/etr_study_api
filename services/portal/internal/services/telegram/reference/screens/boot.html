Screen map: Boot

The warm, on-brand splash rendered while the initial /api/v1/auth/telegram handshake is in flight. Sibling of the Auth error screen — same 120×120 ember-glow canvas, same accent tokens; the only difference is that this one animates the dot around a continuous ring while the error state freezes it off-axis. Owned by AuthGate so no data screen has to spell out «auth still loading».

Anatomy

RegionElementBehaviour
Root <main class="tma-scope tma-boot" role="status" aria-live="polite" aria-busy="true"> Full viewport (--tma-viewport-h), canvas surface, safe-area padding, centred grid.
Fade wrapper .tma-boot__inner Column stack — SVG + title + hint. Opacity 0 for 200 ms, then a 260 ms ease-out fade with a 4 px lift. Skipped under prefers-reduced-motion.
Ember glow <circle r="55" fill="url(#boot-orb-glow)"> Radial gradient identical to ErrorScreen's — --tma-ember-400 at 32 % → transparent. Static; provides the warm halo behind the ring.
Orbit ring <circle r="42" stroke="var(--tma-border-bold)"> Continuous 1.5 px ring at 55 % opacity. Where ErrorScreen's ring is dashed («disconnection»), this one is solid — the dot travels around it.
Rotating arm <g class="tma-boot__orbit-arm"> Group whose transform-origin sits at the ring centre. CSS rotates it 360° per 1.6 s, linear timing — a steady heartbeat, not a bounce. Contains a short trailing arc (16 % of the ring, ember-500 at 55 %) sitting behind the dot so the direction of travel reads at a glance.
Ember dot <circle cx="60" cy="18" r="6" fill="var(--tma-ember-500)"> Filled ember dot at 12 o'clock inside the rotating arm, with a soft 10 px accent halo around it — the same signature ember pair ErrorScreen uses off-axis.
Tagline .tma-boot__title · i18n boot.tagline «Warming up your day» / «Разогреваем твой день». --tma-fs-h2, heavy weight, tight leading, primary ink — same tier as ErrorScreen's title.
Hint .tma-boot__hint · i18n boot.hint «Just a moment while your app wakes up.» / equivalent — body font, tertiary ink, capped at 300 px so it wraps to two lines on narrow phones.

States

  • Normal. Auth in flight → orb breathes, fade-in lands at 200 ms.
  • Fast auth (~150 ms). The fade-in delay outlives the handshake, so the user never sees a flashed splash frame — <AuthGate> swaps to the app tree before the wrapper becomes visible.
  • Slow auth (> 3 s). Orb continues breathing indefinitely; no timeout on this screen. A hung bootstrapAuth() is caught elsewhere (CloudStorage withTimeout(3s), fetch native timeout). Auth failure switches AuthGate to the classified ErrorScreen.
  • Reduced motion. Both keyframe animations collapse to their initial visible state — orb is a static disc, wrapper is opacity: 1 from the first frame.

Accessibility

  • Announced. role="status" aria-live="polite" aria-busy="true" — screen readers read «Warming up your day» when boot starts and silence when it succeeds.
  • Decorative orb. aria-hidden="true" on the orb node — nothing meaningful to announce; the tagline carries the semantics.
  • Not focus-trapping. No interactive controls. Nothing to tab into. First interactive control appears after auth resolves (Today's search icon).
  • Motion budget. prefers-reduced-motion: reduce disables the pulse + the ring loop + the wrapper fade.
