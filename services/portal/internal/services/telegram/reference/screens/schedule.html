Screen map: Schedule

The look-ahead surface — a standalone /schedule route that shows the same summary strip and 90-day heat-map Today already renders, but as a chrome-full dedicated screen. Zero client aggregation: every number comes straight from GET /api/v1/schedule/{summary,history}. Sibling of the design narrative; this is the machine-readable anatomy.

Purpose

Today gives users their day; Schedule gives them their rhythm. The 90-day heat-map is the DAU-30 retention signal — a habit is only a habit when the user can see the pattern of it. The summary strip up top answers «how much work is queued right now / next 24 h / total», so a returning user can decide whether to Focus now or come back later. Every value is server-computed; the client is a pure display layer per the epic's «no client aggregation» rule.

This screen intentionally reuses Today's two hero data blocks — same tokens, same layout, same primitives. The only new-code contribution of T-19 is the wrapper: route registration, header, and the screen-level empty state. If either block ever grows a Schedule-specific variant (e.g. tap-to-drill-down day detail), it moves to screens/Schedule/components/.

Accessibility

  • Landmark. Root <main> with data-density="regular". Header uses a plain <h1> for the screen title.
  • Back button. Explicit aria-label={t('schedule.back')} so the ‹ chevron reads as «Back to Today» to screen readers.
  • Skeletons. Each block's skeleton carries an aria-label so SR users get a loading announcement rather than silence.
  • Heat-map a11y. Delegated to the HeatmapCalendar spec — grid layout with per-cell tooltips + aria-label summarising the 90-day total.
  • Reduced motion. Assemble collapses to instant; block-level animations respect prefers-reduced-motion: reduce.
