Screen map: Auth error

Full-viewport blocking-error state — shown instead of the normal screen output when the auth handshake fails (server unreachable, 401, unexpected exception). Both Today and Conspectus detail short-circuit to this surface when auth.status === 'error'. This map annotates every visible region with the source and copy that feeds it.

Purpose

When one hook errors out on Today or Detail, the corresponding block gets an inline ErrorInline plate — a partial failure the user can keep working around. When the whole app can't proceed (auth handshake didn't produce a JWT), the calm-but-clear response is a full screen with one job: get the user to retry.

Three copy variants are keyed off the exception message from bootstrapAuth() — server unreachable (fetch threw), authorization denied (401 / bad token), or something else entirely. All three land on the same visual; only the title / body / CTA copy changes.

The map

Numbered pins run down the phone's right edge. Click any pin to reveal its callout in the sticky panel on the right. Seed data mirrors the auth.unreachable variant (the one you hit with docker stop study-app-api).

Copy variants — three failure modes

The visual is identical; only the copy shifts. authErrorNs() in the parent screen picks the namespace by matching auth.error?.message.

Data source

No data hook of its own — the screen displays the outcome of a failed auth flow. The parent's auth context provides the error and the retry handle.

SourceWhat it providesWired to
useAuth() · auth.status === 'error' Gate — when this is true, the screen renders instead of the normal Today / ConspectusDetail output. Parent early-return branch.
auth.error?.message Raw exception message from bootstrapAuth(). Two regexes classify it into unreachable / denied / generic error. authErrorNs() helper in each parent.
auth.retry() Bumps the auth-provider's retry tick so its useEffect re-runs bootstrapAuth(). onRetry prop.
i18n · auth.*.title / body / cta All three text slots, per locale, per failure mode. Threaded into the title / body / ctaLabel props.

Interactions

  • Retry button. Only click target on the screen. Calls auth.retry(). If bootstrapAuth() succeeds, auth.status flips to 'authenticated', the parent's early-return unwinds and the real screen paints.
  • Same failure re-lands. Retry counter isn't visible — ErrorScreen just re-renders with the same copy. If this becomes a UX issue we'll add exponential back-off or a «reopen from the bot» escalation after N attempts.
  • Router navigation. There's no <Link> on this screen. The user's escape hatches are the Telegram back-button (real client) or the browser tab (dev). Both leave the screen; neither triggers retry.
  • LangSwitch not shown. Language is already committed on mount by the AuthProvider effect — a locale flip in the middle of a failed handshake wouldn't fix anything. The copy just re-renders in the current locale.

Accessibility

  • Live region. Root carries role="alert" + aria-live="assertive" so screen readers announce the title on mount.
  • Decorative visual. The <svg> is aria-hidden="true". All meaning is in the title + body + CTA.
  • Focus. No auto-focus in v1 — a screen-reader user reads the title / body before being pushed onto the button. Add autoFocus on the retry button if consumer testing shows the current default confuses reachability.
  • Contrast. Title uses --tma-text-primary on --tma-surface-canvas, body uses --tma-text-tertiary. Both AA in dark + light themes.
  • Reduced motion. No animation in v1 — nothing to gate. If we add a slow ring rotation on the SVG, gate it with @media (prefers-reduced-motion: reduce) { … animation: none }.
