Screen map: Encode

The conspectus authoring surface at /encode. One screen with one job: turn a fresh thought into a stored ETR note — three phases (Encode → Trigger → Recall) mapped 1:1 onto the POST /api/v1/conspectuses contract. The three-dot progress ribbon breathes when all phases are filled; the Telegram MainButton unlocks under the same condition so an incomplete draft can never leave the client. Sibling of the ADR 0038 · Screen DNA; this is the machine-readable anatomy.

Anatomy — the three ETR phases

Each phase renders as one <section class="tma-composer__phase"> block with an eyebrow (01 · Encode, 02 · Trigger, 03 · Recall), a prompt, a hint, and its input. The progress ribbon at the top tracks which phases are «filled» — the ribbon reads partial until all three are, then flips to ready with a single breathing pulse.

PhaseUIServer fieldRequiredLimits
01 · Encode Optional title chip + auto-growing <textarea> with live counter (len / 4000, warn state at 90 %) dense_paragraph, optional title Yes (dense_paragraph) title ≤ 256 chars; dense_paragraph 1..4000 chars
02 · Trigger Term chip input (Enter / comma to commit; Backspace on empty entry removes the last chip) + auto-growing question list cue_sheet.terms[], cue_sheet.questions[] No — the phase is optional. The ribbon shows a warm dot at index 2 only if at least one term or non-empty question is present. ≤ 20 terms (each ≤ 64 chars) · ≤ 20 questions (each ≤ 500 chars)
03 · Recall Inline bullet list, ghost «+ Add bullet» button. First bullet is pre-seeded (empty) so the block reads as a promise rather than a bare add button. Enforced-minimum is a validation step on Save, not on typing. bullets[] Yes (≥ 1 non-empty bullet) 1..20 bullets, each 1..500 chars

Chrome — native buttons + in-page fallback

  • Telegram BackButton (T-25d): navigates to / (Today). Bound via useTelegramBackButton.
  • Telegram MainButton: labeled «Save encode»; visible only when isDraftSubmittable(draft) && !create.isPending. In-flight, the label swaps to «Saving…». The unlock condition is the reason the button is bound with useTelegramMainButton(canSubmit ? {…} : null) — hiding the button until the required fields are in prevents a half-formed conspectus from reaching the server.
  • Fallback in-page CTA: for non-Telegram surfaces (Storybook, dev tunnels in a plain browser) the screen paints a sticky <button class="tma-btn tma-btn--primary tma-btn--block"> at the bottom, gated by useIsTelegramClient() === false. Inside real Telegram this button is not rendered — the MainButton is the only submit path.

Mutation — useCreateConspectus

Owned by src/screens/Encode/hooks/useCreateConspectus.ts. Wraps POST /api/v1/conspectuses as a useMutation. Server-authoritative — no optimistic update, because the created row needs a server-assigned conspectus_uuid + initial ConspectusSchedule before it's usable anywhere else.

  • Idempotency-Key: auto-injected by the API-client middleware (T-12). A network retry after a lost reply can never create a duplicate row (ADR 0006).
  • Cache invalidation on success: conspectuses.list (Search overlay) + today.conspectuses.due (Today card list). Both invalidations are scoped to the current user via clientUuid.
  • Owner params: telegramOwnerParams(telegram_user_id) — the API expects a system_user_id + system_uuid pair for legacy owner-column compatibility; identity-mapping stays in shared/auth/identity.ts.

States

Empty

Fresh mount. All three phases empty, ribbon shows three grey dots, label reads encode → trigger → recall. MainButton hidden (draft not submittable). See Storybook Screens/Encode/Empty.

Required only

Encode + Recall filled, Trigger empty. Ribbon shows two warm dots + one grey. MainButton unlocked. See Storybook Screens/Encode/RequiredOnly.

All three phases

Every phase filled. Ribbon flips to data-state="complete", plays a single breathing pulse, label reads «All three · ready to save». MainButton reads «Save encode». See Storybook Screens/Encode/AllThreePhases.

Saving

Mutation in flight. MainButton label swaps to «Saving…»; the fallback in-page CTA disables. On success: notifySuccess haptic + toast + navigate to /. On error: notifyError haptic + error toast, draft preserved for retry.
