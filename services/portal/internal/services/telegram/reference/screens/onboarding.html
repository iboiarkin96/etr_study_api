Screen map: Onboarding

The first-run flow at /onboarding — Screen 08 of ADR 0038. Day-zero StreakOrb hero, three E · T · R moves as the editorial peek, one warm CTA lands the user on Today with the first due card visible. Dismissed once and forever — the flag lives client-side in WebApp.CloudStorage so there is no server round-trip on this surface.

Regions

Four stacked regions in Assemble order. Every region names the React component, the data source, and the CSS primitive.

Region Rendered by Data source CSS
1 · Day-zero orb (hero) OnboardingHero — wraps the shared .tma-orb primitive at digit «0», cap «day one» None. The digit is hard-coded — see ADR 0038 Screen 08 «static hero» deviation. Hooks & endpoints

Hook Storage Notes
useOnboardingSeen WebApp.CloudStorage['onboarding_done'] + window.localStorage['onboarding_done'] Three-state hydration: 'unknown' → 'unseen' or 'seen'. The local mirror wins the race on warm reopens so the surface never flashes; the cloud write is fire-and-forget, so a dropped write only re-shows the screen once on a fresh device.
OnboardingGate Consumes useOnboardingSeen Wraps Today at /. Renders nothing while 'unknown'; effect-navigates to /onboarding when 'unseen'; renders Today inline when 'seen'. Deep links to other routes bypass the gate (they carry their own intent).

There is no server contract. Analytics on «finish vs skip» are deferred until product asks for them; the current model treats both taps identically (flag flips, navigate to Today) so the surface behaves the same no matter which affordance the user takes.
