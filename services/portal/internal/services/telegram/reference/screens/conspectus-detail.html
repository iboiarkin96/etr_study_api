Screen map: Conspectus detail

Read-only view of one ETR note at /conspectus/$conspectus_uuid. Tap into it from any Today due card or peek row; back-link returns to Today. This map annotates every visible region with the React component that draws it, the hook + endpoint that feeds it, and the CSS primitive that styles it.

Purpose

Detail is the drill-down surface for every card the learner touches on Today. Two entry points: tap a row in DueCardsList (due now) or a row in RecentlyReviewedPeek (recently touched). Both funnel here via a <Link to="/conspectus/$conspectus_uuid">. Reviewing happens in the Focus flow — this screen is read-only.

Deep-linkable: refresh on the URL loads the note directly; router preserves back-navigation into Today.

The map

Numbered pins run down the phone's right edge — an arrow leads from a pin to its block when the block sits deeper in the layout. Click any pin to reveal its callout in the sticky panel on the right. Seed data mirrors dev-seed-a1 from seed_dev_data.py — the «Индексы в PostgreSQL» note.

Alternate states

Three variants around the main success render. Auth-in-flight and auth-error states are handled globally by AuthGate, not inline on this screen.

Auth gate

Never a state of this screen — AuthGate owns it. During the handshake it mounts the BootScreen splash instead of Detail; on failure it mounts the Auth error surface. Detail assumes auth.status === 'authenticated' by the time it renders.

Loading

Local <DetailSkeleton> (bottom of ConspectusDetail/index.tsx). Draws a 32-px title block, a short slot-chip line, and three body lines at descending width — matches the height of the real content so mount → data landing doesn't shift layout.

Error

Rendered when useConspectus(uuid).isError === true. Inline plate with --tma-tone-danger ink and the i18n key detail.error. No retry button in T-17 — the user's fallback is back-navigation. Future pass: swap for the shared ErrorInline primitive.

Data source

One hook drives the whole screen. Compared to Today's five parallel hooks, Detail is a single-endpoint read.

HookEndpointGuardNotes
useConspectus(uuid) GET /api/v1/conspectuses/{conspectus_uuid} auth.status === 'authenticated' && !!auth.user && !!conspectusUuid Returns the full ConspectusResponse — title / slot / cue_sheet / dense_paragraph / bullets / content_version. Reactivates on uuid or client_uuid change. Composite-key path param comes from URL; system_user_id + system_uuid from telegramOwnerParams().

Interactions

  • Back link (top-left). <Link to="/"> — returns to Today, router-cached. No re-fetch of Today's five hooks; state (open panels, scroll) survives.
  • Browser back / gesture back. Same route change — TanStack Router treats it identically.
  • Deep link. startapp=conspectus_<uuid> (T-28) lands here without going through Today first. Router opens the URL directly.
  • Review now. Navigates to /focus?conspectus_uuid=<uuid> — Focus opens with a queue of exactly this card (T-17c). Batch review still starts from Today's «Start Focus»; both entry points share useReviewConspectus, so grading here drops the row from Today's due-list cache the same way.
  • LangSwitch. Same behaviour as on Today — i18n flip + localStorage pin.

Accessibility

  • Landmark structure. One <main class="tma-scope">, one <article> wrapping the note, semantic <h1> for the title, <section> for bullets with aria-labelledby="bullets-h".
  • Back link. The ← symbol is inside the anchor's text so screen readers announce it. i18n text is «Back» / «Назад».
  • Focus order. Header buttons → title (not focusable) → paragraph (not focusable) → bullets (rows are not links in T-17, so not focusable either) → Review-now button (T-17c makes it a real tab-stop and the primary action for keyboard users on this screen).
  • State messaging. Loading skeleton is decorative (aria-hidden="true" would be appropriate — currently just visual). Error plate is a plain <div> — should get role="alert" in the follow-up that unifies error styling with the shared ErrorInline primitive.
  • Reduced motion. Screen has no motion of its own; the back-link hover uses a token transition (short, non-essential).
