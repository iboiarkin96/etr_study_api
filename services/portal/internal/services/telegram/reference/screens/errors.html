Screen map: Errors

Append-only journal of retrieval misses at /errors — the ETR methodology's memory of «where I fell down». MissOrb shows the weekly count as the D1 breathing element, three-tone rows list the misses newest-first, and one warm CTA lets the user record today's. Data is one GET /api/v1/errors away; writes are idempotent via the auto-injected header. Sibling of the ADR 0038 · Screen 05 card; this is the machine-readable anatomy.

Purpose

Reviews tell the app what the learner knows; misses tell the learner what they don't. Deletion is deliberately absent: the log is append-only. Three misses in a week clustered around one topic means «reset the schedule on that cluster» — the shape is the signal. Every value on this screen is server-computed or a simple client filter over the returned list; there is no analytics chrome, no chart. The point is not to feel bad about the miss — it's to notice the shape of the misses.

The MissOrb reuses the shared .tma-orb primitive — same visual as Today's StreakOrb, only the state machine differs (warm ↔ rested on count == 0, no celebrate milestones). The Sheet composer wraps the entire viewport in a scrim + slide-up panel so nothing above it is tappable during a save; this pattern will be reused by future add-note surfaces (Search's «create», Onboarding's «name it»).

Hooks & endpoints

Region Hook Endpoint Query key
4 · Miss loguseErrorsGET /api/v1/errors['errors.list', client_uuid]
6 · MissSheet SaveuseCreateErrorPOST /api/v1/errorsMutates ['errors.list', client_uuid] optimistically

The read hook is enabled: false until useAuth().status === 'authenticated'. The write hook is optimistic: a temp row is prepended immediately with error_uuid: 'tmp-…', then replaced with the server row on success or rolled back on error. Idempotency-Key is auto-injected by the client middleware (T-12); a lost 4G reply on retry never doubles the miss (ADR 0006).

Accessibility

  • Landmark. Root <main> with data-density="regular". Header uses a plain <h1> for the screen title.
  • Back button. Explicit aria-label={t('errors.back')} so the ‹ chevron reads as «Back to Today» to screen readers.
  • Add button. Explicit aria-label={t('errors.add')} — the ＋ glyph alone would announce as «plus sign».
  • Orb. role="img" + pluralised errors.orb.aria summarising the weekly count.
  • List. role="list" with aria-label={t('errors.list.aria')}; each row is role="listitem" with the ISO timestamp on the time chip as aria-label.
  • Sheet. role="dialog" + aria-modal="true"; focus moves to the textarea on open; Escape closes; the scrim button carries its own dismiss label so a screen reader user can leave without saving.
  • Skeletons. Loading skeleton carries an aria-label so SR users get a loading announcement rather than silence.
  • Reduced motion. Assemble collapses to instant; sheet slide-up collapses to instant via the prefers-reduced-motion guard in sheet.css.
