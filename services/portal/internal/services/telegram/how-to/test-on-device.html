Test on a real phone via Telegram

Get the local dev app open on your phone, inside a real Telegram WebView, via a BotFather test bot and a Cloudflare Tunnel. Needed when you're validating haptics, safe-area insets, or Telegram-chrome behaviour — anything mockTelegramEnv() can't fake.

Before you start

  • App running locally — see Run the Mini App locally first. Confirm http://localhost:5173 works in your browser.
  • API reachable at a stable local URL — make up from repo root; curl http://localhost:8000/ready returns 200.
  • Cloudflare account (free tier is enough) and cloudflared installed: brew install cloudflared.
  • Telegram bot registered via BotFather with TELEGRAM_BOT_TOKEN in your env/dev. See BotFather docs.

Steps

  1. Expose the Vite dev server via Cloudflare Tunnel. In a fresh terminal: 
    cloudflared tunnel --url http://localhost:5173
    Cloudflare prints a URL like https://random-words-abc123.trycloudflare.com. This is the tunnel origin — public HTTPS pointing at your local Vite. Keep this terminal open; closing it kills the tunnel.
  2. Also expose the API (in another terminal) so the app can call it from the phone: 
    cloudflared tunnel --url http://localhost:8000
    Note this second URL — pass it to Vite as VITE_API_BASE_URL on the next dev-server restart.
  3. Add the tunnel API origin to backend CORS. Edit env/dev, append the API tunnel URL to CORS_ALLOW_ORIGINS. Restart the API (make down && make up).
  4. Restart Vite pointing at the tunnel API. 
    VITE_API_BASE_URL=https://api-tunnel-url.trycloudflare.com make -C services/telegram dev
  5. Register the tunnel URL with your bot in BotFather. Open @BotFather in Telegram → /mybots → pick your bot → Bot Settings → Configure Mini App → paste the Vite tunnel URL. Give it a short-name (e.g. study_app).
  6. Open the bot on your phone. Find your bot in Telegram, send /start. Tap the launch button that appears. The Mini App opens inside Telegram's WebView, backed by your laptop.

Remote DevTools (iOS + Android)

Once open on the phone, you can debug the WebView from your Mac:

  • iOS Telegram: connect the iPhone via USB, enable Web Inspector on the phone (Settings → Safari → Advanced), open Safari on Mac → Develop menu → your iPhone → the Mini App page. Full DevTools.
  • Android Telegram: enable USB debugging, connect via USB, open chrome://inspect in desktop Chrome, find the WebView, click inspect. Full Chrome DevTools.
  • Telegram Desktop: right-click inside the Mini App → Inspect. Full DevTools embedded.

What you can now verify

Things that a plain browser tab can't:

  • Haptics. Every swipe / grade / tap that has a haptic token fires on the device.
  • Real theme. Whatever accent + light/dark mode your Telegram is in — your themeParams reflect it.
  • Safe-area insets. Notch / home-indicator on iOS handled correctly.
  • Native chrome buttons. BackButton, MainButton, SettingsButton — all wired via the SDK, visible in the Telegram UI.
  • Deep-links. Bot sends a message with startapp=conspectus_<uuid>, tap → app opens on the right screen.

Troubleshooting

SymptomFix
Bot launch button opens Telegram's «Sorry, this Mini App is not available» screen.The URL in BotFather is wrong or the tunnel is down. Verify the tunnel URL opens in a normal browser first, then re-set it in BotFather (Configure Mini App → paste the current tunnel URL).
App opens but every request fails with CORS in DevTools.The API tunnel URL isn't in CORS_ALLOW_ORIGINS. Update env/dev, restart the API. Cloudflare's tunnel URL changes on every restart, so this bites often.
App opens on the phone but shows the wrong theme (always light).You're likely on mockTelegramEnv() — check src/main.tsx. The shim should be dev-only; in prod it must be off. Also confirm the URL you registered in BotFather is the tunnel, not http://localhost:5173.
Tunnel URL changes every time you restart cloudflared.Expected for the free ephemeral tunnels. For a stable URL, register a Named Tunnel with a Cloudflare-registered domain: see Cloudflare docs.
cloudflared refuses to start on macOS with a certificate error.Run cloudflared tunnel login once to install the certificate.
