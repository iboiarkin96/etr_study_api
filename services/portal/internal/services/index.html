Spine structure

Six-page spine

Every catalog entry exposes the same six pages per RFC 0004 §4.4. Uniform structure means you can navigate any service the same way without hunting.

01

Entity card

Owner, lifecycle, what it provides and depends on.

02

Architecture

C4 L3 component diagram, repository layout, design notes.

03

API reference

Contracts, functional requirements, critical writes, sequences.

04

Runbooks

Symptom index, NFR targets, failure modes, recovery steps.

05

Dependencies

Upstream and downstream — runtime, build-time, consumers.

06

On-call

Ownership, health signals, alert routing, symptom → action.
