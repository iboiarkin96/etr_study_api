Failure cascade — what breaks what

Read each row as: this component dies → this is what a user or operator actually experiences → this is the first move. The api → postgres edge is the only hard runtime dependency in the system; everything else degrades independently.

Honest limits of this topology

  • One host, one volume, no backups. A disk failure loses the database; ADR 0037's backup follow-up is the open mitigation. Until a restore drill has run, recovery time is unknown, not "fast".
  • Blind-but-healthy is a real mode. The observability stack is optional by design, which means the system can run unmonitored without anything failing loudly. Acceptable while the operator and the user are the same person; the first external user changes that calculus.
  • No horizontal path yet. The in-process rate limiter pins the api to one instance (see architecture narrative); this is a documented constraint, not an oversight.
