Partially updates a user's editable fields.

Overview

Partial update of a user. The body carries any subset of the editable user fields (full_name, username, timezone, invalidation_reason_uuid, is_row_invalid); composite identity fields are immutable.

1. Business context

Lets the integrator update the learner's display name, switch timezone, or flip the soft-invalidation flag without affecting downstream ownership relationships (the learner's client_uuid never changes).

2. User stories

  • US-USER-PATCH-1. As an integrator, I want to update the learner's timezone after they relocate.
  • US-USER-PATCH-2. As an integrator, I want to soft-invalidate a row when an account is offboarded, without losing study history.

10. Side effects

  • On success. One transaction: UPDATE users SET fields, updated_at = now(); INSERT INTO idempotency_keys; emit patch_user_succeeded (info).
  • On idempotent replay. No DB writes; emit patch_user_idempotent_replay (info).
  • On error. Rollback.
  • Tables touched (per ADR 0029): users (write); reads FK references in systems, invalidation_reasons, timezones.

11. Idempotency & concurrency

  • Namespace: PATCH /api/v1/user/{system_uuid}/{system_user_id} with concrete path values.
  • Key composition: Idempotency-Key + SHA-256 of canonical body.
  • Replay status: 200 with stored body.
  • TTL: 24 h.
  • Concurrency: v1 — last writer wins.

13. Edge cases checklist

  • Patch sets username to empty string. Accept (clears the username).
  • Patch with unknown field. Pydantic extra=forbid → 422.
  • Patch ID-fields. Identity fields are read-only via this endpoint; ignored if sent.
  • Concurrent PATCH on same row. Last writer wins; updated_at reflects the last commit.
  • Idempotency replay across a later edit. Returns the original stored body; not the current row.
