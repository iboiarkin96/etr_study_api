api · Dependencies

What the api service depends on (upstream) and what depends on it (downstream). Mirrors the Backstage spec.dependsOn / consumedBy graph at a service level. Pinning policy: ADR 0019 (pip-audit + version pinning).

Upstream — runtime dependencies

ComponentKindUsed for

FastAPI + UvicornWeb framework + ASGI serverRequest handling, OpenAPI generation, dependency injection. Process model in C4 L3.

DatabaseRelational storeSQLite for dev; Postgres-capable schema (ADR 0028 phase 2). Single primary, no read replica.
AlembicMigration toolSchema evolution; runbook on failure: 0002.
Pydantic v2Validation libraryRequest/response schemas, JSON validation per security defaults.
structlogLogging libraryNDJSON structured logs (ADR 0023) consumed by monitoring.
Redis (optional)In-memory storeDistributed rate-limit counters and idempotency cache. Process can run without when scaled to a single instance.

Downstream — who consumes this service

ConsumerSurfaceNotes
