Is the api broken or is monitoring broken?

The single most useful triage step. Run these checks in order:

  1. curl -s :8000/ready on the api host. If 200 → the api is up; the issue is in monitoring.
  2. curl -s :8000/metrics | head. If you see the expected metric names → exposition is fine; check Prometheus targets page (:9090/targets) for a scrape error (TLS, DNS, firewall).
  3. Prometheus targets page shows the scrape error code. Map to 0008.
  4. If /ready returns 503 → the api itself is degraded. Switch context to api · Runbooks.

Dashboard / rule recovery

Dashboards and alert rules should live as JSON in version control so a wipe is a re-import, not a redesign. If a dashboard disappears or is corrupted in Grafana:

  1. Find the JSON in the repo (typically under a dashboards/ directory adjacent to the Compose file).
  2. Import via Grafana UI: Dashboards → New → Import → paste JSON, select the Prometheus data source.
  3. If JSON is missing, mark this as a backlog item — the canonical copy must live in Git.
