Table: timezones

Table timezones Category reference / lookup Status Source app/models/reference/timezone.py Owner Ivan Boyarkin Last reviewed 2026-05-08 Spec template v1

Summary

IANA timezone catalogue keyed by zone string (e.g. Europe/Moscow) with the precomputed UTC offset in minutes. Used as the FK target for users.timezone so that every user has a known, normalisable zone for due-date and review-window arithmetic. This table is one of the few in the schema that uses a string code as PK rather than a UUID — see ADR 0029 §1 (reference tables may use a stable string PK when the value is itself the canonical identifier in an external dictionary).

Schema
