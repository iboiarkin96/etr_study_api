Table: systems

Table systems Category reference / lookup Status Source app/models/reference/system.py Owner Ivan Boyarkin Last reviewed 2026-05-08 Spec template v1

Summary

Catalog of upstream systems that own user identities (e.g. an LMS, an HR system). Every users row is keyed by a (system_user_id, system_uuid) pair; this table is the canonical list of valid system_uuid values plus a human-readable name.

Schema

ColumnTypeNullableDefaultFKIndexDescription
system_uuidString(36)nouuid4()—PKStable identifier exposed in API responses.
codeString(64)no——uniqueShort, stable, human-readable code (e.g. "lms-prod").
nameString(255)no———Display name shown in UIs and logs.

Constraints

  • Primary key: system_uuid.
  • Unique constraint: code (a source system has exactly one stable code).

Relationships

  • 1-N to users via users.system_uuid.

Lifecycle

  • Created by: seed migration / admin tooling. No public HTTP endpoint creates rows.
  • Read by: all users read paths join on system_uuid.
  • Retention: seeded values are immutable; deprecation marks the system as historical rather than removing the row.
