Table: invalidation_reasons

Table invalidation_reasons Category reference / lookup Status Source app/models/reference/invalidation_reason.py Owner Ivan Boyarkin Last reviewed 2026-05-08 Spec template v1

Summary

Lookup table for the why half of the soft-delete contract (ADR 0029). When a core row is marked invalid (e.g. users.is_row_invalid = 1), the row's invalidation_reason_uuid FK MUST point at a row in this table. The catalog is small (a handful of canonical reasons) and serves as an auditable, machine-readable explanation for why a row is no longer authoritative.

Schema

codeString(64)no———Short stable code (e.g. "merged", "data-corrected").

Constraints

  • Primary key: invalidation_reason_uuid.

Relationships

  • 1-N to users via users.invalidation_reason_uuid (and to any future core table that uses the soft-delete pair).

Lifecycle

  • Created by: seed migration / admin tooling.
  • Read by: any operation that emits is_row_invalid = 1 reads the matching reason for the audit trail.
