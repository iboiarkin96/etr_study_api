Screen map: <screen name>

Canonical template for the pin-map subgenre of screen specifications — a live Storybook iframe annotated with numbered pins that map every visible region to its React component, hook, endpoint, and CSS primitive. Used under services/portal/internal/services/<svc>/reference/screens/. Sibling of the enterprise Screen spec template — pick this one when the screen has < 12 visible regions and benefits from a pin-annotated iframe; pick Screen spec when it needs the full anatomy + responsive + browser matrix treatment.

When to use this template

  • Use it when a screen has a stable enough shape to freeze into a Storybook fixture, and the visible regions can be enumerated in ≤ 10 pins on a single mock.
  • Don't use it for pages that lack a canonical layout (dashboards with widget slots, editor surfaces), for cross-viewport variants that need side-by-side comparison, or for pages that need multi-page acceptance tables — reach for Screen spec.
  • Sizing contract: every screen-map under one service must render at the same iframe width. Override only --live-h per page; do not touch --live-w / --live-scale without auditing all sibling screen-maps.
  • Ten sections in order: Metadata · Purpose · The map · States · Hooks & endpoints · Interactions · Accessibility · Copy · Files · History. Optional per-screen extras (e.g. Grade → tag mapping, Copy variants) slot after Interactions.
