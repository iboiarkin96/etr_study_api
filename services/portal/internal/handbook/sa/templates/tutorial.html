Authoring notes (for the writer — delete before shipping)

When to use this template. The reader is a beginner in this domain or stack; tangible output in 15–60 min; one main path (no decision trees); real production work uses the same shape — this is a scaled-down rehearsal, not a toy. If any of these is false → use How-to playbook (task, expert reader) / Reference (lookup) / Explanation (why) instead.

Lede formula. Three imperative verbs in a row, then «the point is to feel … not to …» contrast. Anchors the reader: rehearsal, not production task.

Role pill colour. info for SWE / SA / Architect; teal for QA; danger for SRE; warn for Manager. Last chip is 💻 Local stack for code-driven, 📝 Paper-only for whiteboard / study.

Voice rules. Imperative throughout («Create», «Run», «Open», «Confirm»). Second-person sparingly. No decision trees in the main path. Conditionals ONLY inside checkpoint cards. Cite real artefacts (ADRs, runbook IDs).

Anti-patterns. Lane A / Lane B in the main path → demote to ghost-card shortcut. Long lecture before any action → defer the «why» to Explanation. 80-step megapages → split if > 6 phases. «This is a placeholder» — never ship without verifiable output. Production-volume task as the rehearsal — scale it down.

Checkpoint cards. Every phase except the last ends with the same shape — that is where conditionals belong, never in the main step body. The main step body is always imperative and unconditional.

Before shipping. Delete this «Authoring notes» card from the final tutorial — it's guidance for the writer, not the reader.
