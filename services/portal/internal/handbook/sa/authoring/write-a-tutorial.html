Write a tutorial

Recipe for writing a Diátaxis tutorial: learner-first, hand-held, no conditionals, ends with a feeling of «I did the thing». Copy the tutorial template, fill the phases, the checkpoint cards, the «Next step» close.

Before you start

  • You can name the concrete thing the learner will produce («your first ADR», «your first endpoint», «your first capacity drill»).
  • The learner is a beginner in this domain — they don't know the vocabulary, can't pattern-match yet.
  • You've opened handbook/sa/templates/tutorial.html.

Steps

  1. Place the file under tutorials/<role>/your-first-<thing>.html.
  2. Set body data-page-type to "tutorial" — the live page BE'S a tutorial, even though the SA template carries reference.
  3. Write a verb-first h1: «Your first ADR» / «Your first endpoint».
  4. Open with Before you start (step –) — what the learner needs (Python ≥ 3.12, repo cloned, branch checked out).
  5. State the Goal (step 0) — one sentence: what's on the screen when you're done.
  6. Break the body into numbered phases (1, 2, 3, …). Each phase is a single block of work with one outcome. Drop a checkpoint card after every phase except the last.
  7. Use imperative voice. Never «давайте», «we will», «you can». Always «Run», «Open», «Confirm».
  8. No conditionals. Tutorials are one straight path — if you need branches, write a how-to instead.
  9. End with «You're done» plus an optional «↺ Want to start over?» reset card.
  10. Close with a «Next step» aside — title exactly «Next step» (per memory rule); 1–3 links to the obvious next move (how-to, reference, deeper tutorial).
  11. Add a page-history footer.

Verify

  • data-page-type="tutorial" on the live page (template carries «reference»).
  • H1 is verb-first, names the thing the learner produces.
  • Before-you-start (step –) and Goal (step 0) present.
  • Phases numbered 1..N; checkpoint card after each except the last.
  • «Next step» aside present, title exactly «Next step».
  • No conditionals («If X…») anywhere in the body.
  • pytest tests/visual/test_tutorial_canon.py green when parametrised over this file.

Troubleshooting

  • «Conditionals creep in». «If on macOS: …», «If using Postgres 15: …» belong in a how-to, not a tutorial. Either pick one path and document it, or split into two tutorials.
  • «Tutorial feels like a recipe». If the title is «How to X» — it's a how-to. Tutorial titles are «Your first X».
  • «Authoring notes leak into the reader-facing flow». Drop guidance prose into an «Authoring notes» ghost card at the end that the writer deletes before shipping.
