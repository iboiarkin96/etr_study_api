Write a foundation spec

Recipe for documenting a UI Kit foundation (tokens / type / motion / hotkeys / layout). Copy the foundation-spec template, fill the values, the usage rules, the code handoff.

Before you start

  • The foundation is implemented and used by at least two components.
  • You can name the foundation in one phrase («Color tokens», «Motion easings», «Spacing scale»).
  • You've opened handbook/sa/templates/foundation-spec.html.

Steps

  1. Place the file at services/portal/ui-kit/pages/foundations/<foundation>.html.
  2. Write an overview — what the foundation is for, in one paragraph.
  3. List tokens / values in a table. Each row: name · value · usage example.
  4. Document usage rules — when to use which token, what's forbidden («never hard-code hex colours», «never inline pixel margins»).
  5. Show examples — at least three uses across different contexts.
  6. Code handoff — how the foundation lives in CSS (custom properties, design-token JSON, JS constants).

Verify

  • Overview / values / usage rules / examples / code handoff sections present.
  • Token table includes name + value + usage example for every entry.
  • At least three real examples cited.
  • Listed in UI Kit foundations index.

Troubleshooting

  • «Foundation has one consumer». Not a foundation yet — wait for the second use case, otherwise the abstraction is premature.
  • «Token names drift across files». Single source: the CSS custom-property declarations under assets_v2/ui-kit/tokens/. The spec is generated from / mirrors that file.

Related

  • handbook/sa/templates/component-spec.html — sibling form for components that consume foundations.
