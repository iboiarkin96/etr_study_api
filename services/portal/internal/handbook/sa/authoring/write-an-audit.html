Write an audit

Recipe for authoring a dated assessment under governance/audits/<area>/. Copy the audit template, set the date in the filename, fill the scoring tables, write the architect's-bar verdict, list the top gaps.

Before you start

  • You have an area to assess (docs, ui-ux, api, maturity, bugs, …) and a cycle date (audits are dated snapshots, not living docs).
  • The previous cycle's audit for the same area is open in another tab — you'll cite deltas against it.
  • You've opened handbook/sa/templates/audit.html and have it side-by-side with this page.

Steps

  1. Name the file. governance/audits/<area>/YYYY-MM-DD-<area>-assessment.html. The date is the cycle date, not the publication date.
  2. Copy the template. Keep data-page-type="reference" — audits are reference snapshots, not procedures.
  3. Fill Metadata. Cycle date · Area · Previous cycle (link) · Owner · Last reviewed.
  4. Write Overview. One paragraph: what's improved, what stalled, what regressed. No surprises in the body.
  5. Fill the scoring table. One row per practice in the canon; columns: practice · evidence · justification · score (X/10) · delta vs previous cycle.
  6. Write the architect's-bar verdict. Letter grade per axis (A+/A/A−/B+/B/B−/C+/C/C−/D+/D/F), one-line justification, ten axes.
  7. List the top five gaps with priority (P0/P1/P2), status (TODO/IN PROGRESS/DONE), and the backlog ID that closes each.
  8. Cross-link. Add the new audit to the area's audit index. Link back to the previous cycle in Metadata.
  9. Run drift gate. make -C services/portal docs-fix && make -C services/portal docs-check.

Verify

  • Filename matches YYYY-MM-DD-<area>-assessment.html.
  • data-page-type="reference" on <body>.
  • Metadata <dl> surfaces at least Cycle date · Area · Owner · Last reviewed.
  • Every scoring-table row has a numeric score and a delta arrow.
  • Architect's-bar verdict has ten axes filled with letter grades.
  • Top-five-gaps table has five rows with non-empty priority and status.
  • make -C services/portal verify-local green.

Troubleshooting

  • «Audit feels like a runbook». Audits are reference snapshots; they document state, they don't tell you what to do. If the body is procedural, it belongs in a runbook or how-to.
  • «No delta column for new rows». Mark «—» (em-dash). Don't invent comparisons against absent prior cycles.
  • «Scoring inflation». If almost every row scored 9–10, the rubric is too soft or the assessor is too kind. Re-score with one fewer point until the differential reads.
