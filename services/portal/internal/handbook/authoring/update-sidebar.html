Update the sidebar

The sidebar is hand-maintained — there is no automatic projection from the filesystem yet. Adding any page requires editing two files. Forget one and the page becomes a 404 for anyone who tries to navigate to it.

The two files

  1. Primary manifest (UI Kit v3): 
    services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json
    JSON tree consumed by the sidebar Web Component (ui-kit/components/sidebar.js). Pages declare their nav source via the data-nav-src attribute on the <aside>.
  2. Legacy projection: 
    services/frontend/portal/assets/internal-sidebar.js
    Hand-maintained JS array still consumed by some shells. Keep both in lock-step until the legacy file is retired (planned Phase 2).

Acronym casing

Labels must preserve the casing of acronyms: API, AsyncAPI, SRE, gRPC, REST, SQL, ADR, RFC, QA, CI. Verify the label every time you touch a page — drift here is the most common navigation bug in the portal.

Shape of a nav-tree-internal.json entry

{
  "id": "handbook-templates-adr",
  "label": "ADR",
  "href": "/services/portal/internal/handbook/templates/adr.html"
          }

Rules: id is dot-free, kebab-case, prefixed with the section path. href is absolute from the repo root, must start with /services/portal/internal/, must end with .html. Groups (no href, with children) are fine.

Shape of an internal-sidebar.js entry

{ label: "ADR", path: "internal/handbook/templates/adr.html" }

Rules: path is relative to services/portal/, no leading slash. label uses sentence case with acronym preservation. Groups have children: [...].

Where to put the new entry

  • API operation → under reference → entity hub → Operations
  • ADR → under governance → ADRs → the topical cluster («Documentation», «Security», «API contract», etc.)
  • Runbook → under how-to → Runbooks
  • Tutorial → under tutorials → role group
  • Reference page → under the relevant reference block (api, data, front, qa, manager, uml)
  • Handbook page → under handbook in the right sub-group

Validate before commit

python3 -c "import json; json.load(open('services/frontend/portal/assets_v2/ui-kit/mocks/nav-tree-internal.json'))"

Catches trailing commas and unbalanced braces. The JS file is harder to lint without a Node runtime — eyeball the brace/bracket balance. make docs-check will run internal-sidebar regression assertions for you.
